# 官网视觉宪法 v0.2

版本：v0.2（2026-07-17，增补野心条款与排版三声部/肌理之辨/悬停语义；v0.1 同日首版）
地位：介于《官网叙事规格书》《官网文案》与 `sites/marketeam/index.html` 之间的**中间产物**——从叙事推导视觉的那半条链。页面实现只许引用本宪法与样张（`sites/marketeam/style.html`）中定义的令牌与组件。
使用规则：每条规则必须附带推导理由；无法说出"这条规则在重复哪句潜台词"的规则不得写入。

---

## 〇 · 潜台词（全部视觉决策的唯一出处）

> **你可以走开，因为每一步都有人管、有痕迹。**

（＝规格书中心论点"放手而不失控"的视觉版。）每个视觉决策在评审时要回答同一个问题：**这个选择在重复这句话吗？**在跟它唱反调的，砍。

## 一 · 体裁引用：跟踪单的骨架，不是纸的戏服

受众在日常里亲历过"放手而不失控"的原型是**物流跟踪 / 回执 / 检验单**：交出去的东西在一条轨道上被推进，每站有时间戳、有经手方、可回查。我们引用这个体裁记忆。

注意：真实的快递跟踪页**没有任何纸的拟物**——没有纸纹、锯齿边、邮戳。它被信任靠的是**结构**（站点 + 时间戳 + 状态 + 可回查），不是纸的长相。所以我们继承的只有骨架：

- 站点序列（任务在哪一站）
- 时间戳与经手者（谁、何时）
- 状态流转（对齐 / 通过 / 退回）
- 可下钻的追溯（每一步可展开看原因）

不继承的道具：纸纹、锯齿撕边、橡皮章、回形针、咖啡渍、手写体。

## 二 · 设计系统规则（每条附推导）

1. **色彩即状态。**语义色只有三个——对齐蓝、通过绿、退回橙——且只许出现在"过程正在发生"的图形里；装饰一律用墨色阶。
   ↳ 推导：颜色是感知线索（规格书"一套一致的感知线索"）；线索一旦装饰性使用就成了噪声。
2. **"你"用形不用色。**决策点（该你拍板的时刻）用菱形站点等**结构差异**表达，不再占用一个颜色。
   ↳ 推导：旧版粉色同时表示"你"和"不通过"，语义重叠；形状通道比色彩通道更稳。
3. **文字双轨。**叙事用 Sans（Noto Sans SC），一切过程痕迹（编号、时间戳、状态、单据行）用 Mono（JetBrains Mono）。不得混轨。
   ↳ 推导："讲话的字"与"留痕的字"分轨，正是"主张 vs 底单"这对核心概念的排版化。
4. **动效预算：同屏同一时刻只有一个演员。**叙事讲到哪，哪里才许动；其余静止。入场动效全站只用样张定义的 3–4 种原语。
   ↳ 推导：这条规则本身就是"该问你时才来找你"的视觉翻译。一个什么都在动的页面在反驳自己的文案。
5. **动画一律是 t 的纯函数或显式状态机。**核心动画写成 `render(t)`，可用 `?t=` 冻结到任意相位；板块状态可用 `?state=` 强制。禁止散落的一次性 keyframes 编排。
   ↳ 推导：可冻结 = 可截图核对 = 可验收。这是把"每份产出可验收"用在页面自己身上。
6. **数字纪律。**页面上任何像数据的内容（时间戳、次数、编号）必须落在带"示例记录 · 非产品实拍"标记的容器内。
   ↳ 推导：规格书三禁令的视觉执行。
7. **降字重。**900 字重只留给 H1/H2；正文与组件不超过 700。描边用细线（1.5–2px）与表格线，硬投影贴纸阴影退场。
   ↳ 推导：我们要的感觉是"安静、有章法"；视觉音量喊起来，文案就白写了。
8. **排版三声部。**判断用衬线（Noto Serif SC 900：H1/H2/判决句），解释用无衬线（正文），记录用等宽（过程痕迹）。全页字号动态范围不小于 7×（最大标题 ÷ 最小标注）。
   ↳ 推导：判断、解释、记录是页面的三种发声位置；声部不分，一切同音量。编辑部式的安静靠极端反差实现，不靠压缩动态范围——**安静 ≠ 扁平**。
9. **肌理与戏服之辨。**参数化印刷肌理（点阵网格、半调、套准标记、发丝线）允许且鼓励，集中用于中心作品；戏服（纸纹、撕边、橡皮章、咖啡渍等假装成实物的拟物）仍然禁止。判据：肌理是表面自己的内容，戏服是假装成别的东西。
   ↳ 推导：§三.3"最低戏剧化"禁的是假装，不是丰盛；v0.1 被过度应用成了零材质。
10. **悬停语义。**悬停必须揭示关系（如：底单行 ↔ 轨道站点同亮），不做纯装饰性变换。
   ↳ 推导：交互纵深让组件显形为同一个系统，这是数字介质里的质感。

## 三 · 设计禁令

1. **比喻名词不入画。**设计/文案讨论里的比喻（椅子、抽卡、印章、纺纱机……）是指针，指向结构或感受；**图像没有引号**，画出来就成了产品世界里的实物。比喻归文案，结构归视觉。
2. **指着测试。**指着页面上任何一个图形问："这是产品世界里的东西吗？"答案若是"它是规格书里的一个比喻"，删除。产品原生物件（终端窗、底单记录、任务卡）可以在场。
3. **最低戏剧化。**允许在场的原生物件只保留功能所需的最低表达："非实拍"标记做成等宽字标签即可，不做成旋转的红橡皮章。功能可以留，戏服要脱。
4. **禁徒手路径。**所有图形必须是参数化几何（网格、轨道、站点、表格行、直线）或排版产物（字符、编号、时间戳）。禁止手绘曲线、有机形状、吉祥物、场景插画。
   ↳ 附带收益：参数化几何是坐标算出来的，数学上不可能"画歪"——风格选择与生成者能力在此闭环。
5. **翻译表前置。**文案中每个比喻名词进入视觉层之前，必须先过下面的翻译表；表里没有的名词不得直接出现在图形中。
6. **组件封闭集。**`index.html` 只许使用样张（`style.html`）中定义的令牌、组件与动效原语。新组件先进样张，评审后再进页面。

## 四 · 翻译表（名词 → 结构 → 页面原生表达）

| 讨论中的名词 | 它实际指的结构/行为 | 视觉层的原生表达 |
|---|---|---|
| 椅子（角色移交） | 位置先于占用者存在；占用可移交；空位可见 | 轨道站点常驻；经手者标签从"你"逐个换成角色名；换手瞬间是唯一动效 |
| 抽卡 | 同样输入，产出方差大到不可依赖 | 词留在文案标题；视觉用"同一任务多份记录、结果不同"的对照行表达 |
| 印章 / 戳 | "非实拍"声明；"已验收"事件 | 等宽字标签 `示例记录 · 非产品实拍`；验收事件 = 底单里一行绿色 PASS 记录 |
| 底单 | 可回查的过程记录 | 可下钻的事件行：时间戳 + 状态 + 原因 + 经手，点击展开 |
| 章法 | 何时问、何时返工的规则在稳定运转 | 动效预算 + 状态机本身：页面只在该动时动，就是章法的体感 |
| 团队 | 多角色各司其职、相互交接 | 站点与经手者标签系统；不画小人、头像、办公室场景 |

## 五 · 野心条款（v0.2 新增：禁令防坏，野心防平）

1. **中心作品。**每页恰好一件倾注不成比例工艺的中心作品（当前：首屏跟踪板），占整页工艺预算约一半；其余板块保持克制——反差本身是设计。
   ↳ 推导：质量感知是非线性的，一处非凡拉高整页，十处合格仍是合格。可见的劳动是与"底单"同源的诚意信号：页面自己就是第一份带工艺的交付。
2. **反差锚点。**每一屏至少一处显著反差（尺度、明度、密度任一维度），如旧世界第四拍的墨底反转。均匀的中等 = 没有节奏。
3. **留白必须被安排。**空是构图的指向性元素，不是剩余；每处大面积留白要能回答"它把视线让给了谁"。

## 六 · 执行与验收

- 样张：`sites/marketeam/style.html`（令牌 / 组件 / 动效原语 / hero 原型的唯一来源）。
- 评审清单：① 潜台词测试（§〇）② 指着测试（§三.2）③ 色彩审计（语义色是否出现在非过程图形里）④ 动效预算审计（同屏几个演员）⑤ 数字纪律审计（示例标记齐否）⑥ 声部审计（衬线/无衬线/等宽各就其位，动态范围 ≥7×）⑦ 反差锚点检查（每屏一处）。
- 本宪法修订须同步检查样张与页面，禁止只改单处（同规格书使用规则）。
