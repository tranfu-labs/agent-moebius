# 官网叙事规格书 v1.6.5

版本:v1.6.5（2026-07-13，补回叙事观察性与受众亲历仲裁原则）
状态标注:【定稿】= 已逐字打磨;【初稿】= 骨架层共识,细节待讨论
使用规则:任何上游共识的修改,必须沿文档向下检查所有受影响板块,禁止只改单一板块。已废弃的旧表达不得回流到后续文案或页面方案中。

## 第〇部分 · 上游共识(全页依据)

### 页面目标与语域
首要目标:**阐明思想**——把"可信的自动才稀缺"这一论点讲清楚,让对的人读完说"这就是我一直说不清的那个问题"。转化、证明、案例均为后置目标。
语域纪律:全页使用**论述语域,不用取证语域**。产品尚未完成,页面不承担自证义务;页面表达的角色是**图解主张的思想实验**,不是产品实拍的证物。唯一硬约束:不把概念表达包装成产品实拍。
语言基准:全页以**普通人话**为基准,不因开发者占比高而放宽——普通人话不冒犯开发者,开发者黑话直接流失非开发者。所有词汇选择均以此受众事实为依据。

### 产品事实与页面前提
产品是开源项目,以 Electron 为基础的 macOS Apple Silicon 桌面应用,驱动用户本机已安装并登录的 Codex CLI / Claude CLI(任一即可,可同队混用;CLI 能运行即可)。正式版本仅提供 arm64 DMG 与 ZIP,不提供 Intel Mac、universal、Windows 或 Linux 版本。产品无注册,不收 key。
这些是页面需要如实陈列的产品前提,不是页面叙事或视觉实现的规定。产品不卖模型,只提供"章法"这一层,主张与产品形态保持一致。

### 受众规格
主打两类人,统一于同一条痛点因果链:
其一,有痛点的用户——曾把任务整个交给自动化产品,体验过"进度条哗哗跑,交回来不能用",正在主动比较产品,是最大概率的首批访客;
其二,用过聊天式 AI 的普通用户——回合制使用,说一句动一下,曾"陪聊一下午",因等待被迫在多条业务线间切换。
统一痛点:**交付不可信 → 不敢放手 → 被迫在场(监工/陪聊/多线切换)**。质量不可信是因,不敢离场是果。
共同底座:两类受众统一于"本机已有并登录 Codex CLI / Claude CLI 的人"。开发者占比预计较高,但非开发者体量可观——CLI 用户不等于开发者,很多人本就拿它们干调研、写文档等非代码活。
待验证假设:首批访客以第一类为主。上线后需用访谈或投放数据校验;若第二类占比远超预期,板块二的微场景配比需调整。

### 中心论点
自动早就有了,**可信的自动才稀缺**。我们卖的不是"一句话就能跑"(那是行业标配),而是"放手而不失控"。

### 敌人
抽卡式交付。不是"手动方式",不是任何具名竞品。全页对竞品零点名、零暗讽,叙事位阶保持在"解释整个行业现状"之上。

### 主张类型纪律(无质量证据期的写虚三禁令)
1. 不出现任何无出处的数字;
2. 不使用无法验证的形容词(高质量、更可靠、企业级等,一律删);
3. 所有承诺必须写成**可被观察或感知的行为**,由页面表达**图解**——"质量高"改写为"每份产出送到你手上之前,至少完整验收一次";"流程可控"改写为"它只在该问你的时刻来找你"。表达方式可以变化,但不能退化为无法核对的形容词。页面表达是主张的可视化思想实验,不是证物。

### 隐喻与语言预算
主隐喻:**团队与章法**(一群人不等于一个团队)。丰田式质量思想(安灯绳、内建质检)只作为设计依据,不作为页面用词。
珍妮纺纱机:降级为可选的文案点缀,不承担叙事结构角色。
表达纪律:全页使用一套一致的感知线索表达工作推进与交付发生,具体视觉形式不在本规格中规定。
过程纪律:页面必须让角色、交接、检查、返工的先后关系清晰可理解;不得把叙事落成技术架构图,也不得让页面被误解为需要学习或搭建一套流程的工具。判断旧世界以受众亲历为准,不以行业内部事实替代。
词汇禁令:subagent、编排、orchestration、harness、多智能体等内部词汇不得出现在主叙事;它们属于开发者文档。任何具体配置能力也不进入主叙事。

### 六句话骨架(标题测试 v3)
连读应为完整故事:
你走开,它们继续 / 交付像抽卡,所以你不敢离场 / 缺的角色,一直是你在演 / 该问你时问你,该返工时自己返 / 每份交付,都该带一张过程底单 / 开源免注册,交出你的第一个任务。
第五句强调"都该带",因为板块五提出的是行业标准,不是现成证据。
第六句强调开源和免注册,同时把读者带向自己的第一个任务;不把页面包装成取证现场。

### 全页问题链
每板块出场问题即下一板块进场问题,链条为:
真的能走开?我被烧过 →(板块二收编怀疑)→ 一群自动化参与者离团队差什么? →(板块三揭示:角色都是你在演,并逐个移交)→ 接走的角色,靠谱吗? →(板块四说明:它们接手的是你原来干的活)→ 说得好听,痕迹呢? →(板块五提出标准:痕迹就该长这样)→ 怎么开始?(板块六回答从理解到行动的最短路径)。

## 板块一 · 首屏【定稿】

**进场状态**:冷启动访客,自带"又一个自动化产品"分类器,归类在极短时间内完成,归类即折价。
**唯一工作**:在归类完成前,用一件正在发生的事情拦截;差异化必须以行为呈现,让访客先理解发生了什么,再接收抽象主张。
**感受目标**:目击感,而非被推销感。访客愿意停下来理解这件事,并产生继续探索的欲望。最佳状态是"信一半"——信的那半产生渴望,疑的那半产生问题,问题是继续阅读的燃料。
**出场问题**:真的吗?我上次走开,回来是一坨(带伤疤的怀疑,由板块二收编)。
**失败迹象**:访客扫一眼即滑走(只做到好看,没做到让人理解);或产生"太复杂,不是给我用的"(表达过度炫技)。

---

## 板块二 · 旧世界【定稿】

**进场状态**:从首屏带着伤疤下来,心里最响的是反驳:"我试过,我走开过,回来是一坨。"
**唯一工作**:收编怀疑——替他把怀疑说出来,且说得比他自己更准,使其心理位置从"审视产品"切换为"这产品懂我"。
**感受目标**:被说中 + 我的怀疑被尊重。归因三不怪:不怪用户、不怪模型(怪模型会连带否定自己),只怪组织方式——"一群各干各的自动化参与者,本来就不是一个团队"。归因句同时预埋板块三的"团队"概念。
**出场问题**:一群自动化参与者,离一个团队差什么?
**失败迹象**:读起来像拉踩竞品;或痛点停留在"质量不高、流程不可控"这类内部语言(用户语言是"抽卡"与"陪聊了一下午");或过早进入机制解释(本板块只共情,不推销)。

## 板块三 · 转折【定稿 v4】

**进场状态**:带着"差什么?"而来,全页好奇心峰值;误读高风险区(访客可能把它理解成另一种需要学习的工作流工具)。
**唯一工作**:揭示旧世界的真相并当场翻转——那个团队并不缺角色,传话、对齐、验收一直都有人干,**只是全由你一人客串**;转折演的不是"规矩凭空长出",而是"你的角色被逐个接走"。此框架同时解释板块二的两种痛:亲自演角色的人走不开(陪聊一下午),弃演的人收抽卡(没人把关);原始方式亦然——定义是你写的、产出是你验的,角色仍是你。
**感受目标**:顿悟感,且这次顿悟是对自身处境的重新命名——"原来我不是在用工具,我是在给一群工具当团队"。表达应先让人理解发生了什么,再用文字确认,不把确认文字变成新的解释负担。
**出场问题**:接走的角色,靠谱吗?
**失败迹象**:把故事说成技术架构;让角色看起来像预先固定的系统模块;让旧世界看起来已经自动衔接;使用会把访客带向配置或流程搭建的视觉暗示;文字量超过叙事需要;关键转折无法被顺畅理解。

---

## 板块四 · 机制【初稿·待讨论】

**进场状态**:接受了"章法"概念,追问其运作;其中有痛点的用户在用怀疑者的眼光核对细节。
**唯一工作**:把章法具象成三个可理解的时刻,同时完成一次重定义——**打扰是特性,不是缺陷**:它来找你,是因为那一刻的决策权属于你。
**感受目标**:被打扰得恰到好处。访客应得出"它知道什么时候该闭嘴干活,什么时候必须问我"。
**出场问题**:说得好听,痕迹呢?
**失败迹象**:堕入功能说明书(罗列类型、参数、能力矩阵);只给出抽象名词,却没有让人认出"这是我以前干的事"的具体行为。
**信任转移线索**:三个时刻的可信根源 = 它们接手的正是你原来亲自干的活(对齐、验收、打回);表达应让访客从行为中认出自己的经验。
**素材来源**:场景与措辞优先取自真实工作过程中的对齐、验收和返工经验,保留真实语气,不把它们加工成功能宣传语。

---

## 板块五 · 底单【定稿 v2】

**重定位**:从出示证据转为提出标准。产品未完成,逐单证据暂不存在;本板块的位阶是提出行业标准,不是证明自身已经实现。
**进场状态**:机制听完,最后一层怀疑:"承诺是嘴上的,痕迹在哪?"
**唯一工作**:提出一个标准并使其可理解——**我们认为,每份 AI 交付都应该自带底单**:被打回几次、为什么、谁介入过、对齐过什么。这里呈现的是标准的形状,不是已有产品能力的承诺。
**感受目标**:意外感 + 认同感——"这个东西我在别家没见过"+"对,交付就该长这样"。
**出场问题**:怎么开始?(由板块六接住)
**失败迹象**:标准被美化成只有成功记录的愿景;打回、返工等不体面的过程被抹掉;把设想包装成产品实拍;或出现总体性数字承诺。

---

## 板块六 · 行动【定稿 v2】

**进场状态**:前面几板块读完,标准刚立完,末端问题"怎么开始?"带着具体期待而来——访客想知道自己离第一次使用还有多远。
**唯一工作**:明确一条从"读完的人"到"开始第一个任务的人"的最短路径;同时完成图解→实物的交接:页面上提出的对齐、打回、介入,应当能在实际使用中被访客感知。产品前提如实陈列,一步不藏。
**感受目标**:门是开的,且自己已半只脚在门里——"我已有的工具,原来就差这一层章法"。开源、免注册、不收 key,与首屏"敢展示慢下来检查"是同一血统的信号。
**出场问题**:无(全页终点)。若访客带走一个问题,应当是对着自己机器的"第一个任务交什么?"
**失败迹象**:人为造出产品没有的门槛;隐瞒 macOS、Codex CLI / Claude CLI 等使用前提;让页面重新变成需要学习或搭建流程的工具;用未经证实的数字或外部背书替代叙事;行动语言退回泛化的系统话术。
