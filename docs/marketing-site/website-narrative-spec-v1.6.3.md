## 第〇部分 · 上游共识(全页依据)

### 页面目标与语域(v1.5 新增)
首要目标:**阐明思想**——把"可信的自动才稀缺"这一论点讲清楚,让对的人读完说"这就是我一直说不清的那个问题"。转化、证明、案例均为后置目标。
语域纪律:全页使用**论述语域,不用取证语域**。产品尚未完成,页面不承担自证义务;演示与视觉的角色是**图解主张的思想实验**,不是产品实拍的证物。唯一硬约束:不把概念演示包装成产品实拍。
语言基准(v1.6.3 新增):全页以**普通人话**为基准,不因开发者占比高而放宽——普通人话不冒犯开发者,开发者黑话直接流失非开发者。词汇禁令与工位业务动词命名等既有选择,均以此受众事实为依据。

### 产品形态事实(v1.6 新增)
开源项目,macOS 桌面应用(Electron),群聊式界面:一行用户,多行 agent 轮流交接输出。无注册、无登录、不收 key——驱动用户本机已安装并登录的 codex cli / claude cli(任一即可,可同队混用;无订阅档位要求,cli 能跑即可)。内置预设团队(CEO、dev 等预设角色),开箱可跑,选择预设而非搭建配置。公开测试中。
此形态与中心论点字面咬合:"自动早就有了"有了具体所指——就是访客机器里已经登录的那些 agent;产品不卖模型,只做且只提供"章法"这一层,主张与商业形态零缝隙。群聊界面使过程天然可见:板块四的三个时刻在产品里不是后台日志,是逐条滚出的聊天消息;群聊记录即过程底单的粗坯。

### 受众规格
主打两类人,统一于同一条痛点因果链:
其一,有痛点的用户——曾把任务整个交给 agent 产品,体验过"进度条哗哗跑,交回来不能用",正在主动比较产品,是最大概率的首批访客;
其二,用过聊天式 AI 的普通用户——回合制使用,说一句动一下,曾"陪聊一下午",因等待被迫在多条业务线间切换。
统一痛点:**交付不可信 → 不敢放手 → 被迫在场(监工/陪聊/多线切换)**。质量不可信是因,不敢离场是果。
共同底座(v1.6.3 确认):两类受众统一于"本机已有并登录 codex/claude cli 的人"。开发者占比预计较高,但非开发者体量可观——很多人本就拿 claude code 当助手干调研、写文档等非代码活。cli 用户 ≠ 开发者,此等式作废。
待验证假设:首批访客以第一类为主。上线后需用访谈或投放数据校验;若第二类占比远超预期,板块二的微场景配比需调整。
当前形态注记(v1.6;v1.6.3 修正):产品为 macOS 桌面应用,驱动本机已登录的 codex/claude cli。页面只如实陈列此前提,不为任何受众另修转化旁路。原表述"前提对受众的筛选效应"作废——前提不是墙,前提即受众定义本身:页面面向的全体访客默认已在门内,无需向门外喊话。

### 中心论点
自动早就有了,**可信的自动才稀缺**。我们卖的不是"一句话就能跑"(那是行业标配),而是"放手而不失控"。

### 敌人
抽卡式交付。不是"手动方式",不是任何具名竞品。全页对竞品零点名、零暗讽,叙事位阶保持在"解释整个行业现状"之上。

### 主张类型纪律(无质量证据期的写虚三禁令)
1. 不出现任何无出处的数字;
2. 不使用无法验证的形容词(高质量、更可靠、企业级等,一律删);
3. 所有承诺必须写成**可被旁观的行为**,由演示**图解**(v1.5 修订:原为"由演示担保"。产品未完成,演示担保不了任何东西,但可以图解——它是主张的可视化思想实验,不是证物)——"质量高"改写为"每份产出送到你手上之前,QA 至少完整验收一次";"流程可控"改写为"它只在该问你的时刻来找你"。

### 隐喻与语言预算
主隐喻:**团队与章法**(一群人不等于一个团队)。丰田式质量思想(安灯绳、内建质检)只作为设计依据,不作为页面用词。
珍妮纺纱机:降级为可选的文案点缀,不再承担视觉或结构角色。
动效语言全页唯一:**琥珀色脉冲 = 一次工作交付**。所有板块的"活动感"只用这一种语言表达。
表现形式全页唯一:**横向业务流水线**(左→右为时间与工序),工位以受众的业务动词命名(检索、分析、撰写……),严禁抽象网络图与节点云;工位可动态增设(如架构师介入 = 新工位滑入),以避免"固定工序"的误读。
词汇禁令:subagent、编排、orchestration、harness、多智能体等内部词汇不得出现在主叙事;它们属于开发者文档。md 定义 agent 等大宗商品能力同样只进文档,不进主页。
v1.6 增补:"可配置各角色所用 cli"等配置能力同入禁令——"配置"一词在主页出现,访客分类器即滑向"工作流编辑器",全页"你走开"承诺当场反转。禁的是主页曝光,不是功能:README、文档、设置界面照常。主页仅保留其无害侧影:codex 与 claude 同队共事(无需解释配置即可看懂,且自带记忆点)。前置假设:默认配置开箱能跑(见附录 B-7)。

### 六句话骨架(标题测试 v3)
连读应为完整故事:
你走开,它们继续 / 交付像抽卡,所以你不敢离场 / 缺的角色,一直是你在演 / 该问你时问你,该返工时自己返 / 每份交付,都该带一张过程底单 / 开源免注册,交出你的第一个任务。
注(v1.5):第五句由"都带"改为"都该带",随板块五重定位(出示证据→提出标准)。
注(v1.6):第六句定为"开源免注册,交出你的第一个任务"(候选定稿,连读节奏待终稿前统一打磨);原"眼见为实"属取证语域,已废弃。

### 全页问题链
每板块出场问题即下一板块进场问题,链条为:
真的能走开?我被烧过 →(板块二收编怀疑)→ 一群 agent 离团队差什么? →(板块三揭示:角色都是你在演,并逐个移交)→ 接走的角色,靠谱吗? →(板块四演三个时刻:它们干的就是你原来干的活)→ 说得好听,痕迹呢? →(板块五提出标准:痕迹就该长这样)→ 怎么开始?(v1.6 复活:板块六回答"下载即用",门常开)。

## 板块一 · 首屏【定稿】

**进场状态**:冷启动访客,自带"又一个 agent 产品"分类器,归类在半秒内完成,归类即折价。
**唯一工作**:在归类完成前,用一件正在发生的真事拦截;差异化(可控)是行为级的,可被目击,故证据先于主张。
**感受目标**:目击感,而非被推销感。可检验判据:访客停留等演示跑完、想伸手交互。最佳状态是"信一半"——信的那半产生渴望,疑的那半产生问题,问题是滚动的燃料。
**出场问题**:真的吗?我上次走开,回来是一坨(带伤疤的怀疑,由板块二收编)。
**失败迹象**:访客扫一眼即滑走(只做到好看,没做到目击);或产生"太复杂,不是给我用的"(演示任务炫技过头)。

## 板块二 · 旧世界【定稿】

**进场状态**:从首屏带着伤疤下来,心里最响的是反驳:"我试过,我走开过,回来是一坨。"
**唯一工作**:收编怀疑——替他把怀疑说出来,且说得比他自己更准,使其心理位置从"审视产品"切换为"这产品懂我"。
**感受目标**:被说中 + 我的怀疑被尊重。归因三不怪:不怪用户、不怪模型(怪模型会连带否定自己),只怪组织方式——"一群各干各的 agent,本来就不是一个团队"。归因句同时预埋板块三的"团队"概念。
**出场问题**:一群 agent,离一个团队差什么?
**失败迹象**:读起来像拉踩竞品;或痛点停留在"质量不高、流程不可控"这类内部语言(用户语言是"抽卡"与"陪聊了一下午");或掏出任何机制词(本板块纪律:只共情,不推销,harness/QA/检查点一律不出现)。

## 板块三 · 转折【定稿 v4】

**进场状态**:带着"差什么?"而来,全页好奇心峰值;误读高风险区(访客分类器里的近邻类别:工作流编辑器)。
**唯一工作**:揭示旧世界的真相并当场翻转——那个团队并不缺角色,传话、对齐、验收一直都有人干,**只是全由你一人客串**;转场演的不是"规矩凭空长出",而是"你的角色被逐个接走"。此框架同时解释板块二的两种痛:亲自演角色的人走不开(陪聊一下午),弃演的人收抽卡(没人把关);subagent 等原始方式亦然——定义是你写的、产出是你验的,角色仍是你。表现形式遵循第〇部分:横向业务流水线。
**感受目标**:顿悟感,且这次顿悟是对自身处境的重新命名——"原来我不是在用工具,我是在给一群工具当团队"。抢先半拍机制不变:画面先演,标签后到,只做确认。
**出场问题**:接走的角色,靠谱吗?
**失败迹象**:画成抽象网络图或节点云(表现形式必须是横向业务流水线);工位用产品语言命名(必须用业务动词);旧世界画成工位间已有传送带(受众亲历中工位是断开的,人是唯一搬运工);已废弃视觉词汇回流;画布类暗示(网格底纹、拖拽手柄、配置面板、端点圆点);文字超出 H1 加三个短标签;快滚错过(sticky 加进度门控兜底)。

## 板块四 · 机制【初稿·待讨论】

**进场状态**:接受了"章法"概念,追问其运作;其中有痛点的用户在用怀疑者的眼光核对细节。
**唯一工作**:把章法具象成三个可目击的时刻,同时完成一次重定义——**打扰是特性,不是缺陷**:它来找你,是因为那一刻的决策权属于你。
**感受目标**:被打扰得恰到好处。访客应得出"它知道什么时候该闭嘴干活,什么时候必须问我"。
**出场问题**:说得好听,痕迹呢?
**失败迹象**:堕入功能说明书(罗列 agent 类型、参数、能力矩阵);三个时刻用图标加名词呈现而无微演示(名词不可信,对话片段可信)。
**信任转移线索(v1.3 新增,细节待深聊)**:三个时刻的可信根源 = 它们接手的正是你原来亲自干的活(对齐、验收、打回);微演示的措辞应让访客认出"这是我以前干的事"。
**素材来源(v1.6.3 新增)**:三段微演示对话优先取自自造案例(附录 B-3)的真实群聊记录——建造本官网这个任务中,对齐岗真问过的问题、QA 真打回过的稿、架构师真给过的建议,照真话改写。一石三鸟:措辞毛边有了来源;与板块五"标准的第一份实现"及 B-3 三线合一;巧合印证——本板块原示例"官网的目标用户,是开发者还是普通人?"正是该任务里现实中被问出并回答过的对齐问题。

## 板块五 · 底单【定稿 v2】

**(v1.5 重定位:出示证据 → 提出标准。产品未完成,逐单证据暂不存在;但换到论述语域,本板块位阶反而更高——出示证据是为自己辩护,提出标准是为行业立规矩,与第〇部分"叙事位阶保持在解释整个行业现状之上"同构。)**

**进场状态**:机制听完,最后一层怀疑:"承诺是嘴上的,痕迹在哪?"
**唯一工作**:提出一个标准并画出它的图纸——**我们认为,每份 AI 交付都应该自带底单**:被打回几次、为什么、谁介入过、对齐过什么。档案卡是这一标准的设计图,不是产品截图;明厨亮灶从"看我们的后厨"转义为"后厨就该是明的,这是我们画的图纸"。
**感受目标**:意外感 + 认同感——"这个东西我在别家没见过"+"对,交付就该长这样"。
**出场问题**:怎么开始?(v1.6 复位,由板块六"下载即用"接住)
**失败迹象**:设计图内容像美化过的愿景图(必须保留打回、返工等不体面记录——连设想图里都画着"打回 3 次"的团队,比任何绿色对勾都可信);把设计图包装成产品实拍(违反第〇部分语域纪律);或此处出现总体性数字承诺(违反写虚禁令)。

## 板块六 · 行动【定稿 v2】

**(v1.6 重写:原 A/B/C 三分法随其隐含假设"产品是托管服务"一并作废——产品为开源 macOS 桌面应用,无注册、不收 key,驱动用户本机已登录的 codex/claude cli。访问权不在我们手里,门常开:本板块的问题从"设计一道门"变为"画一条最短路径"。原方向 B 的全部残值由次按钮收纳——开源仓库的 commits 与 issues 天然是建造直播,零内容供给成本。)**

**进场状态**:五屏读完,标准刚立完,末端问题"怎么开始?"带着具体期待而来——访客想知道实物离自己几步。
**唯一工作**:画一条最短路径,从"读完的人"到"本机跑起第一个任务的人";同时完成图解→实物的交接:页面上画的对齐、打回、介入,下载后会以群聊消息的形态逐条出现在访客眼前。前提如实陈列,一步不藏。
**感受目标**:门是开的,且自己已半只脚在门里——"我机器里已经登录的那些 agent,原来就差这一层章法"。无注册、无邮箱墙、不收 key:在遍地候补名单的行业里,这与首屏"敢展示慢下来检查"是同一血统的昂贵信号。
**出场问题**:无(全页终点)。若访客带走一个问题,应当是对着自己机器的"第一个任务交什么?"——首屏输入框占位文案已给出示范。
**失败迹象**:人为造门(产品无注册,页面不得竖邮箱收集墙);隐瞒前提(cli 要求与"仅 macOS"必须出现在下载按钮可视范围,藏进 FAQ 即不诚实——何况前提即卖点:不另收费,用你已有的订阅,藏了反而亏);开源奖状墙(star 数、badge 墙属取证语域数字近亲,违反写虚禁令);配置能力泄入主叙事(见第〇部分词汇禁令 v1.6 增补);按钮文案退回系统语言("免费注册""立即开始",此条自 v1 保留)。

## 附录 A · 传播纪律
上游共识(第〇部分)的任何修改,必须逐板块检查冲刷:骨架六句 → 各板块规格 → 文案要素 → 视觉规格。历史教训:敌人从"手动方式"换为"抽卡式交付"时,首屏 H1、演示高潮帧、板块二画面、骨架标题全部连锁改写;受众修正(路由器→回合制)与证据修正(写实→写虚)同理。改动只落在最后碰过的一屏 = 失配悄悄累积 = 以"说不清哪里难看"还债。
补充:废稿引力。被替换的旧产物不会自动消失,会以缓存形式回流——尤其在视觉词汇层(文字层被冲刷了,画面层没有)。实例:板块三 v1 定稿的帧 1 曾复活已废弃的"星型监工视线"画面,并把行业标配的"交接连线"误当差异来揭示(v1.2 修正)。对策:上游假设被替换时,旧产物显式标注"已废弃";每次产出视觉规格,专项核对画面中是否混入已废弃叙事的元素。
补充:旧世界的仲裁标准。判断"旧世界长什么样"以**受众的亲历**为准,不以行业事实为准——产品或行业内部为真的东西(如多 agent 协作早已存在于各产品内部),若不在受众可见的经验里,就不属于页面上的旧世界。实例:板块三 v1.2 曾以"交接是行业标配"为由把旧世界画成互连流水线,v1.3 修正(受众亲历:agent 是孤岛,人是唯一胶水,且身兼传话、把关、对齐诸角色)。
补充:推理图不等于观众图。为厘清概念而画的抽象图(拓扑、节点云)是设计工具,不得直接充当页面视觉;页面视觉一律用受众的业务语言与业务形态呈现(本页即横向业务流水线)。配套检验——承重隐喻优先:扛过多轮上游变更仍然存活的隐喻(如流水线,自首轮讨论沿用至今)是承重墙,视觉层不得擅自更换。实例:板块三 v3 曾以节点云呈现角色移交,v1.4 修正为业务流水线。
补充(v1.5):语域纪律。页面首要目标是阐明思想,论述语域与取证语域不得混用——"担保、证据、实拍、眼见为实"等取证措辞在产品无兑现能力期间即空头支票,须写成"图解、标准、图纸、设计"等论述措辞。实例:v1 曾要求"所有承诺由演示担保""数字须由演示实时产生",v1.5 修正为"由演示图解""数字与画面一致即可,不作实时表演";板块五由"出示逐单证据"修正为"提出标准并画出图纸"。配套检验:每次产出文案,专项核对是否有取证语域词汇混入;唯一保留的硬约束是诚实底线——不把概念演示包装成产品实拍。
