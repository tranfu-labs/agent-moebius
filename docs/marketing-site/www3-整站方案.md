# Moebius 官网 www3 整站设计方案

版本：v1.0
依据文档：
- 《官网文案》v10.3（唯一文案真源，本方案不改一字，只做落位与分镜设计）
- 《官网叙事规格书》v1.6.5（叙事宪法，本方案的一切设计选择均服从其纪律）
- 《用户画像》v1.3（老周／阿蓝，审稿透镜，不作为页面组织结构）
- `style-demo.html`（已定稿视觉基底，本方案沿用其全部设计令牌）

阅读方式：本文档写给要直接动手实现的高级程序员看，只交代"做什么、为什么这么做、分几块、怎么验收"，不规定变量命名和具体函数实现。

## 一、总览

### 1.1 页面目标

首要目标是**把道理讲清楚**，不是促成转化。访客读完应当明白"可信的自动才稀缺"这件事，而不是被推着点下载。产品尚未完成质量证据，页面只描述产品行为，**不承诺使用结果**（不写"省时""放心""你可以去忙别的"这类话）。转化、下载、说服都是"理解"之后自然发生的副产品，不是页面要主动争取的东西。

### 1.2 叙事问题链（沿用 www2 六板块链条，一字不改）

页面是一条首尾咬合的问题链，每板块的出场问题就是上一板块的收口，每板块的收口问题引出下一板块：

```
真的能走开？我被烧过
    ↓（板块二：收编怀疑，把"走不开"的原因说透）
一群自动化参与者，离一个团队差什么？
    ↓（板块三：揭示——那些角色一直是你在演，现在被逐个接走）
接走的角色，靠谱吗？
    ↓（板块四：说明——它们接手的正是你原来亲自干的活，三个时刻可核对）
说得好听，痕迹呢？
    ↓（板块五：提出标准——痕迹本该是一张过程底单）
怎么开始？
    ↓（板块六：给出从理解到动手的最短路径）
```

六个板块对应文案六个小标题，全页只有这一条链，不做分岔、不做"两类读者各看各的"式并行结构——两个画像的经验要能同时挂在同一个"你"身上，不能出现队列标签。

### 1.3 技术形态

- 单文件静态 `index.html`，不引入任何构建工具、不依赖前端框架，可以直接双击打开或扔进任意静态托管服务。
- 全站只做中文，不做双语切换开关。
- 配图默认全部用纯 SVG 几何图形（`rect`/`circle`/`path`/纯色块 + 黑描边 + 硬偏移阴影）绘制并做动画，不依赖任何写实生图。生图只作为文末"可选增强项"单独列出，不进首版主线。
- 滚动触发效果优先使用原生 `IntersectionObserver`；循环动画优先使用 CSS `@keyframes`；只有个别简单重复动作（如某个图形的自转、单个元素的位移循环）可以用 SVG 原生 `<animateTransform>`/`<animate>`（即 SMIL）实现，减少 JS 代码量。全程不引入 GSAP 或其他动画库，也不做超出"滚动进度触发一段状态切换"范畴的复杂时间线调度。
- 字体延用 `style-demo.html` 的 Google Fonts 引入方式（Noto Sans SC + JetBrains Mono）。

### 1.4 文件清单

产物在技术上只有一个必需文件，另加少量静态资源：

| 文件 | 职责 |
|---|---|
| `index.html` | 唯一页面文件。内部按区块组织：`<head>` 中放设计令牌（CSS 变量）、全局样式、各板块专属样式、`@keyframes` 定义；`<body>` 中依次放页头、六个 `<section>`（板块一至六）、页脚；页面底部一段 `<script>` 负责初始化各板块的 `IntersectionObserver`、板块二的粘性分拍进度判定、`prefers-reduced-motion` 检测与降级开关。 |
| `favicon.svg`（或内嵌为 data URI） | 浏览器标签图标，用设计令牌里的墨黑/纯蓝画一个极简几何图形，不需要额外的图片编辑工具。 |
| `docs/www3-整站方案.md` | 本文档，不参与线上发布。 |

不拆分独立的 `style.css` / `app.js`：单文件形态便于直接分享和部署，也避免多文件间的令牌同步问题。若后续页面显著变大再考虑拆分，本方案不预先做这个动作。

## 二、设计令牌引用表

以下令牌全部照抄自 `style-demo.html` 的 `:root`，本方案不新增、不改值，只补充语义色的业务约定。

### 2.1 颜色

| 变量 | 值 | 定位 | 语义约定 |
|---|---|---|---|
| `--cream` | `#F2EDE4` | 全站背景 · 奶油纸 | 唯一背景色，禁止渐变 |
| `--ink` | `#1A1A1A` | 墨黑 | 描边、正文文字、图形填充的默认色 |
| `--white` | `#FFFFFF` | 卡片白 | 卡片、气泡、单据等前景容器底色 |
| `--blue` | `#00D9FF` | 纯蓝 | **对齐 / 推进中**——凡表示"正在往前走""正在对齐方向"的状态用蓝 |
| `--lime` | `#CCFF00` | 荧光黄绿 | **通过**——凡表示"验收过了""可以往下走"的状态用荧光黄绿 |
| `--orange` | `#FF9500` | 亮橙 | **打回 / 返工**——凡表示"没过、退回重来"的状态用橙 |
| `--pink` | `#FF006E` | 亮粉 | **强调 / 警示 / "你"**——用于标出"你"这个角色本身，以及需要读者格外注意的标记（如"需要你拍板"） |

四色只用于状态标记与"你"这个角色的识别色，不用于大面积背景，也不互相混用语义（不能用蓝色表示"打回"）。

### 2.2 描边与圆角

- `--border-thin`：`3px solid var(--ink)`，用于气泡、小型徽章等次要容器。
- `--border-thick`：`4px solid var(--ink)`，用于卡片、按钮、主要图形节点等一级容器。
- 圆角三档：`--radius-sm 6px` / `--radius 10px` / `--radius-lg 12px`，全站圆角只在这三个值里选，不出现更大的圆角（不做"胶囊形"按钮）。

### 2.3 阴影

三档硬偏移阴影，均无模糊：

- `--shadow-sm`：`4px 4px 0 var(--ink)`
- `--shadow-md`：`6px 6px 0 var(--ink)`
- `--shadow-lg`：`8px 8px 0 var(--ink)`

任何阴影声明的第三个数值（模糊半径）必须为 `0`，这是新野兽派风格的硬约束，不允许出现柔和投影。

### 2.4 字体

- 中文与拉丁混排主字体：`Noto Sans SC`（系统回退 `PingFang SC` / `Microsoft YaHei`），标题用 900 字重，小标题用 900 或 700，正文用 400。
- 等宽字体：`JetBrains Mono`，只用于标签芯片、状态徽章、编号这类"系统气质"的短文本，不用于正文段落。

### 2.5 标签芯片（chip）

黑底白字为默认样式，四个变体对应四个语义色（`chip--blue` / `chip--lime` / `chip--orange` / `chip--pink`），文字全大写、字母间距拉开，字体用等宽体。芯片在本方案中承担"状态徽章"和"轻量图形语言强化"两个作用：既可以贴在动画节点旁边标状态（如 `ALIGN`/`VERIFY`/`REDO`），也可以在板块三"翻转"处标注新就位者的角色职能——但要克制使用，避免把角色钉成"系统零件"（详见板块三设计护栏）。

每板块统一按四项展开：沿用的文案、版式结构、SVG 动画分镜、移动端降级策略。

### 板块一 · 首屏

**沿用文案**

- 眉题（小字）：Moebius —— 免费开源的 macOS 应用
- 主标题：把你在用的 AI，变成一个会自己推进的团队。
- 副标题（三行）：你交出一件事，它一个环节接一个环节往前推。／团队里有盯进度的，有验质量的——不合格，当场打回重做。／只有该你拍板的事，才会到你面前。
- 下滑引导（小字）：往下看：你现在是怎么过的，交出去之后会变成什么样 ↓

首屏不放任何对话气泡（对话气泡母题留给板块四"对齐时刻"专用，首屏不能提前用掉），也不放下载按钮（下载按钮只在板块六出现，首屏不做转化动作）。

**版式结构**

沿用 `style-demo.html` 的 `.hero-demo` 两栏骨架：左栏文字块（眉题 + 主标题 + 副标题 + 下滑引导），右栏 / 下方是主视觉动画区。

```
┌─────────────────────────────┬───────────────────────┐
│ Moebius —— 免费开源的 macOS 应用     │                       │
│                               │                       │
│ 把你在用的 AI，                │      【主视觉】          │
│ 变成一个会自己推进的团队。        │   环节推进 + 中途打回      │
│                               │      的循环示意          │
│ 你交出一件事，它一个环节接一个环节  │                       │
│ 往前推。团队里有盯进度的，有验质    │                       │
│ 量的——不合格，当场打回重做。      │                       │
│ 只有该你拍板的事，才会到你面前。    │                       │
│                               │                       │
│ 往下看：你现在是怎么过的… ↓        │                       │
└─────────────────────────────┴───────────────────────┘
```

**SVG 动画分镜——主视觉**

这是全站第一眼动画，唯一任务是"目击感"：让访客先看懂一件事正在自己往前走，中途被认真检查了一次。不做仪表盘、不做多节点全景架构图。

设计上以 `style-demo.html` 的"机制动画 B·接力推进"为主干骨架（接手 → 处理 → 复核 → 交付四节点横向排布，一个任务色块依次经过并点亮），在此基础上叠加一次"检查未通过"事件：

- 一个任务卡片从"接手"节点出发，依次经过"处理""复核"，抵达"复核"节点时第一次不是直接通过——节点短暂变红、卡片轻微震动后被推回上一节点，稍作停留（这一拍要明显放慢，是全动画里唯一的慢镜头，因为"敢慢"正是差异化所在），再重新经过一次"处理"，第二次抵达"复核"变绿通过，最后抵达"交付"。
- 任务卡片准备两种视觉皮肤交替出现：一种卡片内部用等宽字体的短横线模拟代码块观感，另一种用不等长的横线模拟文档段落观感，对应"任务样例一码一文"。同一条轨道上先跑完一个代码任务的完整循环，再跑一个文档任务的完整循012环，交替往复。
- 全程不出现产品界面元素（不画窗口/按钮/参数面板），只有几何色块、节点框和一条虚线时间轴。

循环时长：约 16～18 秒一个完整循环（含两种任务皮肤各跑一遍、含一次放慢的打回事件），持续循环播放，滚动到首屏视口内即播放，离开视口暂停（用 `IntersectionObserver` 控制 `animation-play-state`，省电）。

**移动端降级**

- 主视觉动画区从右栏移到文字块下方，纵向排布。
- 简化为单一任务皮肤跑一次完整循环（去掉代码/文档交替，只保留一条推进线 + 一次打回），降低同屏元素数量。
- 屏宽小于约 480px 时，动画进一步降级为三帧静态切换（起点 → 打回中 → 终点，用 CSS `transition` 做简单交叉淡化替代关键帧动画），减少移动端重绘负担。

### 板块二 · 旧世界

本板块是全站唯一采用"粘性分拍"结构的板块，文案已经给出五拍骨架（拍 0 进场／拍 1 管／拍 2 放／拍 3 收拢／拍 4 归因），版式和动画紧密耦合，一并说明。

**沿用文案（按拍落位）**

| 拍 | 眉题 | 主句 | 辅句 / 尾句 | 画面状态 |
|---|---|---|---|---|
| 拍 0 · 进场 | — | 交付像抽卡，所以你走不开。 | 一管就死、一放就乱。 | `base`：空场景 |
| 拍 1 · 管 | 管 | 一直盯着 | 担心它理解错了意思，又怕它太自觉、一下子做得太多——一下午过去，比自己干活还累。 | `hold`：多个小剪影盯着文件堆 |
| 拍 2 · 放 | 放 | 信心满满 | 开了好几个任务，以为今天效率倍增——没想到是检查项倍增、返工倍增。 | `letgo`：桌面纸团狼藉 |
| 拍 3 · 收拢（本板块最大字号） | — | 管，是你在替它干；放，是你在替它收拾。 | — | `base`：回到空场景，逐字落墨 |
| 拍 4 · 归因 | — | 是现在模型不够聪明吗？ | 不是——你盯着的时候，它明明做得出来。缺的不是聪明，是管事的：对齐、交接、检查、打回，一个都没有。（尾句，弱化）干活的都在，管事的呢？ | `attrib`：中央工位灯亮，两侧格间暗空 |

**版式结构**

一个粘性容器占满一屏高度，内部叠放四张状态图层（`base`/`hold`/`letgo`/`attrib`），用透明度交叉淡化在拍与拍之间切换；容器外部包一段足够长的滚动轨道（建议整体轨道高度约为视口高度的 4～5 倍，五拍不必等分，拍 2 与拍 4 文字量更大，分配更长的滚动区间）。文字必须落在画面上方"米白干净天空区"内，不与画面主体元素重叠。

拍内三层版式（眉题 / 主句 / 辅句）错峰出场：

```
┌───────────────────────────────────────────┐
│  [眉题·点缀小字]                              │  ← 先出现
│                                             │
│  [主句·名牌大字]                              │  ← 随后出现
│                                             │
│  [辅句·正文级，弱化]                           │  ← 最后出现（拍 2 要隔更明显的一拍再出）
│                                             │
│           【共用场景：格间长桌 SVG】             │
└───────────────────────────────────────────┘
```

拍 2 的错峰是本板块唯一的设计强调点：主句"信心满满"先压在纸团狼藉的画面上停留一瞬（反讽间隙），辅句延后一拍才浮现，让"打脸前的心声"和"打脸后的现场"之间产生错位感。

**SVG 动画分镜**

四张状态图全部是同一场景（连续玻璃格间长桌）的不同状态，几何画法：

- `base`：3～4 个矩形"格间"并排，每格顶部一个小矩形/圆形"桌灯"，灯光用淡黄描边表示，格间内空无一物。
- `hold`：格间内各出现一个简笔人形剪影（圆头 + 梯形躯干），俯身面向桌面的若干小矩形"文件堆"。
- `letgo`：桌面矩形替换为多个不规则揉皱形状（用不规则闭合 `path` 表示纸团），散落多个格间，整体密度明显高于 `hold`，营造狼藉感。
- `attrib`：只有中央一个格间的桌灯保持点亮（用一个缓慢的透明度脉动表示"灯还亮着"），两侧格间整体降低到低透明度 / 灰阶。

四态之间用透明度交叉淡化过渡（每次切换约 0.6～0.8 秒），不做位移动画，保证粘性区域内的性能开销可控。

触发方式：在滚动轨道内部预先放置 5 个不可见的锚点区块（对应五拍），用 `IntersectionObserver` 判断当前进入视口的锚点，据此给粘性容器切换"当前拍"的 CSS 类，驱动画面淡化与文字错峰。选择这种"离散锚点判定"而非连续滚动进度绑定，是因为不需要逐像素跟手的丝滑联动，只需要在五个清晰的拍点之间做状态切换，用离散触发即可满足叙事节奏，同时避免引入连续滚动量计算的复杂逻辑，保持零依赖、逻辑简单。

**移动端降级**

粘性叠层配合触屏滚动容易出现体验问题（粘性区域高度控制难、惯性滚动跳拍）。小屏（约 768px 以下）改为放弃粘性结构，五拍还原为五个独立的纵向卡片区块，每块进入视口时（`IntersectionObserver`）触发一次淡入 + 轻微上移的入场动画（播放一次即停，不循环），每块各配一张对应状态的静态简化 SVG 插图，不做同场景状态复用。牺牲"同一场景渐变"的精细度，换取小屏下的滚动顺畅与实现简单度。

### 板块三 · 转折

**沿用文案**

- 小标题：缺的角色，一直是你在演。
- 开场句：回想你"亲自上"的那些时候。
- 图文单元一：把任务讲清楚的，是你。上一步弄完，把结果递给下一步的，是你。发现跑偏拽回来、一遍遍检查能不能用的，是你。改到能用为止，还是你。对齐、交接、把关、返工——这些角色从来都有人演，那个人，就是你。
- 收拢（大字）：你以为你在用工具。其实是你一个人，在撑起一整个团队。
- 承接：所以你走不开：你一走，角色就集体缺席，抽卡和传送带就都回来了。真正缺的，是章法：一套规矩，让整件事**不靠你推也往前走**（红线④，字面不动），每一步都有人把关。
- 图文单元二 · 翻转：Moebius 只做一件事：把你演的角色，一个一个接走。它只在两种情况下需要你——交出任务的时候；必须由你拍板的时候。
- 过渡：接走的角色，靠谱吗？

**版式结构**

延续板块二的"格间 + 椅子"母题（跨板块视觉线索），但这里不再是粘性叠层，是两组各自独立、滚动触发一次的场景动画，各自配一段文字：

```
【图文单元一】
  文字（左或上）           配图（右或下）
  把任务讲清楚的，是你。      一排椅子，同一个"你"剪影
  ……                     快速在椅子间切换、留下残影
  你以为你在用工具，
  其实是你一个人，
  在撑起一整个团队。

【图文单元二 · 翻转】
  文字                     配图
  Moebius 只做一件事：       同一排椅子，"你"逐把起身，
  把你演的角色，             同构的新剪影逐把坐下，
  一个一个接走。             只留拍板那把椅子给"你"
```

**SVG 动画分镜**

配图一（分身乏版）：一排 4～5 把简笔椅子（矩形座 + 竖线靠背），一个"你"剪影（与板块二格间里的人形剪影同一造型，色标为亮粉）在椅子间做快速位移，位移过程中保留 2～3 个渐隐的残影副本，表示"屁股没坐热就得挪"的分身感。滚动进入视口时播放一次，时长约 3～4 秒，播完停在"你"回到中间椅子的静止姿势，不循环。

配图二（翻转）：同一排椅子。动画分两个阶段：①"你"的剪影从除拍板席之外的每把椅子上依次起身、淡出（逐把间隔约 0.4 秒）；②每把空出的椅子上，一个与"你"**同构**的新剪影依次坐下（同一种简笔造型、同一个格间位置，只是坐下动作和极轻微的色相差异，不做功能标签化的新造型）。最终画面：除拍板席外全部坐上新的就位者，"你"只留在拍板那一把椅子上。滚动进入视口触发一次，时长约 4～5 秒，播完停在终态。

设计护栏（对齐规格书要求）：新就位者绝不允许换成不同的图形语言（比如画成方形机器人图标），必须是同一类剪影、同一个格间位置，只换"坐进去的人"，避免被读成"预置的系统零件"。如需要标注角色职能（对齐 / 交接 / 验收），只允许在剪影旁附一个很小的芯片文字标签，不做视觉主体，且这一处理是可选项，若设计判断标签会强化"系统零件"的误读，允许完全不加标签。

**移动端降级**

椅子数量从 4～5 把减到 3 把；残影数量从 2～3 个减到 1 个；两组动画的触发逻辑不变（滚动进入视口播放一次），仅降低几何复杂度以控制小屏下的绘制开销。

### 板块四 · 机制

**沿用文案**

- 小标题：该问你时问你，该返工时自己返。
- 单元〇 · 推进的底色：交出去之后，它就一直往前推：一个环节干完，下一个环节自己接上——那根接力棒，不在你手里了（红线②）。盯进度这活，归它了（红线③）；过程一直亮在 Moebius 的窗口里，你想看，随时能看。但推进不是闷头跑。路上有三个时刻，桩桩都是你以前亲自干的活——靠不靠谱，看完你自己判断。
- 单元一 · 时刻一（对齐）：你以前接到任务也是这么干的：含糊的地方，宁可多问一句，也不闷头做错。现在换它来问你——问的是"你要什么"，不是"该怎么做"；怎么做，是它自己的事。哪些事必须你拍板，也是这时候说好的。问清楚了，它就去干活，不再出声。画中字：这里有两种理解，你要哪一种？／大纲先按哪个方向走？（代码 / 文档各一）
- 单元二 · 时刻二（验收）：每份产出送到你手上之前，至少被验收一次。验收的标准，就是开工时和你对齐的那份要求——一条一条对着查。以前逐字逐行检查的是你；现在这个位置，由它顶上。
- 单元三 · 时刻三（打回）：验收没过的东西，不会装作没事送到你面前。打回，重做，再验——这一整圈它自己跑完，跑完，才把结果送到你面前。要是打回几圈还过不了，它不硬跑——带着卡住的地方，来找你拍板。画中字：验收未通过，已打回。
- 收拢：能自己定的，它不来烦你；只有你有权定的，才来找你。其余的时间，它不出声，只干活。
- 过渡：说得好听。痕迹呢？

**版式结构**

单元〇作为本板块的背景语境，横贯板块顶部（一条持续循环的接力推进条），单元一、二、三各是独立的图文小节，纵向依次排列，滚动到哪个小节，就让对应的小动画播放、其余弱化：

```
┌──────────────────────────────────────────┐
│         【单元〇：接力推进条，持续循环】            │
│    接手 → 处理 → 复核 → 交付（一直在跑）           │
├──────────────────────────────────────────┤
│ 【单元一：对齐】                                │
│  文字：你以前接到任务也是这么干的……                 │
│  配图：克制提问对话框（两个待选方向）                 │
├──────────────────────────────────────────┤
│ 【单元二：验收】                                │
│  文字：每份产出送到你手上之前，至少被验收一次……          │
│  配图：产出与要求清单并排核对                       │
├──────────────────────────────────────────┤
│ 【单元三：打回】                                │
│  文字：验收没过的东西，不会装作没事送到你面前……          │
│  配图：产出退回、循环打回、亮起"需要你拍板"标记          │
├──────────────────────────────────────────┤
│  收拢：能自己定的，它不来烦你……                     │
└──────────────────────────────────────────┘
```

**SVG 动画分镜**

单元〇：直接复用 `style-demo.html` "机制动画 B·接力推进"母题——四个节点（接手/处理/复核/交付）横排，一个色块依次经过并点亮各节点。持续循环，时长沿用约 12 秒一轮，全程可见（不需要滚动触发暂停，因为它是"背景语境"，暗示推进始终在发生）。

单元一（对齐）：截取并单独放大 `style-demo.html` "机制动画 A"里的对齐节点部分——一个带虚线外环的圆形节点（表示"正在对齐"的持续状态）+ 一个从旁升起的对话气泡。气泡内文字在"这里有两种理解，你要哪一种？"和"大纲先按哪个方向走？"之间交替展示（对应代码/文档场景各一），每条气泡展示约 3 秒后切换。滚动进入视口时开始播放，离开视口暂停。

单元二（验收）：两个卡片并排——左卡"要求清单"（三条横线代表清单条目），右卡"本次产出"，中间有一个打勾标记逐条从左卡移动核对到右卡对应条目，核对到的条目变绿打勾。循环一轮约 6～8 秒，进入视口播放。

单元三（打回）：一份产出（卡片图形）被推回到"复核"节点，一个循环箭头转动 2～3 圈后停下，随后在节点旁亮起一个黑底白字的"需要你拍板"徽章（用亮粉描边强调）。若打回后重新通过，则徽章替换为荧光黄绿的"通过"图章；两种结局交替展示，对应文案"打回几圈还过不了，带着卡住的地方来找你拍板"与"打回、重做、再验，自己跑完"两种情况。循环一轮约 8 秒，进入视口播放。

三个小节的动画建议各自实现为独立的小型 SVG 组件（而非一整条贯穿全板块的大 SVG），便于用 `IntersectionObserver` 独立控制每个组件的播放/暂停，也让移动端裁剪更容易。

**移动端降级**

单元〇的横向接力条改为纵向四节点堆叠（接手→处理→复核→交付从上到下），色块从上往下移动；单元一、二、三的小动画整体缩小尺寸，对话气泡与清单文字做换行处理，保留核心识别度；三个小节的播放时机不变（进入视口播放）。

**沿用文案**

1. 单据抬头先出现（标题"过程底单" + 一条横分隔线）。

**移动端降级**

单据卡片宽度适配满屏；逐行生长动画可直接省略，改为整张单据以一次性淡入的方式出现（不做逐行时序），降低小屏动画复杂度，同时保留内容的完整可读性。

## 四、动画技术清单

| 编号 | 名称 | 所在板块 | 实现方式 | 触发方式 | 循环/时长 | 性能注意 |
|---|---|---|---|---|---|---|
| A1 | 首屏推进主视觉 | 板块一 | CSS `@keyframes`（位移/颜色/透明度） | 进入视口播放，离开暂停 | 循环，约 16～18 秒/轮 | 仅用 `transform`/`opacity`/`color` 驱动，不触发布局重排 |
| A2 | 移动端简化主视觉 | 板块一（小屏） | CSS `transition` 三帧切换 | 进入视口播放 | 循环，约 6 秒/轮 | 元素数量降至最低 |
| B1 | 粘性四态交叉淡化 | 板块二 | CSS `opacity` 过渡 + JS 切换状态类 | `IntersectionObserver` 判定五个锚点区块 | 非循环，随滚动位置切换 | 四态图层预先常驻 DOM，只切透明度，不做增删节点 |

| B3 | 移动端卡片入场 | 板块二（小屏） | CSS `@keyframes` 淡入 + 上移 | `IntersectionObserver`，每卡进入播放一次 | 非循环 | 逐卡独立触发，避免整屏联动 |

| D2 | 验收核对动画 | 板块四 · 单元二 | CSS `@keyframes` | 进入视口播放，离开暂停 | 循环，约 6～8 秒/轮 | 打勾标记用 `opacity`/`transform: scale` |
| D3 | 打回循环 + 拍板徽章 | 板块四 · 单元三 | CSS `@keyframes`（复用机制动画 A 的打回分支） | 进入视口播放，离开暂停 | 循环，约 8 秒/轮 | 循环箭头用 SVG `<animateTransform>` 自转 |
| E1 | 单据逐行生长 | 板块五 | JS 逐行切换 `opacity`/`translateY`（配合 `IntersectionObserver` 判定滚动位置分段触发，或简单 `setTimeout` 序列） | 进入视口播放一次 | 非循环，约 5～6 秒 | 条目预先常驻 DOM，只做逐个淡入，不做增删 |
| E2 | 移动端整单淡入 | 板块五（小屏） | CSS `transition` | 进入视口播放一次 | 非循环 | 单次 `opacity` 过渡 |
| F1 | 印章弹入（✓/✗） | 板块六 | CSS `@keyframes`（scale + rotate） | 进入视口播放一次 | 非循环，约 0.4 秒 | 仅 `transform`/`opacity` |
| F2 | 终端光标闪烁 | 板块六 · 左卡 | CSS `@keyframes` | 进入视口播放，离开暂停 | 循环，约 1 秒/轮 | 单元素 `opacity` |
| F3 | 气泡微动 | 板块六 · 右卡 | CSS `@keyframes` | 进入视口播放，离开暂停 | 循环，约 0.8 秒/轮 | 单元素 `opacity`/`translateY` |

统一原则：所有动画只用 `transform` 与 `opacity`（含颜色/`fill` 这类不触发布局的属性）驱动，不对 `width`/`height`/`top`/`left`/`box-shadow`/`filter` 做动画，避免触发重排或高成本重绘。所有循环动画在 `prefers-reduced-motion: reduce` 下统一降级（见下节验收清单）。

## 五、验收清单

### 5.1 文案红线自查（可用 `grep` 得到 0 命中的项）

禁用词（页面正文，不含引用规范文件名的场景）：

```
grep -n -E "Agent|agent|subagent|编排|orchestration|harness|多智能体" index.html
grep -n -E "更可靠|企业级|高质量|去忙别的|更快(?!捷键)" index.html
```

均应为 0 命中。

承诺红线：全文不得出现承诺使用结果的句子（省时、放心、"你可以去干别的"等），逐句人工比对《官网文案》v10.3 原文，任何超出原文的润色式改写一律打回。

红线哨位四句必须逐字出现在页面文字中（用 `grep -F` 精确匹配全文，应各命中 1 次）：

```
grep -n -F "想想你今天本来打算盯着干完的那件事" index.html
grep -n -F "那根接力棒，不在你手里了" index.html
grep -n -F "盯进度这活，归它了" index.html
grep -n -F "不靠你推也往前走" index.html
```

无出处数字自查：全文搜索所有阿拉伯数字，逐个核对是否有产品事实支撑（如"两台 CLI""macOS 版"这类不是统计数字的表述除外），不得出现"提速 30%"一类无出处的量化承诺。

### 5.2 视觉自查

- 无渐变背景：`grep -n -E "linear-gradient|radial-gradient" index.html` 应为 0 命中。
- 无细灰边：全文搜索边框颜色声明，不应出现 `#ccc`/`#ddd`/`rgba(0,0,0,0.1)` 一类的浅灰细边，所有描边只能是 `--border-thin`/`--border-thick` 两档纯黑描边。
- 无模糊阴影：所有 `box-shadow` 声明的模糊半径（第三个数值）必须为 `0`，逐条核对。
- 圆角只在 6/10/12px 三档取值，人工检查是否有游离圆角值。
- 色板检查：页面中出现的所有颜色值应能对应到设计令牌七色之一，不出现游离色值（品牌色之外的临时调色）。

### 5.3 动画自查

- 循环首尾闭合：逐个检查 `@keyframes` 规则，`0%` 与 `100%` 的视觉状态应一致（或刻意做成无缝循环），不出现循环边界处的画面跳变。
- `prefers-reduced-motion` 降级：确认存在如下媒体查询，全局大幅缩短或关闭位移/旋转类动画，只保留必要的状态切换（如透明度切换）：

```css
@media (prefers-reduced-motion: reduce) {
  /* 所有循环动画时长归零或改为一次性瞬时切换，禁用位移与旋转动画 */
}
```

- 性能自查：确认所有 `@keyframes` 规则只驱动 `transform`/`opacity`/`fill` 一类属性，人工审查有无遗漏。

### 5.4 浏览器实测步骤

1. 用 Safari 与 Chrome 各打开一次页面，检查中文字体（Noto Sans SC）与等宽字体（JetBrains Mono）是否正常加载，无明显字体闪烁跳动。
2. 从头到尾滚动一遍全页，肉眼确认六板块动画在预期时机触发，尤其核对板块二粘性五拍的切拍是否顺畅、板块三的两段动画是否只播放一次、板块五单据是否按预期生长一次后停留在终态。
3. 打开系统"减弱动态效果"开关（系统设置 → 辅助功能 → 显示器 → 减弱动态效果），刷新页面，确认所有动画按 5.3 节的降级规则退化为极简过渡。
4. 将浏览器窗口缩放到移动宽度（约 375px），检查各板块的移动端降级策略是否按方案生效（板块二退化为独立卡片、板块三动画元素精简、板块五改为整单淡入、板块六卡片改为上下堆叠）。
5. 打开浏览器开发者工具的性能面板，完整滚动一遍页面，检查有无明显掉帧，尤其关注板块二粘性区域与板块一的持续循环动画。
6. 检查所有 SVG 是否带有 `role="img"` 与 `aria-label`，确保屏幕阅读器能读出配图大意。

## 六、可选增强项（不进首版）

- **生图插画**：在首屏主视觉或板块二格间场景等位置，用少量写实或半写实 AI 生成插画替换或叠加当前的 SVG 几何示意，增强氛围质感；需谨慎处理与全站几何风格的调和，避免视觉割裂。
- **英文版**：为国际开发者受众提供双语切换，复用同一套设计令牌与页面结构，仅替换文案层（可参考 www2 已有的英文文案维护方式）。
- **深色模式**：跟随系统 `prefers-color-scheme` 提供一套深色变体令牌，需重新校验四个高饱和语义色在深色背景下的可读性与对比度。
- **动效进一步打磨**：引入极轻量的滚动视差效果、SVG 精灵化以减少重复图形定义、无障碍朗读体验的进一步优化。
