# 官网文案 v10.3

## 格式约定（先读，全文以此为准）

- **`文字：`**（或 `文字（角色）：`）后面的内容**原样出现在页面上**。括号里的角色注明版式身份（主标题、小标题、开场句、收拢·大字、过渡、清单、按钮等）。
- **`配图：`** 后面的内容是对网页上该处配图的大致描述（场景｜情绪｜画中字｜吸收细节），**不上页面**，是给设计的输入。
- **图文分工原则**：图片负责临场感与情绪，文字负责逻辑与清晰的语义表达。
- **基本颗粒是"图文单元"**：一段 `文字：` 配一条 `配图：`，成对呈现——文字贴着自己的配图（左右或上下由设计定）。图文单元之外只允许开场句、收拢句、过渡句这类独立文字。
- **除 `文字：` 之外的一切**（结构标题、本约定、上游裁定、上游标记、【版式说明】等）都不会出现在页面上。
- **验收双测试**：只读全页 `文字：`，逻辑与故事完整；只读全页 `配图：`，情绪线完整。

## 上游裁定（用户，优先级最高）

1. **产品主次轴**：主轴是"推进"（整件事一个环节接一个环节自动往前跑），次轴是围绕推进的质量把控（角色化验收、打回）。主标题用"推进"。画像A对"自动推进"话术的伤疤须在页内化解，但不构成主标题选词否决权。
2. **首屏不前置"中途来找你"**（读者会咯噔）：分工表述＋到达式拍板，打扰的正面辩护留在板块四。
3. **词汇纪律**：页面不出现 Agent、subagent、编排、orchestration、多智能体等词，用团队语言表达。
4. **页面首要目标**：让访客轻松、完整、清楚地了解内容；只描述产品行为，**不承诺使用结果**。
5. **反过拟合**：画像是审稿透镜，不是页面的组织结构。软经验不得分轨（"试过的人/没试过的人"式队列标签是缺陷）；硬条件可以分岔（有没有 Mac、装没装 CLI）。
6. **去设计者视角**：不祈使读者、不背诵内部工序链，从读者座位叙述。

## 红线哨位（四处，现状均合规；任何向"放心走开/你去忙别的"方向的加强即触线）

①收尾句"想想你今天本来打算盯着干完的那件事"；②"那根接力棒，不在你手里了"；③"盯进度这活，归它了"及板块四单元〇整段；④"不靠你推也往前走"（板块三章法定义句）。

## 板块二 · 旧世界

本板块采用 sticky 分镜五拍结构：同一场景四张状态全帧图随滚动交叉淡化，文字按"拍"组织。每拍最多三层——眉题（点缀小字）、主句（名牌大字）、辅句（正文级）；尾句在辅句基础上再弱化。画面状态即 base / hold / letgo / attrib 四图的语义：空场景 / 盯着的重影 / 纸团狼藉 / 两侧格间暗空。英文文案维护在页面内（hero-v2.html），不入本文档。

文字（辅句）：不是——你盯着的时候，它明明做得出来。
