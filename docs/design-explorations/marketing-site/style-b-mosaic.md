# style-b-mosaic.html 自然语言描述

> 本描述面向无法直接渲染 / 解析 HTML+CSS+Canvas 的读者（包括其他 LLM），
> 用途类似"网页无障碍语音朗读"：把视觉呈现、动效行为、页面结构逐一转译成文字，
> 使读者不看代码也能准确知道这个页面"看起来是什么样、说了什么、会动什么"。

## 一句话概括

这是产品 **Moebius（莫比乌斯）** 的单页营销落地页（landing page），全中文内容。
产品定位：多 agent 协作流水线——用户说一句指令，调度 agent 拆解任务、分派给专家
agent，agent 之间自动交接与质检，用户全程只需在最初发一句话、最后验收结果。
页面通过"六个对话框各自等你回复"（现状痛点）→"把人对 agent 的连接改成 agent
对 agent 的连接"（解决方案，滚动驱动动画）→ 数据佐证 → 机制解释 → CTA 五段式
叙事推进。

## 整体视觉风格："灰阶马赛克 / 方格世界"

- **配色**：米灰纸色背景（#F1F1EE）、浅灰面板（#E8E8E4）、近黑墨色文字
  （#1B1B19），中间夹杂三级中性灰（浅灰→中灰→深灰，#D3D3CF/#B3B3AD/#8A8A83）。
  唯一的高饱和强调色是一种赭红/陶土橙（#C15F3C），只用在"正在运行/强调/hover"
  等极少数位置，其余画面几乎是无彩色的灰阶网格。
- **核心视觉母题**：页面里所有的图形（Hero 里的网络关系图、滚动重组动画、
  小图标）都不是矢量插画，而是用 Canvas 逐格绘制的"小方块"拼成的——统一的
  12px 方块 + 3px 间隙的网格单位，坐标会被"吸附"到网格点上，所以线条、节点、
  移动的"信使方块"看起来都像马赛克贴图/低分辨率像素画，风格克制、偏工程图纸感，
  不是可爱风格，没有拟人吉祥物或插画角色。
- **字体**：中文/正文用系统无衬线字体；标签、编号、数据、系统状态类文字统一用
  等宽字体（monospace），字间距拉得很宽，营造"终端/代码/仪表盘"的技术感。
- **交互质感**：按钮描边为 1px 实线，hover 时反色（填充变深色或变强调橙色），
  没有阴影、没有圆角，整体是平面、硬边、极简的编辑器/图纸风格。

## 页面结构与内容（从上到下）

### 顶部导航栏（固定悬浮）
半透明毛玻璃背景条。左侧是 Logo：一个 3×2 的小方块阵列图标（其中一格是纯黑、
一格是深灰，其余浅灰）+ 大写字母间距拉宽的 "MOEBIUS" 文字标。右侧是一个描边
按钮"进入产品 →"，指向页面底部 CTA 区，鼠标悬停时反色为深底浅字。

### 00 · Hero（首屏，"目击"）
左右两栏布局。

**左栏（文案）**：
- 顶部一个带边框的小徽章"多 AGENT 协作流水线 · 公测中"，前面有一个橙色小方块
  在缓慢闪烁（呼吸/blink 效果，表示"实时在线"）。
- 大标题（两行）："你说一句，agent 们自己开工"。
- 副标题一句话说明价值：调度 agent 拆解任务、分派给专家 agent，交接与质检自动
  完成——你只验收结果。
- 两个按钮：主按钮"发出第一条指令"（深底实心，hover 变陶土橙）跳到底部 CTA；
  次按钮"先看它怎么协作 ↓"（描边）跳到下方滚动动画区。

**右栏（可交互 Demo 面板）**：一个带边框的"迷你软件窗口"，模拟真实产品界面：
- 顶部标题栏："MOEBIUS / RUN-0611"，右侧一个状态徽章，初始显示橙色的
  "LIVE"，动画播放结束后变为"DONE ✓"。
- 一行命令输入框，预填文字"调研三个竞品，输出一份对比报告"，右侧一个
  "RUN ▸" 发送按钮；回车或点击都会重新触发下面的动画。
- 一块 Canvas 画布，绘制一张"节点关系图"（自动播放的动画，也是本页最核心的
  可视化）：
  - 节点从左到右依次为：**你** → **调度**（枢纽）→ 三个并列的专家节点
    **调研α / 调研β / 调研γ**（纵向排开）与 **撰写**，再汇入 **质检**，
    质检再连回调度、调度再连回"你"，形成一个环。每个节点画成 2×2 的小方块簇。
  - 节点间的连线不是直线，而是"沿网格逐格铺出来的通道"，随时间从无到有慢慢
    "铺设"完成；连通后，通道上会有一两个小方块（"信使/包裹"）持续来回滑动，
    代表消息/工作成果在两端之间流动。
  - 动画同时驱动下方"日志区"逐行打字机式地输出模拟的 agent 间对话记录，
    时间线大致为：调度收到指令→拆解为 8 个子任务→依次派发给调研α/β/γ和撰写→
    调研α、β把资料交接给撰写→质检发现调研γ的定价数据有缺口，把任务**退回**
    调研γ补充→补齐后重新交接→撰写完成初稿交给质检→质检通过（结构完整、引用
    可溯）→调度把最终《竞品对比报告》交付给"你"。
  - 面板底部是一行统计条，实时刷新四个数字：并行任务数、消息数、人工介入次数
    （固定显示 1）、耗时（从 0 数到 9.2s 后动画标记为完成）。
  - 这个 Demo 会在页面加载约 0.9 秒后自动播放一次；用户也可以编辑输入框文字、
    按回车或点击 RUN 按钮重新播放（不过重新播放不会改变脚本化的日志内容，只是
    重跑同一段演示）。这是纯前端模拟，不连真实后端。

### 01 · 现状（"你的屏幕，此刻"）
- 小节编号"01"，标题"六个对话框，都在等你回复"。
- 说明文字：每个 AI 都很能干，但每走一步都要问你一句；你在窗口之间复制粘贴、
  来回传话——AI 越多，你越忙。
- 一个 3 列网格，铺 6 张卡片，每张代表一个正在等待用户回复的独立 AI 会话：
  竞品调研 A（等待 14 分钟）、竞品调研 B（32 分钟）、报告撰写（41 分钟）、
  数据整理（7 分钟）、翻译校对（26 分钟）、本周周报（58 分钟，带⚑旗标，
  样式特殊标红/高亮为"最紧急"）。每张卡片显示该 AI 卡住时问的那句话（例如
  "功能对比表整理好了，需要我继续拆定价吗？""那个网页我打不开，你能把内容
  贴给我吗？"），下方还有一排小方块指示灯，越是等待久的卡片，点亮的方块越多
  （并带闪烁动画），直观地把"等待时长"翻译成"亮灯个数"。
- 收尾一句点题："瓶颈不是模型不够快，是每一步都要路过你。灰块每亮一格，就是
  它们多等你十分钟。"

### 02 · 重组（滚动驱动的全屏动画，无独立小节标题，靠滚动"演出"）
这是一个高度约 330vh 的"滚动轨道"区块，配合一个 `position: sticky` 吸顶画布，
用户往下滚动页面时相当于用手拖动动画进度（而不是自动播放）：
- 滚动开始前，画面上先看到"回到 01 节六张卡片"的抽象版：6 组"你↔窗口"独立
  通道，按 3 行 × 2 列摆放，每条通道都是各自铺好的方块小路，上面有独立的小
  信使方块在爬——直观复刻"六个孤立的一对一连接"。
- 随着继续往下滚，除锚点（代表"你"的那一组）外，其余五条通道会依次从远端
  开始"拆解/消失"，拆下来的方块沿着一条折线路径飞向锚点，汇聚成一个点——
  视觉上是"五个独立会话被收纳进同一个起点"。
- 再往下滚，锚点伸出一条新通道连到一个新的"调度"枢纽节点；随后从枢纽向外
  以扇形辐射出约 10 条新通道，分别连到外围的多个 agent 节点，每条通道上都有
  多个信使方块并行穿梭（且数量随滚动进度增多），代表"从一对一，变成一对多、
  且都在自动流转"。
- 滚动接近底部时，标题层浮现出来："让 agent 对话 agent"，副标题
  "THE REWIRING · 由你的滚动完成"。
- 滚动开始前还有一行提示文案淡入淡出："那六个窗口，抽象成六条灰色通道 —
  滚动，亲手重组它们"，鼓励用户滚动体验。
- 尊重"减少动态效果"系统设置：如果用户开启了 reduce motion，这段动画会直接
  定格在终态，不做过程演出。

### 03 · 证据
- 小节编号"03"，标题"效率不是形容词，是这些数字"。
- 一行四列、外框加粗的数据栏，进入可视区域时数字会从 0 用缓动动画滚动计数到
  目标值：
  - **8** 子任务并行推进
  - **12** 次 agent 间自动交接
  - **1 次** 人工介入 = 最初那句话
  - **9.2 s** 从指令到交付
- 脚注："全部来自首屏那次真实演示的运行记录，非营销估算。"（呼应 Hero 区
  Demo 面板最终定格的耗时 9.2s，数字前后自洽。）

### 04 · 机制
- 小节编号"04"，标题"流水线为何不出错"。
- 引导句先抛出用户可能的质疑："agent 互相对话，不会越传越错吗？"，紧接着
  回答："会，如果靠 agent 小心。所以我们靠工位。"——把"可靠性"归因于流程
  设计而非依赖 agent 自觉。
- 三列卡片，每张卡片顶部有一个用小方块拼成的极简图标（Canvas 绘制），
  内容分别是：
  1. **分工**：每个 agent 只做一类事；复杂任务先被调度 agent 拆成一串简单
     任务再上线，简单任务不容易出错。
  2. **标准化交接**：agent 之间用结构化格式交接，不靠"感觉传话"；每次交接
     都有记录，可追溯、可回放。
  3. **质检**：审察 agent 把守出口，不合格的交付原路退回重做；用户验收的
     永远是已经过检的结果。

### 05 · CTA（结尾行动号召）
- 居中排版，背景切换为浅灰面板色以做视觉分隔。
- 小标签"05 · 轮到你"，大标题"发出你的第一颗指令"，副标题"一句话的成本，
  一条流水线的产出。"
- 一个和 Hero Demo 面板同款风格的输入框 + 按钮组合：占位文字"例：调研三个
  竞品，输出一份对比报告"，按钮文字"发出"。
- 点击按钮或在输入框按回车后，下方淡入一行确认文案："▪ 指令已进入流水线 ·
  公测名额将通过邮件发放"。这同样是纯前端模拟效果，没有真实提交/发信行为。

## 动效技术小结（供理解交互行为，非视觉必需）

- 各小节标题/正文首次进入视口时，会有一个轻微的"从下方 18px 处淡入到位"的
  揭示动画（`.reveal` → `.in`），由 IntersectionObserver 触发，一次性播放
  （滚回去不会重播）。
- 所有 Canvas 动画（Hero 关系图、滚动重组、机制区小图标）共用同一套
  "网格吸附"绘制逻辑：坐标先按 15px（12px 方块+3px 间隙）取整，再落笔，
  所以画面里一切"运动"本质上都是整格跳跃，而非平滑滑动，这是贯穿全站的
  视觉母题。
- 页面遵循系统级"减少动态效果"偏好：如果用户开启该系统设置，揭示动画直接
  显示终态、平滑滚动改为瞬时跳转、滚动驱动动画和 Demo 动画都直接跳到播放
  完成后的状态，不做渐进过程演出。
- 页面里没有任何图片资源，全部图形都是内联 SVG/Canvas 绘制或纯 CSS。
