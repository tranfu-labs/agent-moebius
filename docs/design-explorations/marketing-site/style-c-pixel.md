# style-c-pixel.html 自然语言描述

> 本描述面向无法直接渲染 / 解析 HTML+CSS+Canvas+图片的读者（包括其他 LLM），
> 用途类似"网页无障碍语音朗读"：把视觉呈现、动效行为、页面结构、嵌入图片逐一
> 转译成文字，使读者不看代码也能准确知道这个页面"看起来是什么样、说了什么、
> 会动什么"。

## 一句话概括

这是产品 **Moebius（莫比乌斯）** 的单页营销落地页，与 `style-b-mosaic.html`
是**同一份文案与同一套五段式叙事结构**（现状痛点 → 解决方案 → 数据佐证 →
机制解释 → CTA）的另一种视觉皮肤：把"灰阶马赛克/图纸感"换成了**复古像素游戏
+ 终端（terminal）** 的美术风格。产品定位不变：多 agent 协作流水线，用户说
一句指令，调度 agent 拆解任务分派给专家 agent，交接与质检自动完成，用户只需
发指令和验收结果。

## 整体视觉风格："复古像素 + 终端"

- **配色**：暖色羊皮纸/牛皮纸底色（#F2EDDF），深暖黑正文色（#2B2921）。
  强调色是一组暖色系：琥珀金（#F0A840）、陶土/锈红（#C4593B）、橄榄绿
  （#7A7A4D）。页面里多处出现深色"终端窗口"，背景近黑棕色（#26241E），
  终端内文字用终端绿（#B8C27A）和米白（#EFE6CC），整体观感像老式电脑终端 /
  8-bit 像素游戏的配色，比 style-b 的灰阶版本更暖、更有"复古游戏机"质感。
- **按钮/卡片质感**：普遍使用 2px 实线黑边 + 生硬的偏移投影（例如
  `4px 4px 0 #000` 这种没有虚化的"硬阴影"），悬停时整个元素往左上轻移、
  投影加长；点击时元素往右下压入、投影缩短——模拟"像素按钮被按下"的物理
  反馈，是明显的 8-bit/像素风 UI 惯用手法（而非 style-b 的纯反色 hover）。
- **分隔线**：小节之间用一条"像素分割条"（`.pxrule`）——用等宽黑白色块
  重复渐变画出的横条，看起来像一排小方块砖缝，而不是普通细线。
- **导航栏 Logo**：不是文字方块阵列，而是一个 13×13 像素的手绘位图小图标
  （用 Canvas 按硬编码的像素地图逐点填色画出，配色为墨/琥珀/锈红三色）。
  源码注释特别注明这是"自绘，非任何吉祥物"——即它是一个抽象像素符号，
  不是拟人角色或品牌吉祥物。
- **字体**：小节编号、终端内容、数据、标签统一用等宽字体；标题和正文用
  系统无衬线字体（含中文字体栈）。

## 页面结构与内容（从上到下）

### 顶部导航栏（固定悬浮）
半透明模糊背景条。左侧是上文所说的 13×13 像素小图标 + "MOEBIUS" 等宽字标；
右侧是一个像素按钮"$ 进入产品"（注意按钮文案带一个终端提示符 `$` 前缀，
呼应整站的终端主题），按钮有硬阴影，hover/active 有位移反馈。

### 00 · Hero（首屏）
纯居中排版（不是左右两栏，这点与 style-b 不同）：
- 顶部一个带黑色描边的徽章"多 AGENT 协作流水线 · BETA"，前面有一个琥珀色
  小方块闪烁指示"实时"。
- 大标题（两行）："你说一句，**agent 们自己开工**"，其中"agent 们自己开工"
  几个字是锈红色，且文字下方衬了一条半透明琥珀色底纹（像马克笔手写高亮）。
- 副标题：调度 agent 拆解任务、分派给专家 agent，交接与质检自动完成——你只
  验收结果。
- 两个居中并排的按钮："▸ 发出第一条指令"（琥珀色实心像素按钮，硬阴影）跳到
  底部 CTA；"先看它怎么协作 ↓"（描边像素按钮）跳到下方"换接法"小节。

**Hero 配图**：标题下方是一张跨栏大图（内联 base64 JPEG，约 190KB），
图片下方配一行说明文字"工位分工 → 逐站交接 → 出口质检 —— 下面这个终端里是
同一件事，真实发生"。

> **图片内容描述（相当于 alt text）**：一张宽幅像素艺术（pixel art）插画，
> 米色/奶油色背景。画面中有 **5 个圆头机器人**站在一条工业传送带上，从左到右
> 排成一列：机器人为白色/米色头盔、黑色护目镜配琥珀黄色"眼睛"、橄榄绿色的
> 耳部/侧面零件、橘红色躯干、橄榄绿手臂和腿。它们正在把一枚**发光的琥珀色/
> 金色六边形晶体（宝石/能量块）**沿队列依次传递：第 2 个机器人把晶体递给第 3
> 个，第 3 个再递给第 4 个……队列最右侧的机器人手持一个放大镜，正凑近晶体
> 检查它（象征"质检"环节）。传送带下方画了工业风的支撑结构：齿轮、活塞、
> X 形连杆、控制箱（带指示灯和格栅散热口）、立柱等，整体是一条像素风格的
> "机器人装配流水线"插画，与页面文案"分工 / 交接 / 质检"的比喻完全对应——
> 本质上是把"多 agent 协作流水线"画成了一条像素风格的机器人传送带。

### 终端演示区（无独立编号，紧接 Hero 之后，是全站视觉重心）
一个独立的深色终端窗口卡片，居中，比 Hero 配图更窄：
- 窗口顶栏：左侧三个圆点（锈红/琥珀/橄榄绿，模拟经典终端窗口的三色控制点），
  右侧文字"moebius · live run"。
- 主体是一段**打字机效果**逐行输出的模拟命令行日志，首次滚动进入可视区域时
  自动播放一次，播放节奏与 style-b Hero Demo 的日志时间线完全一致（同一套
  故事脚本）：
  1. `$ moebius run "调研三个竞品，输出一份对比报告"`（绿色提示符）
  2. `┌─ 调度   拆解为 8 个子任务，建立流水线`（琥珀色高亮）
  3. 依次派发给调研α/调研β/调研γ和撰写（各自负责"竞品 1/2/3：功能与定价"
     和"模板就绪，等待素材"）
  4. 调研α、调研β把资料交接给撰写（标记 ✓）
  5. 质检发现调研γ的定价数据有缺口，**退回**给调研γ补充（这一行用锈红色
     高亮，带 ↩ 符号，视觉上明确标出"出错-被拦截-返工"这一步）
  6. 调研γ补齐后重新交接（✓）
  7. 撰写完成对比报告初稿，交给质检
  8. 质检通过（结构完整、引用可溯）
  9. 调度把《竞品对比报告》交付给"你"（琥珀色高亮）
  10. 空行后输出总结行：`✔ done in 9.2s · 8 tasks parallel · human turns: 1`
      （绿色，与页面反复强调的"9.2 秒、人工介入 1 次"两个关键数字对齐）
- 终端窗口底部有一行统计条：并行任务数、消息数、人工介入（固定 1）、耗时，
  数字随日志播放实时刷新；右侧有一个"↻ replay"按钮可以重新播放整段演示。
- 这一区块本质上是把 style-b 里"Canvas 网络关系图 + 独立日志面板"两块内容，
  合并成了一个更聚焦、更像"看一段真实终端录屏"的单一演示。

### 01 · 现状（"六个对话框，都在等你回复"）
- 小节编号 `[01]`，标题同 style-b。
- 说明文字同 style-b：每个 AI 都很能干，但每走一步都要问你一句，AI 越多，
  你越忙。
- 3 列网格，铺 6 张**迷你终端窗口卡片**（而非 style-b 的"卡片+指示灯"样式），
  每张也是"深色终端窗口"造型（带三色控制点顶栏），内容包括：
  - 该 AI 会话名（竞品调研-A / 竞品调研-B / 报告撰写 / 数据整理 / 翻译校对 /
    本周周报⚑，最后一个同样特殊标记为"最紧急/hot"）
  - AI 卡住时说的那句话（与 style-b 文案一致）
  - 一行"等待你的回复 · mm:ss"状态，前面缀一个**持续旋转的 ASCII 转圈符号**
    （`⠋⠙⠹⠸⠼⠴...` 盲文点阵字符轮播，模拟命令行里常见的 loading spinner），
    且这个计时器是**实时递增**的（每秒 +1 秒，页面停留越久数字越大），比
    style-b 的静态"14 MIN"文本更"活"——直观传达"它们真的一直在原地等你，
    时间还在走"。
- 收尾文案换成了强调 spinner 的说法："六个转圈的 spinner 永远在转，因为
  瓶颈不是模型不够快——是每一步都要路过你。"

### 02 · 换接法（对应 style-b 的"02 重组"，但不是滚动驱动动画，而是静态并排
对比 + 持续自动播放的小动画）
- 小节编号 `[02]`，标题"让 agent 对话 agent"；背景切换为浅一档的米色面板
  以做视觉分区。
- 说明文字："改动只有一处：把「人守着每个 agent」换成「agent 直接连接
  agent」。你从每条线上退出，只保留发令与验收两端。"
- 并排两个终端窗口，做**左右对比**（这是与 style-b 滚动重组动画在呈现方式上
  的最大差异——style-b 靠"滚动"讲故事，style-c 靠"两个终端窗口并排+各自
  持续跑一段 ASCII 动画"讲故事）：
  - **左侧 `topology --before （现在）`**：用 ASCII 字符画出"你"与六个任务
    分别各自一条独立连线（`你 ──●──▶ [竞品调研-A]` 这种形式），线上的
    "●"符号做脉冲移动动画，且每条线里"●"有一半时间会停在原地不动、旁边
    浮现"⠿ 等你回复"字样，直观表现"六条线各自独立、且经常卡住等待"。
  - **右侧 `topology --after （Moebius）`**：用 ASCII 字符画出一张更复杂的
    拓扑图——"你"只连一条线到"调度"，调度再用三条分支线连到调研α/β/γ，
    三者汇入"撰写"，撰写连到"质检"，质检再用一条线绕回连到"你"（形成闭环），
    图里还专门画了一行"质检 ✗ 不合格 ──●── 退回重做"，表现质检失败会被打回
    这一机制。右侧几条线上的"●"移动得比左侧快，且是持续脉冲（不会像左侧
    那样"卡住不动"），暗示"自动流转、不再等人"。
  - 两个终端窗口标题栏分别标注了"（现在）"和"（Moebius）"作为强调标签。
- 图注："● = 一次工作交付。左边每条线一颗，颗颗要过你；右边你只出现在两端，
  中间连成川流。"

### 03 · 证据
- 小节编号 `[03]`，标题"效率不是形容词，是这些数字"。
- 四列卡片（每张卡片有独立黑色描边+硬阴影，比 style-b 的"一整条通栏表格"更
  像四张独立像素卡片），每张包含：
  - 一个从 0 计数动画滚动到目标值的等宽大字数字
  - 数字下方一排 10 格的"像素电量条"（`.meter`），点亮格数与该项数据的量级
    成正比（例如 8 个并行任务→点亮 8 格中的一部分，具体点亮数由数据自定义），
    用像素电量条替代了 style-b 的"闪烁小方块指示灯"
  - 四项数据同 style-b：8 子任务并行推进 / 11 次 agent 间自动交接（注意这里
    是 **11**，与 style-b 的 **12** 不同——两个版本这一项数字略有出入）/
    1 次人工介入 = 最初那句话 / 9.2s 从指令到交付
- 脚注："全部来自上面终端里那次真实运行，非营销估算。"（这里明确指代的是
  本页"终端演示区"，而不是 style-b 里 Hero 区右侧的关系图 Demo——因为
  style-c 把关系图动画替换成了终端演示，指代对象随之改变。）

### 04 · 机制
- 小节编号 `[04]`，标题"流水线为何不出错"；背景同样切到浅米色面板分区。
- 引导句与 style-b 基本一致："agent 互相对话，不会越传越错吗？——会，如果
  靠 agent 小心。所以我们靠工位。"
- 三列像素卡片（同样带黑描边+硬阴影），每张顶部是一个 8×8 网格拼出的极简
  像素图标（比 style-b 的图标网格更粗粝、颗粒感更强），内容：
  1. **01 分工**：每个 agent 只做一类事，复杂任务先被拆成一串简单任务。
  2. **02 标准化交接**：结构化格式交接，不靠"感觉传话"，每次交接可追溯、
     可回放。
  3. **03 质检**：文案略有加强，写作"审察 agent **拿着放大镜**守在出口，
     不合格的交付原路退回重做"——这句"拿着放大镜"与 Hero 配图里最后一个
     机器人手持放大镜检查晶体的画面是**呼应/联动**的细节。

### 05 · CTA（结尾行动号召）
- 居中排版，标签 `[05]`，标题"发出你的第一颗指令"，副标题"一句话的成本，
  一条流水线的产出。"
- 一个终端窗口造型的输入区（标题栏"moebius · new run"），内部是一行
  `$` 提示符 + 输入框（占位文字用命令行语法呈现：
  `moebius run "调研三个竞品，输出一份对比报告"`）+ 一个"RUN ↵"像素按钮。
- 点击按钮或按回车后，下方淡入确认文案："▸ 指令已进入流水线 · 公测名额将
  通过邮件发放"。同样是纯前端模拟，没有真实提交行为。

## 动效技术小结（供理解交互行为，非视觉必需）

- 小节标题/正文进入可视区域时同样有"从下方淡入"的一次性揭示动画，逻辑与
  style-b 相同。
- 终端打字机日志、等待卡片的旋转 spinner、拓扑对比区的 ASCII 脉冲动画、
  数字计数与像素电量条填充，全部是**持续/循环播放**的小动画（而不是
  style-b 那种"一次性播完就停"的关系图 Demo），所以 style-c 整体给人的
  感觉是页面"一直在轻微地活动"，氛围更接近"实时监控终端"。
- 页面遵循系统级"减少动态效果"偏好：开启该设置后，旋转 spinner、拓扑脉冲
  等循环动画会停止刷新，揭示动画和平滑滚动也会被禁用。
- 页面中只有一处真实图片资源（Hero 下方那张像素机器人流水线插画，内联
  base64 JPEG），其余全部图形（导航 Logo、机制区图标）仍是 Canvas/HTML
  绘制，不是图片文件。

## 与 style-b-mosaic.html 的关键差异速览

| 维度 | style-b（马赛克） | style-c（像素/终端） |
|---|---|---|
| 基调色 | 冷灰阶 + 单一陶土橙强调 | 暖牛皮纸 + 琥珀/锈红/橄榄多色 |
| 核心视觉隐喻 | 关系图 Canvas 动画（网络节点+连线） | 深色终端窗口（打字机日志） |
| "解决方案"小节呈现方式 | 330vh 滚动驱动的沉浸式重组动画 | 静态并排的双终端 ASCII 拓扑对比 |
| Hero 是否配图 | 无实图，纯 Canvas/文字 | 有一张像素机器人流水线插画（真实图片资源） |
| 按钮/卡片质感 | 平面描边，hover 反色 | 硬阴影像素按钮，hover/active 有位移反馈 |
| 等待中卡片形式 | 静态时长文本 + 闪烁指示灯 | 深色终端卡片 + 实时递增计时器 + 旋转 spinner |
| 页脚落款 | 仅公司/产品名 + 年份 | 多一句彩蛋文案"MADE OF PIXELS & PIPELINES" |
| 数据："agent 间自动交接"次数 | 12 | 11 |
