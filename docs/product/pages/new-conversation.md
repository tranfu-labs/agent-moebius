# 新建对话窗口

> 状态：讨论中草案
>
> 所属页面：主页面
>
> 当前范围：先承接主页面侧边栏“新建对话”入口、项目选择和添加项目的产品边界；窗口的完整布局与创建流程继续逐项讨论。
>
> 产品基线：[agent-moebius 产品需求文档](../prd.md)
>
> 关联入口：[主页面侧边栏](./main-sidebar.md)

## 页面目标

新建对话窗口让用户先确定对话所属项目和 Agent 团队，再开始描述目标。添加项目属于这个窗口，不属于主页面侧边栏。

## 用户场景

- 用户从侧边栏固定的“新建对话”入口开始一段对话。
- 用户从某个项目行的新建入口进入，并希望该项目已经被预选。
- 用户尚未添加任何项目，需要先添加项目，之后才能创建对话。

## 入口与去向

- 从[主页面侧边栏](./main-sidebar.md)顶部“新建对话”进入。
- 从侧边栏项目行的新建入口进入时，携带该项目作为预选项目。
- 创建成功后进入该对话的主页面时间线。
- 取消后返回进入前的主页面状态，不创建对话。

## 页面结构

当前只确认功能骨架，具体排版继续讨论：

```text
┌ 新建对话 ────────────────────────────┐
│ 项目       [选择已有项目] [添加项目] │
│ Agent 团队 [选择团队]                │
│                                      │
│ 描述你的目标…                        │
│                         [取消] [创建] │
└──────────────────────────────────────┘
```

## 区域与信息

- 项目：对话必须归属一个可用项目。
- 添加项目：通过系统文件夹选择器添加本地项目，并在成功后选中它。
- Agent 团队：创建对话前选择负责该目标的团队；默认规则沿用总 PRD，具体控件继续讨论。
- 目标描述：具体输入形态和是否允许先创建空白对话仍待讨论。

## 操作与反馈

- 没有可用项目时，“创建”不可用，并明确提示用户先添加项目。
- 添加项目被取消或失败时，不创建项目或对话，并保留窗口中的其他输入。
- 所选文件夹已经绑定一个活动项目时，不重复添加；已经移除的旧项目不占用活动绑定，重新添加同一路径会创建新项目，旧项目的归档对话不会自动混入。
- 从项目行进入时预选对应项目，但用户在创建前可以改选其他可用项目。
- 创建进行中时防止重复提交。

## 页面状态

| 状态 | 表现 | 可用操作 |
| --- | --- | --- |
| 有可用项目 | 显示项目和团队选择 | 选择项目、选择团队、填写目标、创建或取消 |
| 没有项目 | 创建不可用，提示先添加项目 | 添加项目、取消 |
| 添加项目中 | 显示明确进度，防止重复选择 | 取消行为是否可中止系统选择器由平台能力决定 |
| 创建中 | 创建操作显示进度并防止重复提交 | 等待结果 |
| 创建失败 | 保留已填内容，显示可理解错误 | 重试或取消 |

## 弹层与危险操作

- 添加项目使用系统文件夹选择器，不删除、移动或修改用户文件。
- 本窗口当前没有删除类危险操作。

## 响应式与窗口行为

- 窗口必须在主页面最小可用宽度下保持项目、团队和主要操作可达。
- 具体使用独立页面、模态窗口还是主内容区状态，继续讨论后确定。

## 关联流程

- 添加第一个项目并创建第一段对话。
- 为已有项目创建对话并选择 Agent 团队。

对应流程 PRD 尚未建立；本页暂只记录已确认的单页边界。

## 指标与验收

1. 从侧边栏顶部点击“新建对话”能够进入本窗口。
2. 没有可用项目时不能创建对话，并能从本窗口发起添加项目。
3. 从某个项目行进入时，该项目被预选。
4. 取消、添加失败或创建失败不会产生重复项目、半成品项目或重复对话。

## 非目标

当前草案不定义：

- 新建对话窗口的高保真视觉。
- Agent 团队选择器的完整交互。
- 创建后主时间线的结构和行为。
- 项目重命名、排序、重新定位或移除。
