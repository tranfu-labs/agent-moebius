# 主页面侧边栏

> 状态：讨论中草案
>
> 所属页面：主页面
>
> 范围：只定义左侧栏及其直接触发行为；搜索弹窗、Agent 团队管理和设置页面分别定义。
>
> 产品基线：[agent-moebius 产品需求文档](../prd.md)

## 页面目标

侧边栏是主页面中稳定存在的导航与任务入口。它让用户可以快速发起对话、在项目之间定位已有对话、识别需要关注的新结果或人工处理事项，并进入 Agent 团队管理或应用设置。

侧边栏只回答三个问题：

1. 我现在可以从哪里开始一段对话？
2. 我的项目和对话在哪里，哪一段正在运行或等待我？
3. 我如何进入 Agent 团队管理和应用设置？

侧边栏不是工作流图、运行日志或项目仪表盘。用户无需先理解内部编排结构，便能找到项目和对话。

## 用户场景

- 用户打开应用，从最近使用的项目和对话继续工作。
- 用户不关心具体项目入口，直接从顶部创建一段新对话。
- 用户已明确目标项目，直接在该项目下创建新对话。
- 用户在多个项目间切换，查找正在运行、产生新结果或等待人工处理的对话。
- 用户需要添加、重命名、排序或移除项目。
- 用户需要进入 Agent 团队管理、搜索或设置。
- 用户希望临时扩大对话区域，关闭侧边栏后继续工作。

## 入口与去向

### 入口

- 应用完成首次启动配置后，默认进入主页面并显示侧边栏。
- 用户从 Agent Teams、设置或搜索结果返回主页面时，恢复离开前的项目展开状态和当前对话。
- 侧边栏关闭时，主内容区顶部保留“打开侧边栏”按钮。

### 去向

- “新建对话”在当前项目下创建空白对话；没有可用项目时先进入添加项目流程。
- “搜索”打开全局搜索弹窗。搜索弹窗的内容与行为不在本文档定义。
- “Agent Teams”进入 Agent 团队管理界面。
- “添加项目”打开系统文件夹选择器。
- 项目内“新建对话”在指定项目下创建并选中空白对话。
- 点击对话进入对应主页面时间线。
- “设置”进入应用设置界面。

## 页面结构

### 默认展开

```text
┌──────────────────────────────┐
│                       [关闭] │  标题栏与窗口拖拽区
├──────────────────────────────┤
│ ＋ 新建对话                  │
│ ⌕ 搜索                       │
│ ◇ Agent Teams                │
├──────────────────────────────┤
│ 项目                  ＋ 添加 │
│                              │
│ ▾ 项目 A            [＋][⋯] │
│    对话 1                  ● │  红点：需要人工处理
│  » 对话 2                  ● │  闪烁点：运行中
│    对话 3                    │
│    对话 4                  ● │  蓝点：有未读结果
│                              │
│ ▸ 项目 B              ● [⋯] │  折叠时显示最高优先状态点
├──────────────────────────────┤
│ ⚙ 设置                       │
└──────────────────────────────┘
```

图例：

- `»`：当前选中的对话。
- 红色 `●`：需要用户回答、确认、验收或处理异常。
- 蓝色 `●`：Agent 本轮已经结束并产生用户尚未查看的新结果。
- 闪烁 `●`：Agent 正在运行。
- 无点：当前静止，且没有未读结果或人工处理事项。
- 折叠项目只显示一个代表其子对话最高优先状态的点，优先级为红点、蓝点、闪烁点；展开后由对话行自行显示，不在项目行重复。

### 侧边栏关闭

```text
┌──────────────────────────────────────────────┐
│ [打开侧边栏]    当前项目 / 当前对话          │
├──────────────────────────────────────────────┤
│                                              │
│                 主内容区                     │
│                                              │
└──────────────────────────────────────────────┘
```

关闭后侧边栏完全隐藏，主内容区占用释放出的宽度。打开按钮属于主页面在侧栏关闭状态下的恢复入口；本文档只规定它必须存在，不规定主内容区其他结构。

### 没有项目

```text
┌──────────────────────────────┐
│                       [关闭] │
├──────────────────────────────┤
│ ＋ 新建对话                  │
│ ⌕ 搜索                       │
│ ◇ Agent Teams                │
├──────────────────────────────┤
│ 项目                  ＋ 添加 │
│                              │
│ 还没有项目                   │
│ 添加一个本地文件夹开始工作   │
│ [添加项目]                   │
├──────────────────────────────┤
│ ⚙ 设置                       │
└──────────────────────────────┘
```

顶部“新建对话”仍然可点击，但点击后进入添加项目流程，不创建没有项目归属的对话。

### 项目目录不可用

```text
│ ▾ 项目 A                  [⋯] │
│   原文件夹已不可用             │
│    历史对话 1                  │
│    历史对话 2                  │
```

历史对话仍可查看，但全部只读。项目目录不可用时，不允许创建新对话、发送新消息、切换团队或启动新的 Agent 执行；项目也不提供“重新绑定到其他目录”的能力。

项目在 Agent 运行期间变为不可用时：已经拥有有效隔离工作副本的执行可以完成当前一步，但完成后停止自动接力并把对话标为红点；依赖原目录、已经无法安全继续的执行立即停止并标为红点。侧边栏不得在执行已经无法继续时仍长期显示闪烁运行点。

## 区域与信息

### 标题栏与关闭按钮

- 顶部第一行保留桌面窗口拖拽区域。
- macOS 为系统交通灯预留空间，不在此区域重复显示产品品牌名。
- “关闭侧边栏”固定在顶部右侧，不随项目列表滚动。
- 图标按钮必须有“关闭侧边栏”辅助名称和悬停提示。

### 应用级操作

应用级操作固定在项目列表上方，不随项目列表滚动：

1. 新建对话。
2. 搜索。
3. Agent Teams。

三个入口使用相同的导航行高度、文字层级和交互样式，不把 Agent Teams 藏入设置，也不通过特殊颜色或徽标额外突出。它的常驻位置本身负责让用户意识到产品存在可配置的 Agent 团队。

### 项目标题行

- 显示“项目”区标题和“添加项目”操作。
- “添加项目”始终可发现，不藏入设置或项目菜单。
- 项目列表是侧边栏唯一可滚动的主要区域；顶部应用级操作和底部设置保持固定。

### 项目行

每个项目行包含：

- 展开或折叠控制。
- 项目显示名称。
- 项目内新建对话按钮。
- 项目更多菜单。
- 折叠时必要的状态聚合。
- 用户排序所需的拖动或等价键盘操作。

项目默认显示本地文件夹名。用户设置显示名称后，只改变应用内名称，不重命名磁盘上的文件夹。

项目顺序由用户控制并持久保存，不因对话运行或产生新消息而自动变化。刚添加的项目默认出现在列表最上方、自动展开并成为当前项目。用户可以拖动项目调整顺序；键盘用户可以通过项目菜单执行等价的上移、下移和移到顶部操作。

### 对话行

每个对话行只显示：

- 一行对话标题。
- 一个必要的状态图标。
- 当前选中态。

长标题单行截断，悬停显示完整标题。侧边栏不显示运行角色、消息摘要、消息数量、阶段名称或文件路径。

子对话和普通对话在侧边栏中平铺为同级条目，不使用无限缩进的树。自动创建的子对话必须具有可理解的标题，悬停说明和辅助名称标明“来自：<父对话名称>”；父子关系的完整表达进入会话内容和对应流程。

### 底部设置

- “设置”固定在侧边栏底部，不随项目列表滚动。
- 底部不常驻数据库路径、运行目录、内部标识或原始错误。
- 本地引擎异常若导致主功能不可用，可以在设置入口附近显示非技术化状态提示；诊断细节进入设置或诊断界面。

## 操作与反馈

### 关闭与打开侧边栏

- 点击关闭按钮后立即隐藏侧边栏，主内容区扩展。
- 主内容区出现固定的打开按钮；点击后恢复侧边栏。
- 恢复时保留当前对话、项目展开状态和项目列表滚动位置。
- 应用记住用户最后一次选择；下次启动恢复关闭或展开状态，但在首次启动引导期间始终保持侧边栏可见。

### 应用级新建对话

- 存在当前可用项目时，在当前项目下创建并选中一个空白对话。
- 创建后的主内容区必须明确显示目标项目和默认 Agent 团队；具体控件由主内容区文档定义。
- 空白对话在首次发送前允许从主内容区切换项目。
- 对话产生消息、开始执行或形成父子关系后，项目归属锁定。
- 没有可用项目时，打开添加项目流程；用户取消后不创建任何项目或对话。
- 创建进行中时避免重复创建，并在操作位置显示明确进度。

### 搜索

- 点击打开全局搜索弹窗。
- 侧边栏保持在背景中，关闭弹窗后恢复原选中状态。
- 搜索范围、结果分组、键盘行为和空态由搜索弹窗文档定义。

### Agent Teams

- 点击进入 Agent 团队管理界面。
- 进入后该入口呈现当前选中态。
- 返回主页面时恢复离开前的项目、对话和侧边栏状态。

### 添加项目

- 点击后打开系统原生文件夹选择器。
- 用户取消时不产生项目、对话或错误记录。
- 选择已存在于列表中的同一文件夹时，不重复添加；系统选中并展开已有项目。
- 添加成功后，项目出现在列表最上方、自动展开并成为当前项目；主内容区立即指向该新项目的空态，但不创建持久化空白对话。
- 新项目空态提供明确的新建对话入口。只有用户主动新建或开始输入时，才产生对话记录。
- 文件夹无法访问时，不写入半个项目；在侧边栏操作附近给出可理解的失败提示和重试入口。

### 项目内新建对话

- 点击项目行的加号，直接在该项目下创建并选中空白对话。
- 项目目录失效、项目正在变更或创建操作尚未完成时，该按钮不可用并说明原因。
- 一次点击只创建一个对话；重复点击或慢响应不得生成重复空白对话。

### 展开和折叠项目

- 点击展开控制切换项目子列表，不改变当前对话。
- 项目展开时，由子对话自行显示状态，项目行不重复展示状态点。
- 项目折叠时只显示一个代表子对话最高优先状态的点，不显示数量。红点优先于蓝点，蓝点优先于闪烁点。
- 当前选中对话所在项目不能因导航自动折叠；用户仍可手动折叠，主内容区继续显示当前对话。

### 选择对话

- 点击对话行打开对应时间线，并将选中态移动到该行。
- 切换失败时保留原对话，不出现侧栏选中了新对话而主内容仍是旧对话的双事实状态。
- 打开带蓝点的对话并成功展示最新结果后，清除蓝点。
- 打开带红点的对话不会仅因查看而清除红点；只有用户完成所需操作或任务重新进入可自动推进状态后才清除。

### 项目与对话菜单

项目更多菜单至少包含：

- 在文件管理器中显示。
- 修改显示名称。
- 上移、下移和移到顶部，用于键盘等无法拖动的场景。
- 移除项目。

菜单打开时必须明确对应哪个项目。菜单操作不得因为列表滚动或项目状态刷新而错误作用到其他项目。

每个对话行在悬停、键盘聚焦或上下文操作时提供对话菜单。当前侧栏范围内，对话菜单至少包含“归档”；后续增加的操作不得混入项目菜单。

## 页面状态

| 状态 | 侧边栏表现 | 可用操作 |
| --- | --- | --- |
| 正常 | 显示应用级入口、项目、对话和设置 | 全部正常操作 |
| 没有项目 | 项目区显示添加引导 | 添加项目、Agent Teams、设置；新建对话转入添加项目 |
| 项目没有对话 | 项目展开后显示“还没有对话” | 项目内新建对话、项目菜单 |
| 项目目录不可用 | 项目行显示非技术化警告；历史对话只读；无法继续的运行转为红点 | 查看历史、移除；禁止新建、发送、切换团队和执行 |
| 项目正在添加 | 添加入口显示进度，防止重复提交 | 已有项目和对话仍可浏览 |
| 对话正在创建 | 对应新建入口显示进度并防止重复创建 | 其他项目仍可浏览；同一目标项目不再创建 |
| 数据加载中 | 项目区显示保持结构的加载占位 | 顶部入口和设置按依赖情况可用 |
| 加载失败 | 项目区显示简短错误与重试 | Agent Teams、设置和重试可用 |
| 侧边栏关闭 | 侧边栏隐藏，主内容区显示打开按钮 | 当前对话继续运行和展示 |
| 应用级配置变更中 | 仅禁用可能冲突的入口并显示进度 | 无关浏览和已运行任务不被阻断 |

### 对话状态点与顺序

侧边栏只用一个小圆点表达需要用户感知的状态：

1. **红点**：需要用户回答、确认、验收或处理异常；红点表示人的动作会阻塞后续推进。
2. **蓝点**：本轮 Agent 已结束并产生用户尚未查看的新结果。蓝点不代表任务已经完成，也不要求用户立即行动。
3. **闪烁点**：Agent 正在运行。闪烁必须克制，不造成持续视觉干扰。
4. **无点**：对话静止，且没有未读结果或需要人工处理的事项。

同一时刻只显示一个点：红点优先于蓝点，蓝点优先于闪烁点。点的颜色不是唯一信息来源，悬停提示和辅助名称必须分别读作“需要你处理”“有新结果”“正在运行”。

对话按最近一次人工打开或创建的时间排列。后台 Agent 的流式输出、新结果和计时更新只改变状态点，不自动重排列表，避免并行任务让侧边栏持续跳动。

### 归档

- 产品不设置“已完成”侧栏状态，也不自动把对话收入完成分组。
- 用户可以从对话菜单主动归档一段不再需要出现在侧边栏中的对话。
- 归档不会删除消息、执行记录或交付结果。
- 运行中的对话不能归档；用户需要先中止或等待当前运行结束。
- 当前对话被归档后，选择同项目中相邻的可见对话；没有其他对话时显示项目空态。
- 归档后的查找和恢复入口由搜索或后续归档管理文档定义，本文不提前规定其界面。

## 弹层与危险操作

### 修改项目显示名称

- 使用轻量弹层输入应用内显示名称。
- 空名称恢复为文件夹名，而不是产生“未命名项目”。
- 修改不影响磁盘目录、已有会话或 Agent 执行路径。

### 移除项目

- “移除项目”必须经过确认弹窗。
- 弹窗明确说明：移除只让项目从 agent-moebius 中消失，绝不删除或修改磁盘上的项目文件夹。
- 默认保留该项目的本地对话历史，重新添加同一路径时允许恢复；永久删除历史属于数据管理能力，不与普通移除混为一个默认动作。
- 项目存在运行中的 Agent 时禁止移除，提示用户先回到对应对话中止或等待运行结束。
- 确认移除后，侧边栏选择相邻可用对话；没有其他项目时进入无项目状态。
- 移除失败时项目保持可见，当前选择不变，并提供重试。

搜索弹窗、Agent 团队管理和设置页面的内部弹层不属于本文档。

## 响应式与窗口行为

- 侧边栏在正常桌面窗口中保持稳定宽度，不因项目名或对话名变长挤压主内容区。
- 初始版本不提供拖拽调整侧边栏宽度；用户通过关闭和打开控制可用空间。
- 长名称单行截断，并提供完整名称的悬停提示和辅助名称。
- 窗口高度不足时只滚动项目列表，顶部操作和底部设置保持可达。
- 窗口宽度低于主页面最低可用宽度时，可以自动关闭侧边栏；窗口恢复后遵循用户最近一次明确选择，不反复自动开合。
- macOS 集成标题栏下，侧边栏顶部正确避让系统交通灯并保留可拖拽区域；其他平台保留等价的窗口操作空间。
- 所有图标按钮必须可键盘聚焦，并提供可读名称和悬停说明。

## 关联流程

以下流程将在对应流程 PRD 建立后链接，本文档只记录侧边栏职责：

- 首次启动与添加第一个项目。
- 添加、目录不可用和移除项目。
- 创建对话并选择 Agent 团队。
- 从等待、失败或中断状态恢复任务。
- 搜索并跳转到项目或对话。
- 归档并恢复对话。

## 指标与验收

### 产品指标

- 用户从打开应用到进入目标对话所需的操作次数和时间。
- 用户从任意主页面状态创建一段项目内对话所需的操作次数。
- 项目或对话选择失败后产生错误选中状态的次数，目标为零。
- 重复添加项目、重复创建空白对话和误移除项目的次数，目标为零。
- 红点对话被用户定位并完成所需动作的比例。

指标采集必须遵守本地产品的数据边界；是否上传或仅保存在本地由设置与隐私规则另行定义。

### 验收标准

1. 打开主页面，侧边栏顶部依次显示关闭按钮、新建对话、搜索和 Agent Teams；三个应用级入口使用相同视觉层级，项目区与设置分别位于中部和底部。
2. 点击关闭侧边栏，主内容区扩展并出现打开按钮；重新打开后，当前对话、项目展开状态和滚动位置保持不变。
3. 没有项目时点击新建对话，进入添加项目流程；取消后不产生项目或对话。
4. 添加一个新文件夹，项目只出现一次、置于列表最上方、自动展开并成为当前项目；主内容区指向该项目空态，但不存在持久化空白对话。
5. 重复添加同一文件夹，不创建重复项目，而是选中已有项目。
6. 点击项目行加号，只在该项目下创建一个对话；慢响应或重复点击不产生重复空白对话。
7. 红点、蓝点和闪烁点分别表达需要人工处理、有未读结果和运行中；静止对话无点，且辅助名称不依赖颜色即可读懂状态。
8. 后台 Agent 输出和状态变化只更新状态点，不改变对话顺序；对话按最近一次人工打开或创建的时间稳定排列。
9. 对话没有“已完成”状态或完成折叠组；归档后从侧栏消失但记录仍保留，运行中的对话不能归档。
10. 子对话在项目下与其他对话平铺，不因父子深度增加侧栏缩进；悬停和辅助名称能够识别其父对话来源。
11. 项目目录不可用后仍可只读查看历史对话，但不能新建、发送、切换团队或启动 Agent；侧栏不提供重新绑定到其他目录的能力。
12. 用户可以拖动项目排序，键盘用户可以通过菜单完成等价排序；重启应用后顺序保持，新项目默认置顶。
13. 移除项目必须明确说明不会删除磁盘文件；存在运行任务时不能移除，移除失败时项目和当前选择保持不变。
14. 搜索只打开搜索弹窗，Agent Teams 和设置只进入对应界面；本文档定义的侧边栏不承载这些界面的内部内容。
15. 项目或对话切换失败时，侧边栏选中态与主内容保持在原对象，不出现双事实状态。
16. 项目数量超过一屏时，只有项目列表滚动，顶部操作、侧栏关闭按钮和底部设置始终可达。
17. 所有图标入口均可使用键盘操作，并具有无需依赖图标外形即可理解的辅助名称。
