# 主页面侧边栏

> 状态：讨论中草案
>
> 所属页面：主页面
>
> 范围：只定义左侧栏及其直接触发行为；搜索弹窗、Agent 团队管理和设置页面分别定义。
>
> 产品基线：[agent-moebius 产品需求文档](../prd.md)

## 页面目标

侧边栏是主页面中稳定存在的导航与任务入口。它让用户可以快速发起对话、在项目之间定位已有对话、识别需要关注的会话状态，并进入 Agent 团队管理或应用设置。

侧边栏只回答三个问题：

1. 我现在可以从哪里开始一段对话？
2. 我的项目和对话在哪里，哪一段正在运行或等待我？
3. 我如何进入 Agent 团队管理和应用设置？

侧边栏不是工作流图、运行日志或项目仪表盘。用户无需先理解内部编排结构，便能找到项目和对话。

## 用户场景

- 用户打开应用，从最近使用的项目和对话继续工作。
- 用户不关心具体项目入口，直接从顶部创建一段新对话。
- 用户已明确目标项目，直接在该项目下创建新对话。
- 用户在多个项目间切换，查找正在运行、等待处理或已经完成的对话。
- 用户需要添加、重命名、重新绑定或移除项目。
- 用户需要进入 Agent 团队管理、搜索或设置。
- 用户希望临时扩大对话区域，关闭侧边栏后继续工作。

## 入口与去向

### 入口

- 应用完成首次启动配置后，默认进入主页面并显示侧边栏。
- 用户从 Agent 团队管理、设置或搜索结果返回主页面时，恢复离开前的项目展开状态和当前对话。
- 侧边栏关闭时，主内容区顶部保留“打开侧边栏”按钮。

### 去向

- “新建对话”在当前项目下创建空白对话；没有可用项目时先进入添加项目流程。
- “搜索”打开全局搜索弹窗。搜索弹窗的内容与行为不在本文档定义。
- “管理 Agent 团队”进入 Agent 团队管理界面。
- “添加项目”打开系统文件夹选择器。
- 项目内“新建对话”在指定项目下创建并选中空白对话。
- 点击对话进入对应主页面时间线。
- “设置”进入应用设置界面。

## 页面结构

### 默认展开

```text
┌──────────────────────────────┐
│                       [关闭] │  标题栏与窗口拖拽区
├──────────────────────────────┤
│ ＋ 新建对话                  │
│ ⌕ 搜索                       │
│ ◇ 管理 Agent 团队            │
├──────────────────────────────┤
│ 项目                  ＋ 添加 │
│                              │
│ ▾ 项目 A            [＋][⋯] │
│    对话 1                 ✋ │
│  » 对话 2                  ● │
│    对话 3                    │
│    ▸ 已完成 (3)              │
│                              │
│ ▸ 项目 B             ●1 [⋯] │
├──────────────────────────────┤
│ ⚙ 设置                       │
└──────────────────────────────┘
```

图例：

- `»`：当前选中的对话。
- `✋`：等待用户采取行动。
- `●`：Agent 正在运行。
- 无状态图标：当前没有运行，也不等待用户。
- 项目折叠时的 `●1`：该项目内有一个正在运行的对话；同类聚合状态只在子对话不可见时出现。

### 侧边栏关闭

```text
┌──────────────────────────────────────────────┐
│ [打开侧边栏]    当前项目 / 当前对话          │
├──────────────────────────────────────────────┤
│                                              │
│                 主内容区                     │
│                                              │
└──────────────────────────────────────────────┘
```

关闭后侧边栏完全隐藏，主内容区占用释放出的宽度。打开按钮属于主页面在侧栏关闭状态下的恢复入口；本文档只规定它必须存在，不规定主内容区其他结构。

### 没有项目

```text
┌──────────────────────────────┐
│                       [关闭] │
├──────────────────────────────┤
│ ＋ 新建对话                  │
│ ⌕ 搜索                       │
│ ◇ 管理 Agent 团队            │
├──────────────────────────────┤
│ 项目                  ＋ 添加 │
│                              │
│ 还没有项目                   │
│ 添加一个本地文件夹开始工作   │
│ [添加项目]                   │
├──────────────────────────────┤
│ ⚙ 设置                       │
└──────────────────────────────┘
```

顶部“新建对话”仍然可点击，但点击后进入添加项目流程，不创建没有项目归属的对话。

### 项目目录失效

```text
│ ▾ 项目 A                  [⋯] │
│   原文件夹已不可用             │
│   [重新选择]                   │
│    历史对话 1                  │
│    历史对话 2                  │
```

历史对话仍可查看。项目目录恢复前，不允许创建新对话或启动新的 Agent 执行。

## 区域与信息

### 标题栏与关闭按钮

- 顶部第一行保留桌面窗口拖拽区域。
- macOS 为系统交通灯预留空间，不在此区域重复显示产品品牌名。
- “关闭侧边栏”固定在顶部右侧，不随项目列表滚动。
- 图标按钮必须有“关闭侧边栏”辅助名称和悬停提示。

### 应用级操作

应用级操作固定在项目列表上方，不随项目列表滚动：

1. 新建对话。
2. 搜索。
3. 管理 Agent 团队。

“新建对话”是唯一的主操作，视觉权重高于另外两个入口。“搜索”和“管理 Agent 团队”使用普通导航行样式，不与项目混在同一层级。

### 项目标题行

- 显示“项目”区标题和“添加项目”操作。
- “添加项目”始终可发现，不藏入设置或项目菜单。
- 项目列表是侧边栏唯一可滚动的主要区域；顶部应用级操作和底部设置保持固定。

### 项目行

每个项目行包含：

- 展开或折叠控制。
- 项目显示名称。
- 项目内新建对话按钮。
- 项目更多菜单。
- 折叠时必要的状态聚合。

项目默认显示本地文件夹名。用户设置显示名称后，只改变应用内名称，不重命名磁盘上的文件夹。

项目按用户添加顺序保持稳定，不因对话运行或产生新消息而重新排序。刚添加的项目出现在列表末尾并自动展开、成为当前项目。

### 对话行

每个对话行只显示：

- 一行对话标题。
- 一个必要的状态图标。
- 当前选中态。

长标题单行截断，悬停显示完整标题。侧边栏不显示运行角色、消息摘要、消息数量、阶段名称或文件路径。

子对话和普通对话在侧边栏中平铺为同级条目，不使用无限缩进的树。父子关系在会话内容和对应流程中表达。

### 底部设置

- “设置”固定在侧边栏底部，不随项目列表滚动。
- 底部不常驻数据库路径、运行目录、内部标识或原始错误。
- 本地引擎异常若导致主功能不可用，可以在设置入口附近显示非技术化状态提示；诊断细节进入设置或诊断界面。

## 操作与反馈

### 关闭与打开侧边栏

- 点击关闭按钮后立即隐藏侧边栏，主内容区扩展。
- 主内容区出现固定的打开按钮；点击后恢复侧边栏。
- 恢复时保留当前对话、项目展开状态、已完成分组展开状态和项目列表滚动位置。
- 应用记住用户最后一次选择；下次启动恢复关闭或展开状态，但在首次启动引导期间始终保持侧边栏可见。

### 应用级新建对话

- 存在当前可用项目时，在当前项目下创建并选中一个空白对话。
- 空白对话在首次发送前允许从主内容区切换项目。
- 对话产生消息、开始执行或形成父子关系后，项目归属锁定。
- 没有可用项目时，打开添加项目流程；用户取消后不创建任何项目或对话。
- 创建进行中时避免重复创建，并在操作位置显示明确进度。

### 搜索

- 点击打开全局搜索弹窗。
- 侧边栏保持在背景中，关闭弹窗后恢复原选中状态。
- 搜索范围、结果分组、键盘行为和空态由搜索弹窗文档定义。

### 管理 Agent 团队

- 点击进入 Agent 团队管理界面。
- 进入后该入口呈现当前选中态。
- 返回主页面时恢复离开前的项目、对话和侧边栏状态。

### 添加项目

- 点击后打开系统原生文件夹选择器。
- 用户取消时不产生项目、对话或错误记录。
- 选择已存在于列表中的同一文件夹时，不重复添加；系统选中并展开已有项目。
- 添加成功后，项目出现在列表末尾、自动展开并成为当前项目，同时创建一个可立即开始输入的空白对话。
- 文件夹无法访问时，不写入半个项目；在侧边栏操作附近给出可理解的失败提示和重试入口。

### 项目内新建对话

- 点击项目行的加号，直接在该项目下创建并选中空白对话。
- 项目目录失效、项目正在变更或创建操作尚未完成时，该按钮不可用并说明原因。
- 一次点击只创建一个对话；重复点击或慢响应不得生成重复空白对话。

### 展开和折叠项目

- 点击展开控制切换项目子列表，不改变当前对话。
- 项目展开时，由子对话自行显示状态，项目行不重复展示同一状态计数。
- 项目折叠时，聚合显示等待用户、需要处理和运行中的数量；已完成数量不冒泡。
- 当前选中对话所在项目不能因导航自动折叠；用户仍可手动折叠，主内容区继续显示当前对话。

### 选择对话

- 点击对话行打开对应时间线，并将选中态移动到该行。
- 切换失败时保留原对话，不出现侧栏选中了新对话而主内容仍是旧对话的双事实状态。
- 选中的已完成对话必须保持可见；若通过搜索等外部入口打开，自动展开对应项目和“已完成”分组。

### 项目更多菜单

项目更多菜单至少包含：

- 在文件管理器中显示。
- 修改显示名称。
- 重新选择文件夹，仅在目录失效或用户主动发起时出现。
- 移除项目。

菜单打开时必须明确对应哪个项目。菜单操作不得因为列表滚动或项目状态刷新而错误作用到其他项目。

## 页面状态

| 状态 | 侧边栏表现 | 可用操作 |
| --- | --- | --- |
| 正常 | 显示应用级入口、项目、对话和设置 | 全部正常操作 |
| 没有项目 | 项目区显示添加引导 | 添加项目、团队管理、设置；新建对话转入添加项目 |
| 项目没有对话 | 项目展开后显示“还没有对话” | 项目内新建对话、项目菜单 |
| 项目目录失效 | 项目行显示非技术化警告和重新选择入口 | 查看历史、重新选择、移除；禁止新建和执行 |
| 项目正在添加 | 添加入口显示进度，防止重复提交 | 已有项目和对话仍可浏览 |
| 对话正在创建 | 对应新建入口显示进度并防止重复创建 | 其他项目仍可浏览；同一目标项目不再创建 |
| 数据加载中 | 项目区显示保持结构的加载占位 | 顶部入口和设置按依赖情况可用 |
| 加载失败 | 项目区显示简短错误与重试 | 团队管理、设置和重试可用 |
| 侧边栏关闭 | 侧边栏隐藏，主内容区显示打开按钮 | 当前对话继续运行和展示 |
| 应用级配置变更中 | 仅禁用可能冲突的入口并显示进度 | 无关浏览和已运行任务不被阻断 |

### 对话状态与排序

侧边栏使用以下用户可理解状态：

1. **等待你**：流程必须由用户确认、回答或验收才能继续，排在最前。
2. **需要处理**：自动恢复已经停止或任务遇到无法自行解决的问题，排在等待你之后。
3. **运行中**：Agent 正在执行，排在需要处理之后。
4. **静止**：当前没有运行，也不等待用户，不显示状态图标。
5. **已完成**：达到正式完成条件，进入默认折叠的“已完成”分组。

状态只在真实状态迁移时改变排序。流式输出、新消息或计时更新不得让对话行持续跳动。同一状态内按进入该状态的时间稳定排列；如果没有新的状态迁移，保持已有顺序。

侧边栏不提供“未读”状态或未读数量。新消息本身不等于用户需要行动，不能制造额外的阅读义务。

## 弹层与危险操作

### 修改项目显示名称

- 使用轻量弹层输入应用内显示名称。
- 空名称恢复为文件夹名，而不是产生“未命名项目”。
- 修改不影响磁盘目录、已有会话或 Agent 执行路径。

### 重新选择项目文件夹

- 主要用于原目录移动、重命名或失效后的恢复。
- 重新选择前说明已有历史将继续归属于当前项目记录。
- 选择成功前保留原项目事实；失败或取消不得产生部分绑定。
- 目标文件夹已经属于另一个项目时停止操作，并引导用户选择已有项目或取消。

### 移除项目

- “移除项目”必须经过确认弹窗。
- 弹窗明确说明：移除只让项目从 agent-moebius 中消失，绝不删除或修改磁盘上的项目文件夹。
- 默认保留该项目的本地对话历史，重新添加同一文件夹时允许恢复；永久删除历史属于数据管理能力，不与普通移除混为一个默认动作。
- 项目存在运行中的 Agent 时禁止移除，提示用户先回到对应对话中止或等待运行结束。
- 确认移除后，侧边栏选择相邻可用对话；没有其他项目时进入无项目状态。
- 移除失败时项目保持可见，当前选择不变，并提供重试。

搜索弹窗、Agent 团队管理和设置页面的内部弹层不属于本文档。

## 响应式与窗口行为

- 侧边栏在正常桌面窗口中保持稳定宽度，不因项目名或对话名变长挤压主内容区。
- 初始版本不提供拖拽调整侧边栏宽度；用户通过关闭和打开控制可用空间。
- 长名称单行截断，并提供完整名称的悬停提示和辅助名称。
- 窗口高度不足时只滚动项目列表，顶部操作和底部设置保持可达。
- 窗口宽度低于主页面最低可用宽度时，可以自动关闭侧边栏；窗口恢复后遵循用户最近一次明确选择，不反复自动开合。
- macOS 集成标题栏下，侧边栏顶部正确避让系统交通灯并保留可拖拽区域；其他平台保留等价的窗口操作空间。
- 所有图标按钮必须可键盘聚焦，并提供可读名称和悬停说明。

## 关联流程

以下流程将在对应流程 PRD 建立后链接，本文档只记录侧边栏职责：

- 首次启动与添加第一个项目。
- 添加、失效、重新绑定和移除项目。
- 创建对话并选择 Agent 团队。
- 从等待、失败或中断状态恢复任务。
- 搜索并跳转到项目或对话。

## 指标与验收

### 产品指标

- 用户从打开应用到进入目标对话所需的操作次数和时间。
- 用户从任意主页面状态创建一段项目内对话所需的操作次数。
- 项目或对话选择失败后产生错误选中状态的次数，目标为零。
- 重复添加项目、重复创建空白对话和误移除项目的次数，目标为零。
- 等待用户的对话被用户定位并采取行动的完成率。

指标采集必须遵守本地产品的数据边界；是否上传或仅保存在本地由设置与隐私规则另行定义。

### 验收标准

1. 打开主页面，侧边栏顶部依次显示关闭按钮、新建对话、搜索和管理 Agent 团队，项目区与设置分别位于中部和底部。
2. 点击关闭侧边栏，主内容区扩展并出现打开按钮；重新打开后，当前对话、项目展开状态、已完成分组和滚动位置保持不变。
3. 没有项目时点击新建对话，进入添加项目流程；取消后不产生项目或对话。
4. 添加一个新文件夹，项目只出现一次、自动展开并选中一个可输入的空白对话。
5. 重复添加同一文件夹，不创建重复项目，而是选中已有项目。
6. 点击项目行加号，只在该项目下创建一个对话；慢响应或重复点击不产生重复空白对话。
7. 对话按等待你、需要处理、运行中、静止、已完成排序；流式输出期间列表不持续跳动。
8. 已完成对话默认折叠；从外部入口打开一个已完成对话时，对应项目和已完成分组自动展开，使选中行可见。
9. 子对话在项目下与其他对话平铺，不因父子深度增加侧栏缩进。
10. 项目目录失效后仍可查看历史对话，但不能新建对话或启动 Agent；重新选择成功前不改变原绑定。
11. 移除项目必须明确说明不会删除磁盘文件；存在运行任务时不能移除，移除失败时项目和当前选择保持不变。
12. 搜索只打开搜索弹窗，管理 Agent 团队和设置只进入对应界面；本文档定义的侧边栏不承载这些界面的内部内容。
13. 项目或对话切换失败时，侧边栏选中态与主内容保持在原对象，不出现双事实状态。
14. 项目数量超过一屏时，只有项目列表滚动，顶部操作、侧栏关闭按钮和底部设置始终可达。
15. 所有图标入口均可使用键盘操作，并具有无需依赖图标外形即可理解的辅助名称。
