# 设置

> 状态：讨论中草案
>
> 所属页面：应用级设置
>
> 当前范围：先承接主页面侧边栏的设置入口，以及查看已移除项目和已归档对话的产品边界；完整设置项继续逐项讨论。
>
> 产品基线：[agent-moebius 产品需求文档](../prd.md)
>
> 关联入口：[主页面侧边栏](./main-left-sidebar.md)

## 页面目标

设置页面承载应用级配置和不适合常驻主界面的数据管理入口。项目从 Sidebar 移除后，其对话会被归档；用户可以在设置中查看已移除项目和已归档对话。

## 用户场景

- 用户从 Sidebar 进入设置。
- 用户需要查看已经从 Sidebar 移除的项目。
- 用户需要查看因项目移除或主动归档而隐藏的对话。

## 入口与去向

- 从[主页面侧边栏](./main-left-sidebar.md)底部“设置”进入。
- 返回主页面时恢复离开前的 Sidebar 开关、项目展开状态和当前对话；若原项目已被移除，则返回新建对话且未选择项目的状态。

## 页面结构

当前只确认与 Sidebar 相关的功能骨架：

```text
┌ 设置 ────────────────────────────┐
│ 常规                             │
│ Agent 与团队                     │
│ 已移除项目与已归档对话           │
└──────────────────────────────────┘
```

## 区域与信息

- “已移除项目与已归档对话”区域必须区分项目的“已移除”和对话的“已归档”语义。
- 重新添加相同本地路径会创建新的活动项目；旧项目及其归档对话继续作为历史记录保留，不自动并入新项目。
- 该区域至少允许查看记录；恢复、永久删除和数据保留期限继续讨论。

## 操作与反馈

- 从设置返回主页面时，不得让已移除项目重新出现在 Sidebar。
- 查看归档记录不得改变其状态。

## 页面状态

| 状态 | 表现 | 可用操作 |
| --- | --- | --- |
| 没有归档记录 | 显示明确空态 | 返回其他设置或主页面 |
| 有已移除项目 | 显示项目及其归档对话 | 查看 |
| 加载失败 | 显示可理解错误 | 重试或返回 |

## 弹层与危险操作

当前范围只定义查看，不定义永久删除；因此没有新增危险操作。

## 响应式与窗口行为

设置导航和归档记录在主窗口最小可用宽度下保持可达；完整布局继续讨论。

## 关联流程

- 移除项目并归档其对话。
- 查看已移除项目和已归档对话。

## 指标与验收

1. 从 Sidebar 点击“设置”能进入本页面。
2. 项目移除后可在设置中看到“已移除项目”，其对话显示为“已归档”。
3. 查看这些记录不会让项目重新出现在 Sidebar。

## 非目标

当前草案不定义：

- 完整设置分类和全部设置项。
- 已移除项目或归档对话的恢复规则。
- 永久删除和数据保留期限。
