# 主页面侧边栏

> 状态：已确认，可开发
>
> 所属页面：主页面
>
> 范围：只定义左侧栏及其直接触发行为；[新建对话窗口](./new-conversation.md)、[全局搜索](./search.md)、Agent 团队管理和[设置](./settings.md)分别定义。
>
> 产品基线：[moebius 产品需求文档](../prd.md)

## 页面目标

侧边栏是主页面中稳定存在的导航与任务入口。它让用户可以快速发起对话、在项目之间定位已有对话、识别需要关注的新结果或人工处理事项，并进入 Agent 团队管理或应用设置。

侧边栏只回答三个问题：

1. 我现在可以从哪里开始一段对话？
2. 我的项目和对话在哪里，哪一段正在运行或等待我？
3. 我如何进入 Agent 团队管理和应用设置？

侧边栏不是工作流图、运行日志或项目仪表盘。用户无需先理解内部编排结构，便能找到项目和对话。

## 用户场景

- 用户打开应用，从最近使用的项目和对话继续工作。
- 用户不关心具体项目入口，直接从顶部创建一段新对话。
- 用户已明确目标项目，直接在该项目下创建新对话。
- 用户在多个项目间切换，查找正在运行、产生新结果或等待人工处理的对话。
- 用户尚未添加项目时，从固定的新建对话入口进入添加项目流程。
- 用户需要重命名、排序、修复或移除已有项目。
- 用户需要进入 Agent 团队管理、搜索或设置。
- 用户希望临时扩大对话区域，关闭侧边栏后继续工作。

## 入口与去向

### 入口

- 应用完成首次启动配置后，默认进入主页面并显示侧边栏。
- 非首次启动时，应用恢复用户上次成功打开的项目和用户发起的对话，让用户从离开的位置继续工作。
- 没有可恢复的历史选择，或上次选择的对话已归档、所属项目已移除时，主内容区进入“新建对话”且不预选项目；应用不得擅自打开项目列表中的第一个项目或对话。
- 用户从 Agent 团队、设置或搜索结果返回主页面时，恢复离开前的项目展开状态和当前对话。
- 侧边栏关闭时，主内容区顶部保留“打开侧边栏”按钮。

### 去向

- “新建对话”打开[新建对话窗口](./new-conversation.md)；项目选择、添加项目和对话创建由该窗口承接。
- “搜索”打开[全局搜索](./search.md)。搜索内容与结果行为由该页面定义。
- “Agent 团队”进入 Agent 团队管理界面。
- 项目内“新建对话”打开同一个[新建对话窗口](./new-conversation.md)，并预选对应项目。
- 点击对话进入对应主页面时间线。
- “设置”进入[设置](./settings.md)。

## 页面结构

### 默认展开

```text
┌──────────────────────────────┐
│ ● ● ●                [关闭] │  红绿灯安全区与侧栏控制行
│ [Logo] Moebius               │  独立品牌行与窗口拖拽区
├──────────────────────────────┤
│ ＋ 新建对话                  │
│ ⌕ 搜索                       │
│ ◇ Agent 团队                 │
├──────────────────────────────┤
│ 项目                         │
│                              │
│ ▾ 项目 A            [＋][⋯] │
│    对话 1                  ● │  红点：需要人工处理
│  » 对话 2                  ● │  闪烁点：运行中
│    对话 3                    │
│    对话 4                  ● │  蓝点：有未读结果
│                              │
│ ▸ 项目 B              ● [⋯] │  折叠时显示最高优先状态点
├──────────────────────────────┤
│ ⚙ 设置                       │
└──────────────────────────────┘
```

图例：

- `»`：当前选中的对话。
- 红色 `●`：需要用户回答、确认、验收或处理异常。
- 蓝色 `●`：Agent 本轮已经结束并产生用户尚未查看的新结果。
- 闪烁 `●`：Agent 正在运行。
- 无点：当前静止，且没有未读结果或人工处理事项。
- 折叠项目只显示一个代表其子对话最高优先状态的点，优先级为红点、蓝点、闪烁点；展开后由对话行自行显示，不在项目行重复。

### 侧边栏关闭

```text
┌──────────────────────────────────────────────┐
│ [打开侧边栏]    当前项目 / 当前对话          │
├──────────────────────────────────────────────┤
│                                              │
│                 主内容区                     │
│                                              │
└──────────────────────────────────────────────┘
```

关闭后侧边栏完全隐藏，主内容区占用释放出的宽度。打开按钮属于主页面在侧栏关闭状态下的恢复入口；本文档只规定它必须存在，不规定主内容区其他结构。

### 没有项目

```text
┌──────────────────────────────┐
│ ● ● ●                [关闭] │
│ [Logo] Moebius               │
├──────────────────────────────┤
│ ＋ 新建对话                  │
│ ⌕ 搜索                       │
│ ◇ Agent 团队                 │
├──────────────────────────────┤
│ 项目                         │
│                              │
│ 还没有项目                   │
│ 从“新建对话”添加第一个项目   │
├──────────────────────────────┤
│ ⚙ 设置                       │
└──────────────────────────────┘
```

顶部“新建对话”仍然可点击，并打开[新建对话窗口](./new-conversation.md)。没有可用项目时，该窗口禁止创建对话并提供添加项目入口；侧边栏自身不承载添加项目流程。

### 项目目录不可用

```text
│ ▾ 项目 A            [🔧][⋯] │  红色扳手：修复项目文件夹
│   原文件夹已不可用             │
│    历史对话 1                  │
│    历史对话 2                  │
```

历史对话仍可查看，但全部只读。项目目录不可用时，不允许创建新对话、发送新消息、切换团队或启动新的 Agent 执行。项目行显示独立的红色扳手修复按钮；悬停提示“当前项目本地文件夹未找到，可以指定新的文件夹”，点击后让用户指出同一项目被移动或重命名后的新位置。修复入口不得藏入项目更多菜单。

项目在 Agent 运行期间变为不可用时：已经拥有有效隔离工作副本的执行可以完成当前一步，但完成后停止自动接力并把对话标为红点；依赖原目录、已经无法安全继续的执行立即停止并标为红点。侧边栏不得在执行已经无法继续时仍长期显示闪烁运行点。

## 区域与信息

### 品牌、标题栏与关闭按钮

- 顶部第一行使用统一的窗口 header 高度并保留桌面窗口拖拽区域；系统交通灯和该行内控件由同一容器自然垂直居中，不使用单独的位置补偿。
- 顶部固定显示产品 Logo 和品牌名 `Moebius`，形成主窗口持续可见的产品识别；`moebius` 只作为项目与技术标识，不作为该位置的界面文案。
- macOS 为系统交通灯预留空间；Logo、品牌名、拖拽区域和系统交通灯不得重叠。
- “关闭侧边栏”固定在顶部右侧，不随项目列表滚动。
- 图标按钮必须有“关闭侧边栏”辅助名称和悬停提示。

### 应用级操作

应用级操作固定在项目列表上方，不随项目列表滚动：

1. 新建对话。
2. 搜索。
3. Agent 团队。

三个入口使用相同的导航行高度、文字层级和交互样式，不把 Agent 团队藏入设置，也不通过特殊颜色或徽标额外突出。它的常驻位置本身负责让用户意识到产品存在可配置的 Agent 团队。英文界面对应名称为 `Agent Teams`；当前产品暂不展开完整 i18n 设计。

### 项目标题行

- 显示“项目”区标题。
- 添加项目不在侧边栏出现，由[新建对话窗口](./new-conversation.md)统一承接。
- 项目列表是侧边栏唯一可滚动的主要区域；顶部应用级操作和底部设置保持固定。

### 项目行

每个项目行包含：

- 展开或折叠控制。
- 项目显示名称。
- 项目内新建对话按钮。
- 项目更多菜单。
- 折叠时必要的状态聚合。
- 项目目录不可用时的独立红色扳手修复按钮。
- 项目行自身承载的拖动排序能力，不另设独立拖动把手。

项目默认显示本地文件夹名。用户设置显示名称后，只改变应用内名称，不重命名磁盘上的文件夹。

项目顺序由用户控制并持久保存，不因对话运行、产生新消息或出现任何状态点而自动变化。刚添加的项目默认出现在列表最上方、自动展开并成为当前项目。用户直接拖动项目行调整顺序，不显示独立拖动把手；点击与拖动必须通过明确的移动阈值区分，拖动结束不得误触展开或折叠。项目菜单不提供上移、下移或移到顶部命令。

### 对话行

每个对话行只显示：

- 一行对话标题。
- 一个必要的状态图标。
- 当前选中态。

长标题单行截断，悬停显示完整标题。侧边栏不显示运行角色、消息摘要、消息数量、阶段名称或文件路径。

侧边栏只列用户自己发起的对话。团队为了推进一个目标而拆出的子对话不出现在这里，它们的入口是父对话时间线中的卡片，见[主页面会话区](./main-conversation.md#区域与信息)。

> **子对话是否进侧边栏已在 2026-07-21 被重新裁决。** 本文原先规定子对话与普通对话平铺为同级条目、以“来自：<父对话名称>”标明来源。该规定被推翻，理由是：子任务是团队的内部产物，不是用户主动管理的对象，而侧边栏是用户管理自己工作的地方；让它们进入侧边栏会使侧栏长度随团队的拆分行为膨胀，用户拆得越细、侧栏越不可用。代价是用户不能从侧边栏直接跳到某个子任务，必须先回到父对话——这被接受，因为子任务本来就只在父对话的语境里有意义。

### 底部设置

- “设置”固定在侧边栏底部，不随项目列表滚动；点击进入[设置](./settings.md)。
- 底部不常驻数据库路径、运行目录、内部标识或原始错误。
- 本地引擎异常若导致主功能不可用，可以在设置入口附近显示非技术化状态提示；诊断细节进入设置或诊断界面。

## 操作与反馈

### 关闭与打开侧边栏

- 点击关闭按钮后立即隐藏侧边栏，主内容区扩展。
- 主内容区出现固定的打开按钮；点击后恢复侧边栏。
- 恢复时保留当前对话、项目展开状态和项目列表滚动位置。
- 应用记住用户最后一次选择；下次启动恢复关闭或展开状态，但在首次启动引导期间始终保持侧边栏可见。

### 应用级新建对话

- 点击后打开[新建对话窗口](./new-conversation.md)，不在侧边栏内直接创建空白对话。
- 没有可用项目时仍可进入该窗口，但创建操作不可用，用户需要先在窗口内添加项目。
- 项目选择、Agent 团队选择、取消、创建和失败恢复由新建对话窗口定义。

### 搜索

- 点击打开[全局搜索](./search.md)。
- 侧边栏保持在背景中，关闭弹窗后恢复原选中状态。
- 搜索范围、结果分组、键盘行为和空态由全局搜索文档定义。

### Agent 团队

- 点击进入 Agent 团队管理界面。
- 进入后该入口呈现当前选中态。
- 返回主页面时恢复离开前的项目、对话和侧边栏状态。

### 项目内新建对话

- 点击项目行的加号，打开[新建对话窗口](./new-conversation.md)并预选该项目；不会在侧边栏点击瞬间直接创建持久化空白对话。
- 项目目录失效或项目正在变更时，该按钮不可用并说明原因。

### 展开和折叠项目

- 点击项目行的展开箭头或项目名称切换项目子列表，不改变当前对话；项目本身不进入独立项目主页，也不产生项目选中态。
- 项目行同时是拖动排序载体；超过拖动阈值后只执行排序，结束时不得继续触发点击展开或折叠。项目行内的 `＋` 和 `⋯` 是独立按钮，不触发行点击或拖动排序。
- 项目展开时，由子对话自行显示状态，项目行不重复展示状态点。
- 项目折叠时只显示一个代表子对话最高优先状态的点，不显示数量。红点优先于蓝点，蓝点优先于闪烁点。
- 当前对话所在项目允许被用户手动折叠；主内容区继续显示当前对话，不退出、不切换，应用也不得仅因为当前对话属于该项目而自动重新展开。
- 项目折叠时暂时隐藏其对话列表，不产生项目选中态；再次展开后，原对话仍显示选中态。

### 选择对话

- 点击对话行打开对应时间线，并将选中态移动到该行。
- 切换失败时保留原对话，不出现侧栏选中了新对话而主内容仍是旧对话的双事实状态。
- 打开带蓝点的对话并成功展示最新结果后，清除蓝点。
- 打开带红点的对话不会仅因查看而清除红点；只有导致红点的那件事被处理掉（重试成功、换人接手推进、项目文件夹或团队恢复可用）之后才清除。
- 闪烁运行点在运行结束时自动消失；若本轮结束后产生尚未查看的新结果且最后一条未提及任何成员则转为蓝点，若这一步没跑起来、卡住或反复重试仍未成功则转为红点。

### 项目与对话菜单

项目更多菜单只包含：

- 在文件管理器中显示。
- 修改显示名称。
- 移除项目。

菜单打开时必须明确对应哪个项目。菜单操作不得因为列表滚动或项目状态刷新而错误作用到其他项目。项目文件夹修复不属于更多菜单，而由项目行上的独立红色扳手按钮承载。

每个对话行在悬停、键盘聚焦或上下文操作时提供对话菜单。当前侧栏范围内，对话菜单只包含“归档”和“复制对话记录路径”；后续增加的操作不得混入项目菜单。

### 复制对话记录路径

- 服务复盘：用户想把一段对话的完整经过交给对话之外的工具或另一个 AI 阅读分析。
- 每段对话的完整经过——消息、系统记录和各步运行过程——本来就以机器可读的记录文件持续落在磁盘上（见 [ADR-0004](../../adr/0004-jsonl-session-fact-log.md)）。触发菜单项时把该文件的路径复制到剪贴板；成功反馈说明“路径已复制”，让用户知道下一步是把它粘给要读的工具。
- 路径指向的是持续更新的记录本身，不是导出时刻的快照：对话继续推进时，拿到路径的阅读工具可以重新读取并跟上最新经过。运行中的对话同样可以复制。
- 同一段对话的路径稳定不变，复制一次即可长期使用。
- 复制路径是显式动作；这不违背“界面文案不出现内部标识”——菜单项和反馈文案不展示路径本身，路径只进剪贴板。
- 该文件就是对话记录的事实本身：外部工具应只读；文件被用户或外部工具修改、删除时，对话历史随之受损，产品按加载异常如实呈现，不承诺恢复。

## 页面状态

| 状态 | 侧边栏表现 | 可用操作 |
| --- | --- | --- |
| 正常 | 显示应用级入口、项目、对话和设置 | 全部正常操作 |
| 没有项目 | 项目区提示从“新建对话”添加第一个项目 | 新建对话、Agent 团队、设置 |
| 项目没有对话 | 项目展开后显示“还没有对话” | 项目内新建对话、项目菜单 |
| 项目目录不可用 | 项目行显示独立红色扳手修复按钮及提示；历史对话只读；无法继续的运行转为红点 | 查看历史、修复项目文件夹、移除；禁止新建、发送、切换团队和执行 |
| 数据加载中 | 项目区显示保持结构的加载占位 | 顶部入口和设置按依赖情况可用 |
| 加载失败 | 项目区显示简短错误与重试 | Agent 团队、设置和重试可用 |
| 侧边栏关闭 | 侧边栏隐藏，主内容区显示打开按钮 | 当前对话继续运行和展示 |
| 应用级配置变更中 | 仅禁用可能冲突的入口并显示进度 | 无关浏览和已运行任务不被阻断 |

### 对话状态点与顺序

侧边栏只用一个小圆点表达需要用户感知的状态：

1. **红点**：这段对话遇到了需要用户处理的事，红点表示它已经无法自己往下走。触发来源由[主页面会话区](./main-conversation.md#侧边栏状态点的触发)定义，本文不重复列举。
2. **蓝点**：本轮 Agent 已结束并产生用户尚未查看的新结果，且最后一条消息没有提及任何成员。蓝点不代表任务已经完成，也不要求用户立即行动；最后一条提及了某个成员时属于团队内部仍在传递，不给蓝点。
3. **闪烁点**：Agent 正在运行。闪烁必须克制，不造成持续视觉干扰。
4. **无点**：对话静止，且没有未读结果或需要用户处理的事项。用户自己按下“停下”之后是无点。

同一时刻只显示一个点：红点优先于蓝点，蓝点优先于闪烁点。点的颜色不是唯一信息来源，悬停提示和辅助名称必须分别读作“需要你处理”“有新结果”“正在运行”。

> **红点来源已在 2026-07-21 被重新裁决。** 本文原先定义红点为“需要用户回答、确认、验收或处理异常”。该定义被推翻，理由是：它假设 Agent 会主动声明自己需要人工裁决，而产品目前没有这样的信号源，导致红点既可能漏报（Agent 卡住却不声明），也可能误报（把正常的团队内部传递当成等待用户）。新定义只认那些**产品能确定判断**的事实，从而保证一条硬性质——用户被红点带回一段对话时，页面上一定有一条说明发生了什么的记录，不会出现“被叫回来却看不出要我干什么”。让成员主动声明“我需要用户裁决”是更精确的红点来源，需要另行约定，见[主页面会话区](./main-conversation.md#待讨论)的待讨论条目。

对话只按创建时间倒序排列，最新创建的对话固定在最上面，最早创建的在最下面。人工打开、当前选中、后台 Agent 的流式输出、新结果、等待人工、运行状态和计时更新都只改变对应界面状态，不得改变对话顺序，避免并行任务让侧边栏持续跳动。

### 归档

- 产品不设置“已完成”侧栏状态，也不自动把对话收入完成分组。
- 用户可以从对话菜单主动归档一段不再需要出现在侧边栏中的对话。
- 归档不会删除消息、执行记录或交付结果。
- 运行中的对话不能归档；用户需要先中止或等待当前运行结束。
- 当前对话被归档后，选择同项目中相邻的可见对话；没有其他对话时显示项目空态。
- 归档后的查找和恢复由[全局搜索](./search.md)承接；搜索提供“包含已归档对话”开关，并为归档结果提供恢复入口。

## 弹层与危险操作

### 修改项目显示名称

- 使用轻量弹层输入应用内显示名称。
- 空名称恢复为文件夹名，而不是产生“未命名项目”。
- 修改不影响磁盘目录、已有会话或 Agent 执行路径。

### 修复项目文件夹

- 该能力只处理同一个项目文件夹在应用外被移动或重命名的情况，不用于把一份项目历史改接到另一个项目。
- 项目文件夹不可用时，项目行显示独立的红色扳手图标按钮；悬停提示“当前项目本地文件夹未找到，可以指定新的文件夹”。
- 用户点击该修复按钮后，通过系统文件夹选择器指出项目的新位置。
- 确认前同时显示原位置和用户选择的新位置，说明应用将从新位置继续使用原项目历史。
- 确认成功后更新当前项目的位置，保留项目顺序、显示名称、对话历史和当前选择，并恢复新建、发送、团队切换和 Agent 执行。
- moebius 不移动、复制或重命名磁盘上的任何文件；它只更新自己记录的项目位置。
- 用户取消、所选目录不可访问或更新失败时，项目继续保持原记录和只读状态，不产生第二个项目或部分更新。
- 一个本地文件夹同一时刻只能绑定一个项目。所选目录已经被另一个现有项目使用时，禁止重复绑定或合并两个项目记录；提示用户该目录已绑定其他项目，并允许转到已有项目或重新选择。

### 移除项目

- “移除项目”必须经过确认弹窗。
- 弹窗明确说明：移除只让项目从 moebius 中消失，绝不删除或修改磁盘上的项目文件夹。
- 移除后项目从 Sidebar 消失，项目语义为“已移除”；该项目下的对话统一转为“已归档”，消息、执行记录和交付结果仍保留，并可在[设置](./settings.md)中查看。
- 重新添加与已移除项目相同的本地路径时，创建一个新的活动项目；旧项目继续作为“已移除项目”保留，旧对话继续归档，不自动合并、恢复或混入新项目。项目被移除后释放该路径的活动绑定，因此这不违反“一个文件夹同一时刻只能绑定一个活动项目”。
- 项目存在运行中的 Agent 时，不直接执行移除，而是显示独立警告弹窗；用户可以取消，也可以选择强制中止全部相关运行后继续移除。强制中止和移除必须作为受控顺序执行，任一步失败都不得伪装成完整成功。
- 确认移除当前项目后，不自动切换到相邻项目或对话；主内容区进入[新建对话窗口](./new-conversation.md)且项目未选择的状态。
- 移除失败时项目保持可见，当前选择不变，并提供重试。

全局搜索、Agent 团队管理和设置页面的内部弹层不属于本文档。

## 响应式与窗口行为

- 侧边栏提供可拖动的右侧分隔边界，用户可以在保证 Sidebar 与主内容仍可用的最小、最大范围内调整宽度；精确范围由高保真设计和实现验证确定。
- 长名称单行截断，并提供完整名称的悬停提示和辅助名称。
- 窗口高度不足时只滚动项目列表，顶部操作和底部设置保持可达。
- 窗口宽度低于主页面最低可用宽度时，可以自动关闭侧边栏；窗口恢复后遵循用户最近一次明确选择，不反复自动开合。
- macOS 集成标题栏下，侧边栏顶部正确避让系统交通灯并保留可拖拽区域；侧边栏打开和关闭状态下的控制按钮都与系统交通灯垂直中心对齐。其他平台保留等价的窗口操作空间。
- 所有图标按钮必须可键盘聚焦，并提供可读名称和悬停说明。

## 关联流程

以下流程将在对应流程 PRD 建立后链接，本文档只记录侧边栏职责：

- 首次启动与添加第一个项目。
- 添加、目录不可用、重新定位和移除项目。
- 创建对话并选择 Agent 团队。
- 从等待、失败或中断状态恢复任务。
- 搜索并跳转到项目或对话。
- 归档并恢复对话。

## 指标与验收

### 产品指标

- 用户从打开应用到进入目标对话所需的操作次数和时间。
- 用户从任意主页面状态创建一段项目内对话所需的操作次数。
- 项目或对话选择失败后产生错误选中状态的次数，目标为零。
- 重复添加项目、重复创建空白对话和误移除项目的次数，目标为零。
- 红点对话被用户定位并完成所需动作的比例。

指标采集必须遵守本地产品的数据边界；是否上传或仅保存在本地由设置与隐私规则另行定义。

### 验收标准

1. 打开中文主页面，侧边栏首行只保留 macOS 红绿灯安全区和与红绿灯垂直中心对齐的关闭按钮，产品 Logo 与品牌名位于下一独立行，随后依次显示新建对话、搜索和 Agent 团队；三个应用级入口使用相同视觉层级，项目区与设置分别位于中部和底部。
2. 点击关闭侧边栏，主内容区扩展并出现打开按钮；重新打开后，当前对话、项目展开状态和滚动位置保持不变。
3. 点击新建对话进入新建对话窗口；没有项目时该窗口禁止创建并提供添加项目入口，侧边栏不直接承载添加流程。
4. 点击项目行加号进入同一个新建对话窗口，并预选对应项目；进入窗口本身不创建持久化空白对话。
5. 红点、蓝点和闪烁点分别表达需要人工处理、有未读结果和运行中；静止对话无点，用户按下“停下”后无点，且辅助名称不依赖颜色即可读懂状态。红点的具体触发来源按[主页面会话区](./main-conversation.md#侧边栏状态点的触发)核对。
6. 对话只按创建时间倒序稳定排列，最新创建的在最上面；人工打开、当前选中、后台 Agent 输出和任何状态变化都不得改变顺序。
7. 对话没有“已完成”状态或完成折叠组；归档后从侧栏消失但记录仍保留，运行中的对话不能归档。
8. 团队拆出的子对话不出现在侧边栏；项目下只列用户自己发起的对话，侧栏长度不随团队的拆分行为变化。子对话的入口由[主页面会话区](./main-conversation.md#区域与信息)的时间线卡片承担。
9. 项目目录不可用后，项目行显示带明确提示的独立红色扳手修复按钮；历史对话仍可只读查看，但不能新建、发送、切换团队或启动 Agent。选择同一项目的新位置且通过“一个文件夹只能绑定一个项目”的校验后，原项目和对话历史保持不变并恢复工作能力。
10. 用户可以直接拖动项目行排序且无需独立拖动把手；拖动不误触展开或折叠，重启应用后顺序保持，新项目默认置顶，任何运行或状态变化都不自动重排项目。
11. 移除项目必须明确说明不会删除磁盘文件；项目从 Sidebar 消失，其对话变为归档并可在设置中查看。重新添加同一路径时创建新项目，旧历史不自动混入。存在运行任务时必须先经独立警告弹窗选择取消或强制中止；移除当前项目后进入新建对话且项目未选择的状态，移除失败时项目和当前选择保持不变。
12. 搜索只打开全局搜索，Agent 团队和设置只进入对应界面；本文档定义的侧边栏不承载这些界面的内部内容。
13. 项目或对话切换失败时，侧边栏选中态与主内容保持在原对象，不出现双事实状态。
14. 用户可以拖动 Sidebar 右侧边界调整宽度；长名称仍单行截断并可查看完整名称。
15. 项目数量超过一屏时，只有项目列表滚动，红绿灯安全区、侧栏关闭按钮、Logo、顶部操作和底部设置始终可达。
16. 窗口过窄时 Sidebar 自动关闭；窗口重新变宽后，最近一次明确选择为打开则恢复打开，用户手动关闭过则保持关闭。
17. 所有图标入口均可使用键盘操作，并具有无需依赖图标外形即可理解的辅助名称。
18. 选择第二个项目中的任意用户对话并重启应用后，恢复同一项目和对话；没有历史选择或历史选择已不可恢复时进入未选择项目的新建对话，不回退到第一个项目的默认或最早对话。
19. 对话菜单提供“复制对话记录路径”：触发后这段对话记录文件的稳定路径进入剪贴板；把该路径交给对话之外的阅读工具即可读到包含消息、系统记录和各步运行过程的完整经过，且对话继续推进后重新读取能跟上最新内容；复制失败时有可理解说明且剪贴板不被写入。

## 现状参考与产品缺口

现有 `ConversationSidebar` 已实现项目分组、项目内新建会话、会话选择、状态排序和已完成折叠；现有主页面壳已实现打开项目、项目列表、底部诊断入口和本地引擎状态。

这些实现是现状参考，不是本文档的需求上限。相较完整侧边栏，当前仍缺少或尚未达到本文要求的主要产品能力包括：

- 侧边栏关闭、恢复和状态记忆。
- 应用级新建对话、搜索和 Agent 团队入口。
- 标准设置入口。
- 项目折叠和状态聚合。
- 项目显示名称、手动排序、文件夹重新定位、移除及安全确认。
- 无项目、项目空白、目录不可用和局部失败状态。
- 红点、蓝点和闪烁点状态。
- 对话归档与恢复入口。
- 对话菜单的“复制对话记录路径”：对话消息目前只存在应用数据库中，磁盘上没有可直接阅读的对话记录文件；存储分层见 [ADR-0004](../../adr/0004-jsonl-session-fact-log.md)。

旧 `docs/wireframes/pages/console.md` 和 `openspec/changes/conversation-console/wireframes.md` 中与侧边栏有关的有效原则已被重新评估。本文保留项目与对话两层结构和运行状态可见；旧设计中的“已完成折叠”和“无未读语义”被本轮产品裁决取代，改为红点、蓝点、闪烁点和用户主动归档。2026-07-21 另有两项裁决：红点来源改由[主页面会话区](./main-conversation.md#侧边栏状态点的触发)定义，子对话不再进入侧边栏——两处的推翻理由都记在对应正文的引述块里。本文已经确认，后续侧边栏产品需求以本文为事实源，旧 Wireframe 不再新增侧边栏事实。
