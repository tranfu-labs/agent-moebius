## 页面目标

全局搜索帮助用户跨项目找到对话，并在需要时把已归档对话纳入搜索范围。搜索是已归档对话重新被发现和恢复的入口之一。

## 用户场景

- 用户从侧边栏固定的“搜索”入口查找已有对话。
- 用户只想搜索当前仍显示在侧边栏中的对话。
- 用户需要查找已经归档的对话，并选择是否恢复它。

## 入口与去向

- 从[主页面侧边栏](./main-left-sidebar.md)顶部“搜索”进入。
- 选择未归档对话后，返回主页面并打开对应时间线。
- 选择已归档对话时，允许查看并执行恢复；恢复后的去向继续讨论。
- 关闭或取消搜索后，恢复进入前的主页面选择和侧边栏状态。

## 页面结构

当前只确认功能骨架，具体排版继续讨论：

```text
┌ 搜索 ─────────────────────────────────┐
│ [输入关键词…]   [包含已归档对话：开/关] │
├────────────────────────────────────────┤
│ 搜索结果                               │
│ 项目 A / 对话 1                        │
│ 项目 B / 已归档对话 2       [恢复]     │
└────────────────────────────────────────┘
```

## 区域与信息

- 搜索输入：接收用户查询。
- “包含已归档对话”开关：明确控制归档对话是否进入本次搜索范围。
- 结果列表：至少能区分项目、对话和归档状态；完整分组与摘要仍待讨论。
- 恢复入口：只对已归档对话出现。

## 操作与反馈

- 切换“包含已归档对话”后，当前查询立即按新的范围刷新。
- 关闭该开关时，结果中不得出现已归档对话。
- 打开该开关时，已归档对话必须有明确的归档标识，不能与普通对话混淆。
- 恢复成功后，对话重新出现在所属项目的侧边栏列表中，并保持原创建时间，不作为新对话置顶。
- 搜索、跳转或恢复失败时保留查询和开关状态，并提供重试。

## 页面状态

| 状态 | 表现 | 可用操作 |
| --- | --- | --- |
| 初始 | 显示搜索输入和归档范围开关 | 输入查询、切换范围、关闭 |
| 搜索中 | 保留输入内容并显示进度 | 修改查询或关闭 |
| 无结果 | 明确说明当前范围没有匹配项 | 修改查询或切换归档范围 |
| 有普通结果 | 显示项目与对话结果 | 打开对话 |
| 有归档结果 | 明确标识已归档 | 查看或恢复 |
| 搜索失败 | 显示可理解错误 | 重试、修改查询或关闭 |
| 恢复中 | 对应结果显示进度并防止重复恢复 | 等待结果 |

## 弹层与危险操作

- 恢复归档不是危险操作，不需要二次确认。
- 搜索页面不提供永久删除对话或历史记录的能力。

## 响应式与窗口行为

- 搜索输入、归档范围开关和结果必须在主页面最小可用宽度下保持可达。
- 具体使用弹窗、抽屉还是独立主内容状态，继续讨论后确定。

## 关联流程

- 搜索并跳转到项目或对话。
- 归档并恢复对话。

对应流程 PRD 尚未建立；本页暂只记录已确认的单页边界。

## 指标与验收

1. 从侧边栏点击“搜索”能够进入本页面。
2. 页面提供明确的“包含已归档对话”开关。
3. 开关关闭时不返回归档对话，打开后可以返回并明确标识归档对话。
4. 恢复归档对话后，它重新出现在原项目中，且创建时间与内容不变。
5. 搜索或恢复失败不会清空查询，也不会产生重复对话。

## 非目标

当前草案不定义：

- 搜索结果的高保真视觉。
- 完整搜索字段、匹配算法和相关性排序。
- 永久删除归档对话。
- 项目、Agent 团队和设置内容的搜索范围。
