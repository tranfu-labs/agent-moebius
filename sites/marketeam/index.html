03 / 三个时刻

该问你时问你，
该返工时自己返。

交出去之后，一个环节干完，下一个环节自己接上。那根接力棒，不在你手里了。 盯进度这活，归它了。

02 处理

01 · ALIGN

先对齐：只问“你要什么”。

含糊的地方，宁可多问一句，也不闷头做错。怎么做，是它自己的事；哪些事必须由你拍板，在开工前说好。

目标有两种，请你选结果
A本周可上线的最小范围
B供内部评审的完整方案
它问目标，不把做法丢给你。
02 · VERIFY

再验收：按开工时说好的，一条一条查。

每份产出送到你手上之前，至少被验收一次。以前逐字逐行检查的是你；现在这个位置，由它顶上。

按开工要求检查
目标范围一致通过
必要内容齐全通过
交付说明缺失退回
03 · REDO

没通过：打回，重做，再验。

这一整圈它自己跑完。打回几圈还过不了，它不硬跑——带着卡住的地方，再来找你拍板。

复核未通过缺少必要说明
带原因退回处理只重做未通过的部分
修改后再次提交重新进入复核

能自己定的，它不来烦你；
只有你有权定的，才来找你。
其余的时间，它不出声，只干活。

04 / 痕迹

每份交付，都该带一张过程底单。

今天任何一个 AI 交回来的东西，你要么凭感觉决定信不信，要么自己从头再看一遍。过程刷过去就没了，手边没有一张能对着查的单子。

底单上会有不体面的记录——被打回、返工、重来。但恰恰是这些记录让你看得见：验收真的发生过。

完整的底单，我们还在做——标准先立在这里。这套标准，先管我们自己。

05 / 第一个任务

开源免注册，
交出你的第一个任务。

你需要的只有两样：一台 Mac，以及可用的 Codex CLI 或 Claude CLI。打开“终端”，任选一条版本命令；能返回版本号，就可以开始。

✓

这样确认：命令能返回版本号

$ codex --version

› 返回版本号 = 可以使用

$ claude --version

×

不能这样确认：只看图标或窗口样式

聊天窗口
请帮我做件事。
当然，请继续输入。

不确定自己有没有装？不要凭图标或界面猜。打开“终端”，运行 codex --version 或 claude --version；任一命令能返回版本号，就说明已经可用。

  • 免费开源，代码公开可查
  • 不用注册，不填密码，也不收任何密钥
  • Codex CLI 或 Claude CLI，任一即可
  • 任务在你的电脑上跑，用的是本机 CLI
  • 额度就是你平时用的那一笔，不另算
  • Moebius 本身不收费，也不加价
  • 对齐、验收和打回，都会出现在第一个任务里
