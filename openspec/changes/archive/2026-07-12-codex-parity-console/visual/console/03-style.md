# 三、风格 + 实践 — 本地多 agent 操作台

> 目的：在已确认的显示框架上锁定 Codex 桌面端的视觉密度，并验证多 agent 状态不会形成第二套控制台语言。

## 1. 页面性质判定

- **位置**：高度实用。
- **理由**：主场景是发起任务和连续阅读协作，视觉必须安静、直接、长时间可读；品牌表达让位于内容和输入。

## 3. 调性候选

### 候选「Codex 原生密度」

- **对标**：用户提供的 Codex 桌面端截图。
- **色板**：画布 `#FFFFFF` / 侧栏 `#F1F1F1` / 选中 `#DEDEDE` / 文字 `#222222` / 次文字 `#6E6E6E` / 边框 `rgba(0,0,0,0.10)` / 危险事实 `#C94141`。
- **字号比例**：11 / 12 / 13 / 14 / 16 / 26 像素。
- **留白密度**：侧栏紧、对话中、空态松。
- **圆角与阴影**：列表行 7 像素；输入器 18 像素并保留一层轻阴影；消息不使用卡片阴影。
- **HTML**：`03-alt-codex-native.html`。

### 候选「Codex 多 agent 强提示」

- **对标**：Codex 骨架加重角色与运行状态提示。
- **色板**：与 Codex 原生密度相同，但 agent 头像与状态面使用更深灰阶和更明显边框。
- **字号比例**：11 / 12 / 13 / 14 / 16 / 26 像素。
- **留白密度**：侧栏紧、对话中。
- **圆角与阴影**：输入器保持 Codex 形式，消息状态使用浅灰卡片。
- **HTML**：`03-alt-agent-emphasis.html`。

## 4. 选定 + 原因

- **选定**：候选「Codex 原生密度」。
- **理由**：用户明确要求一比一复刻 Codex，只在同一对话流中加入 agent 身份与交棒逻辑。角色应通过头像、名称和行内状态表达，不应重新形成卡片化控制台。
- **选定文件**：`03-selected.html`。
