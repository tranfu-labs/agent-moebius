# 一、显示信息 — 本地多 agent 操作台

> 目的：锁定 Codex 骨架下真正需要显示的信息，让多 agent 能力进入对话而不是另造控制台。

## 1. 用户画像

- **本地项目操作者**：选择项目，用自然语言发起任务，并持续阅读多个 agent 在同一任务中的协作过程。
- **开发者与验收者**：需要识别当前发言 agent、运行状态、交棒关系和需要自己介入的节点。
- **故障排查者**：偶尔需要日志与机器信息，但不应让诊断数据干扰日常对话。

## 2. 用户场景

### 主场景

- **发起任务**：选择项目或会话，在底部输入目标并提及一个角色，进入同一条多 agent 对话流。
- **跟进协作**：阅读不同 agent 的连续回复，识别运行中、交棒和等待自己确认的节点，并继续输入。

### 次场景

- **切换任务**：在项目下的平铺会话列表中选择任一会话，恢复对应时间线。
- **处理异常**：在时间线中看到失败或卡住的可读摘要，并按需打开日志。
- **选择工作区**：在发送前确认本地或隔离工作区和当前分支。

### 反场景

- 不把 SQLite 路径、内部编号、运行目录、工作目录或原始输出常驻展示在对话页。
- 不为每个 agent 建立并排面板、仪表盘或独立信息架构。
- 不把会话裂变关系渲染成树状导航、缩进子项或展开层级。
- 不复制 Codex 中 Moebius 尚未提供的定时任务、插件、站点或拉取请求入口。
- 不用顶部统计条汇总通过、运行中或等待数量。

## 3. 浏览路线

- **发起任务路线**：进入空会话 → 确认项目/工作区/分支 → 输入目标并选择 agent → 发送 → 进入对话态。
- **跟进协作路线**：进入已有会话 → 顺序阅读多 agent 消息 → 查看运行进展或交棒 → 等待时继续输入 → 完成。
- **切换任务路线**：浏览项目下的同级会话列表 → 选中目标会话 → 中央时间线切换 → 继续对话。
- **处理异常路线**：看到失败或卡住消息 → 阅读可读原因 → 按需打开日志或开发者诊断 → 返回对话。
- **选择工作区路线**：打开输入器上下文 → 选择本地或隔离 → 确认分支 → 发送任务。

## 4. 字段清单 + 优先级

| 字段 | 数据源 | 量级 | 优先级 | 服务于哪些场景 |
|---|---|---:|---|---|
| 当前项目名 | 项目摘要 | 单值 | 必看 | 发起任务、切换任务、选择工作区 |
| 会话标题 | 会话摘要 | 每会话一个 | 必看 | 跟进协作、切换任务 |
| 用户消息正文 | 消息记录 | 每轮一个 | 必看 | 发起任务、跟进协作 |
| agent 名称与头像 | 消息角色 | 每条 agent 消息一个 | 必看 | 跟进协作 |
| agent 回复正文 | 消息记录 | 每条 agent 消息一个 | 必看 | 跟进协作 |
| 输入内容与发送动作 | 本地编辑态 | 单值 | 必看 | 发起任务、跟进协作 |
| 工作区模式 | 项目工作区设置 | 单值 | 重要 | 选择工作区 |
| 当前分支 | 工作区事实 | 单值 | 重要 | 选择工作区 |
| 会话状态 | 会话摘要 | 每会话一个 | 重要 | 切换任务、跟进协作 |
| 运行摘要与中断动作 | 运行快照 | 活跃运行一个 | 重要 | 跟进协作 |
| 等待真人原因与动作 | 消息/运行事实 | 按需 | 重要 | 跟进协作 |
| 会话列表顺序 | 会话摘要 | 每项目多个 | 重要 | 切换任务 |
| 会话裂变来源 | 会话摘要 | 按需 | 留档 | 运行时编排与恢复，不参与页面导航 |
| 失败或卡住摘要 | 运行结果 | 按需 | 重要 | 处理异常 |
| 新会话、搜索、侧栏折叠 | 页面交互 | 固定 | 辅助 | 发起任务、切换任务 |
| 日志入口 | 运行结果 | 按需 | 辅助 | 处理异常 |
| 开发者诊断入口 | 桌面宿主 | 固定 | 辅助 | 处理异常 |
| 路径、内部编号、运行目录、原始输出 | 运行与状态库 | 多值 | 留档 | 故障排查 |

## 5. 对比/聚合单元

一条侧栏记录的粒度：
- 是：一个可恢复的会话，无论是否由其它会话裂变，均显示为同级标题和最小状态信号。
- 不是：一条 agent 消息、一个运行或一个统计指标。
