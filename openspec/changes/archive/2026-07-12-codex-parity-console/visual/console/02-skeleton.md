# 二、显示框架 — 本地多 agent 操作台

> 目的：把已锁定字段落入 Codex 式侧栏、对话画布和底部输入器，不在本阶段决定颜色或装饰细节。
> 配套：`02-skeleton.html`，使用模拟数据静态展示六类状态。

## 1. 区域划分

| 区域名 | 容纳字段 | 主要服务场景 |
|---|---|---|
| 窗口与侧栏区 | 窗口控制、侧栏折叠、搜索、新会话、项目名、会话标题、会话状态、会话列表顺序、开发者诊断入口 | 发起任务、切换任务、处理异常 |
| 对话画布区 | 用户消息、agent 名称与头像、agent 回复、运行摘要、等待真人、失败或卡住摘要、日志入口 | 跟进协作、处理异常 |
| 输入器上下文区 | 当前项目名、工作区模式、当前分支 | 发起任务、选择工作区 |
| 输入编辑区 | 输入内容、角色补全、发送动作、运行时中断动作 | 发起任务、跟进协作 |

对应关系自查：所有必看与重要字段均至少进入一个区域；路径、内部编号和原始输出保持留档，不进入主框架。

## 2. 状态机

- **空会话态**：侧栏选中空会话；对话画布只显示简短引导；输入器可用并显示项目、工作区与分支。
- **多 agent 对话态**：侧栏选中已有会话；对话画布按时间顺序混排用户、开发、测试和产品消息；输入器可继续输入。
- **运行中态**：当前 agent 消息位置显示运行摘要、耗时和中断动作；输入器仍在原位但不可发送。
- **等待真人态**：时间线显示等待原因和建议动作；侧栏会话行出现最小等待信号；输入器恢复可用。
- **失败与卡住态**：时间线分别显示失败与卡住的可读摘要和日志入口，不直接展示原始机器数据。
- **裂变会话列表态**：项目下包含原会话和裂变产生的新会话；所有条目保持同级、相同缩进和相同选择模型，只切换中央单一时间线。

## 3. 区域间关系

- 侧栏选择项目或会话 → 对话画布和输入器上下文同步切换。
- 选择任一同级会话 → 选中态更新，但对话区结构不改变；裂变来源不参与导航呈现。
- 输入器上下文选择工作区 → 后续运行的工作目录来源改变；不新增页面区域。
- 输入并选择 agent → 消息进入同一时间线；agent 身份显示在对应回复上。
- 运行开始 → 时间线插入运行中块、发送动作禁用；运行结束或中断 → 转为普通结果消息并恢复输入。
- 失败或卡住消息的日志入口 → 打开开发者诊断；返回后仍停留在当前会话。

## 4. 浏览路线落到区域

- **发起任务路线**：窗口与侧栏区选项目/会话 → 输入器上下文区确认环境 → 输入编辑区发消息 → 对话画布区进入运行中态。
- **跟进协作路线**：对话画布区顺序阅读不同 agent → 等待真人态出现 → 输入编辑区继续 → 新 agent 回复仍进入同一画布。
- **切换任务路线**：窗口与侧栏区浏览同级会话列表 → 选择目标会话 → 对话画布区刷新对应时间线。
- **处理异常路线**：对话画布区看到失败或卡住 → 打开日志 → 开发者诊断 → 返回当前会话。
- **选择工作区路线**：输入器上下文区选择本地或隔离 → 确认分支 → 输入编辑区发送。
