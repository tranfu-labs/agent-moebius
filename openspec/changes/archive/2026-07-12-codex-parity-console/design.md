# 设计：codex-parity-console

## 方案
1. 先以 `visual/console/01-display-info.md` 锁定主场景、反场景和字段优先级，再以 `02-skeleton.md` 与 `02-skeleton.html` 锁定六类页面状态。用户确认线框后才进入风格和实现。
2. `OperatorConsole` 收敛为 Codex 式两栏窗口：左侧负责全局入口、项目和会话层级；右侧是无顶部工具栏的连续对话画布；输入器固定在内容底部并携带项目、工作区模式和分支上下文。
3. 多 agent 只改变时间线消息的身份、状态和交棒提示，不改变“一个会话一条连续时间线”的阅读模型。会话裂变仍由运行时保存来源关系，但侧栏只显示同级会话列表，不缩进、不画树线、不提供层级展开。
4. worktree 功能保持现有运行时语义，但从常驻开关改为输入器上下文选择器。项目路径、SQLite、会话/运行编号、runDir、cwd、原始输出统一由开发者诊断承载；错误消息主层只保留可读摘要和日志入口。
5. Electron 使用与 Codex 相同方向的集成标题栏，并为 macOS 窗口控制、侧栏拖拽区和跨平台回退建立显式宿主契约。

## 权衡
- 选择一比一复刻 Codex 的骨架，而不是在现有三段工具栏上微调，是因为主要偏差来自信息架构和空间分配；继续保留现有结构无法通过字号或配色修复。
- 不复制 Codex 当前不属于 Moebius 的“定时任务、插件、站点、拉取请求”等入口，避免制造无功能导航；只复刻其结构和交互位置。
- 保留 worktree 能力但降低视觉层级，因为它是安全能力而不是用户每轮必须操作的主任务。
- 原始机器数据不删除，只从对话主界面下沉到诊断，兼顾产品清晰度与故障排查。

## 风险
- 集成标题栏在 macOS、Windows、Linux 的安全拖拽区和窗口按钮行为不同，需要宿主级回归验证。
- 现有测试可能依赖旧标题、按钮或状态标签，需要以可访问名称和行为契约更新，避免只按像素测试。
- 一比一布局在窄窗口中可能压缩输入器和侧栏，需要定义最小窗口宽度及侧栏折叠策略。
- 回滚可恢复旧 `OperatorConsole` 组合和 BrowserWindow 配置；运行时数据结构不迁移。
