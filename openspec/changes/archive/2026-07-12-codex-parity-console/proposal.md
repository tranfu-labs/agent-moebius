# 提案：codex-parity-console

## 背景
当前 Electron 本地操作台把项目状态、会话统计、诊断入口、工作区开关和原始机器信息同时抬到首屏，形成宽侧栏、双层顶部工具栏、居中表单卡片和大量边框容器。它与 Codex 桌面端“紧凑侧栏、单一内容画布、底部上下文输入器”的信息架构和视觉密度差异明显，也让 Moebius 真正需要突出的多 agent 对话被控制台信息淹没。

## 提案
- 以用户提供的 Codex 桌面端截图作为窗口骨架、布局比例、留白、控件密度和视觉语言的直接基线，并保留中文界面。
- 改为集成式 macOS 标题栏；主界面仅保留项目/会话侧栏、同流多 agent 对话画布和底部上下文输入器。
- 把 agent 身份、交棒、运行中、等待真人、失败/卡住等状态作为同一时间线中的消息语义呈现，不新增并排 agent 面板；裂变产生的新会话仍以同级列表呈现，不展示树状关系。
- 把工作区模式放入输入器上下文行，交互位置与 Codex 的 Local/分支上下文一致；把 SQLite 路径、内部编号、runDir、原始输出等机器信息移出主界面，统一下沉到开发者诊断。
- 覆盖空会话、普通多 agent 对话、运行中、等待真人、失败/卡住、裂变会话列表六类完整状态。

## 影响
- 影响 `packages/console-ui` 的操作台复合组件、令牌、侧栏、消息、运行块、空状态和输入器呈现。
- 影响 Electron BrowserWindow 标题栏配置、拖拽区域和诊断入口组织。
- 影响 `console-ui`、`desktop-shell` 的行为规格与 `docs/wireframes/pages/console.md` 版式事实源。
- 不改变本地会话持久化、agent mention、编排、运行中断、worktree 创建或诊断数据本身。
