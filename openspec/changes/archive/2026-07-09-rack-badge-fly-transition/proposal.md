# 提案：rack-badge-fly-transition

## 背景
底部工牌架当前的点击详情弹窗是「整个弹窗面板 rotateY 34°→0° 翻转入场」，工牌与弹窗之间没有视觉连续性。用户最终口径（两轮收敛）：要 **iOS 打开 App 式的容器变形**——就像点一个 App 图标、那个圆角矩形慢慢变成应用界面一样，点击一张侧立工牌，**这张工牌慢慢变成详情弹窗本身**（不是工牌飞到弹窗里的工牌位）。已确认：关闭要逆向动画（弹窗缩回工牌）；不引入 motion 等第三方动画框架（原生 WAAPI）。

## 提案
只改 `sites/marketeam/index.html`，把弹窗开合改为**工牌 → 弹窗的容器变形转场**：

1. **打开**：点击/Enter 激活某张侧立工牌 → 架上原卡隐位（占位保留）→ 两层同路径运动交叉淡化：
   - **幽灵卡**（克隆该卡侧立卡面，fixed 图层）：从架上投影位置（rotateY 44°）一边转正、一边沿路径向弹窗中心放大，前段实、中段淡出；
   - **弹窗面板**（真实 sheet，整体含内容）：从工牌足迹（非等比压缩到卡的投影矩形）放大长到最终矩形，前段透明、中段淡入。
   两者交叉即「工牌慢慢变成弹窗」。遮罩正常淡入。
2. **关闭**：Esc / 遮罩 / 关闭按钮 → 逆向：弹窗面板缩回工牌足迹并淡出，幽灵卡后段淡入、转回侧立 44° 落回架上原位 → 弹窗隐藏、架上原卡恢复、焦点还给触发工牌。
3. **降级与防错**：`prefers-reduced-motion` 或无 WAAPI 时走现行为（直接开/关，仅淡入淡出）；转场进行中防重入（连点/连按 Esc 不叠加动画）。

## 影响
- 只改 `sites/marketeam/index.html`（工牌架弹窗一节的 CSS 与 JS 编排）；工牌架布局、hover 行为（只抬起不转正）、弹窗内容结构均不动。
- 修改 `marketing-site` 域 spec：MS.12 增强为容器变形转场契约，新增降级/防重入场景。
- 归档时回流 `docs/wireframes/flow.md` 与 `docs/wireframes/pages/marketeam-landing.md` 中「rotateY≈34°→0 翻转入场」的动效注記（版式字符图本身不变，无需本 change 建 wireframes.md）。
- 零第三方依赖，保持单文件自包含。

## 已知取舍
- **sheet 入场不得再含位移/缩放/旋转类 CSS 动画**：变形的位移与缩放全部由 WAAPI 承担，CSS 侧只留纯 opacity 淡入——否则变形终点坐标量不准。
- **变形起点是非等比压缩的面板**：面板从卡片投影矩形（约 0.2×0.6 的非等比缩放）长出，起始几帧内容是压扁的，但此时面板透明度接近 0、且有幽灵卡盖在同路径上，实际不可见；不为此做圆角/内容的逐属性变形。
- **弹窗打开期间页面仍可滚动**：关闭时重新测量架上原位坐标，滚动后逆向变形仍落准；不加滚动锁。
