# 设计：rack-badge-fly-transition

## 总体思路：容器变形（container transform）= 双层同路径交叉淡化
iOS 打开 App 的观感由两个事实构成：**容器矩形连续变化**（图标框 → 应用窗口）与**内容交叉淡化**（图标淡出、应用界面淡入），两者共享同一条运动路径。对应到本页：

- **容器/内容层 A（幽灵卡）**：克隆被点击卡的侧立卡面（`.rack-face`），放进独立 fixed 图层，从架上位置（rotateY 44°、架上缩放）向弹窗中心运动，同时转正、放大；opacity 前 30% 保持 1、到 75% 处淡到 0。
- **容器/内容层 B（真实弹窗面板）**：`.rack-modal-sheet` 本体做 FLIP——起始 transform 把它非等比压进卡的投影矩形（`translate(Δ) scale(sx, sy)`），终点 `transform: none`；opacity 前 50% 从 0 线性淡入到 1。
- 两层动画同 duration（560ms），中段交叉即「工牌慢慢变成弹窗」。关闭是同一函数的关键帧逆序（offset 取 1-x 后反转数组）。
- **每个元素 transform 与 opacity 拆成两条独立动画**（共四条）：运动用弹性缓动 `cubic-bezier(.22,1,.36,1)`、淡化用线性缓动——缓动诉求不同是其一；其二，混写稀疏关键帧（中间帧只有 opacity）时，options 里的 easing 按 spec 逐段生效，会把交叉淡化压得过快，且实测在部分渲染环境（如预览面板的隐藏页）会被错误解析成全程插值。

跨越「rack 的 3D 场景」与「fixed 弹窗」两个布局上下文，同一节点无法连续过渡，所以必须有克隆层；但**面板层直接动真节点**（FLIP），不克隆弹窗内容——落定帧即真实 DOM，零换装痕迹。

不引第三方动画库（已与用户确认）：页面是自包含单文件（spec 硬约束），四条 WAAPI 动画即可覆盖，motion 类库在此省不了代码。

## 关键技术点

### 1. 侧立卡的投影盒子怎么用
`rotateY(44°)` 的卡 `getBoundingClientRect()` 是投影后的歪盒子：
- **中心点可直接用**（rotateY 不移动中心）——幽灵定位、面板位移差都以中心点计算。
- **宽度不可直接用**（被压缩）——幽灵起始缩放取「未旋转宽 `offsetWidth` × rack 累积缩放」；rack 累积缩放（断点下 `.rack-inner` 有 `scale(0.85/0.72/0.56)`）= `rack-inner 投影宽 ÷ offsetWidth`。
- **面板的起始压缩矩形用投影宽高**（`sx = 投影宽/面板宽`，`sy = 投影高/面板高`）——面板要「从卡片在屏幕上占的那块地方长出来」，用的就是投影足迹。

### 2. 终点测量的前提：CSS 入场动画不得影响布局
`rmIn` 保持纯 opacity 淡入（上一版已改），面板 `getBoundingClientRect()` 量到的即最终矩形；WAAPI 的 opacity 关键帧运行期间会压过 CSS 动画，互不打架。面板不设 `transform-origin`（默认中心），FLIP 的中心差位移才成立。

### 3. 幽灵图层
`.fly-layer`：`position:fixed; inset:0; z-index` 高于弹窗、`pointer-events:none`、`perspective:1400px`（幽灵的 rotateY 需要透视）。幽灵 = `.rack-face` 克隆，锁定自然宽高、带上该卡 `--glow`，用完即弃。姿态统一写成 `translate(x,y) translate(-50%,-50%) rotateY(θ) scale(s)`。

### 4. 状态编排（全在现有 rack-modal IIFE 内改造）
- 打开：填充数据 → 显示弹窗（此刻可量终点）→ 原卡加 `.is-away`（隐位占位）→ 双层动画 → 完成后拆幽灵层、focus 关闭钮、挂 Esc。
- 关闭：卸 Esc → `.is-closing`（遮罩淡出）→ 重新测量架上原位（应对弹窗期间页面滚动）→ 双层逆向 → 完成后 `hidden`、清类名、原卡恢复、焦点还给触发卡。
- **防重入**：模块级 `animating` 布尔量，变形期间忽略 open/close 请求。
- **降级**：`prefers-reduced-motion` 或无 `Element.animate` → 走现行为（即时开关 + 纯淡入），不建幽灵、不做 FLIP。
- 动画用 `fill:'both'`，完成回调里先落定状态（拆层/隐藏弹窗）再 `cancel()` 释放填充，避免残留的 filled animation 干扰下一轮开合。

## 权衡与风险
- **面板起始非等比压缩**：起始几帧内容压扁，但 opacity≈0 且幽灵卡盖在同路径上，不可见；换取「真节点 FLIP、落定零换装」。不做圆角/布局逐属性变形（那需要克隆整个面板再换真身，复杂度不成比例）。
- **幽灵只在前半程可见**：它的职责是把「这就是你点的那张卡」的连续感带进变形；后半程主角是已淡入的真面板。幽灵终点缩放只需大致贴合面板高度，精确度不敏感。
- **`.rack` 的 mask 渐隐**不影响幽灵（独立 fixed 层）；起飞瞬间原卡若处于 mask 渐隐区，亮度有细微差，可接受。
- `closeBtn.focus()` 挪到变形完成后，避免运动中焦点环闪现。

## 测试口径
纯展示动效（DOM/视觉编排，无纯函数、无数据转换），按测试门槛豁免单测，理由：全部逻辑是「量坐标 → 播动画 → 切类名」的浏览器副作用，无可剥离的纯逻辑。走 AI 验证流程（见 tasks.md 末条）。
