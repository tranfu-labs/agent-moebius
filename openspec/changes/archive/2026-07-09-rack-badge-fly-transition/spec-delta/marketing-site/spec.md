# marketing-site spec delta：rack-badge-fly-transition

把工牌架详情弹窗的开合从「弹窗整体 3D 翻转入场」升级为「工牌 → 弹窗的容器变形转场」（iOS 打开 App 式）。

## 修改（归档时生效）

### 「员工工牌与工牌架」规则第 3 条（点击/激活）修改为
- 工牌架点击/激活 MUST 打开详情并**此时才显示工牌正面**：被点击的工牌 MUST 以**容器变形转场**慢慢变成详情弹窗——工牌卡面从架上侧立（≈44°）位置一边转正放大一边淡出，弹窗面板从该工牌的屏幕足迹处同步放大淡入，两者共享同一条运动路径；弹窗容器自身 MUST NOT 再做独立的 3D 翻转入场。
- 关闭（Esc / 遮罩 / 关闭按钮）MUST 逆向动画：弹窗面板缩回工牌足迹并淡出、工牌卡面转回侧立落回架上原位，架上原卡恢复、焦点还给触发工牌；弹窗打开期间页面发生滚动后逆向变形 MUST 仍落准架上当前位置。
- 转场 MUST 用原生 Web Animations API 实现，MUST NOT 引入第三方动画库；转场进行中 MUST 防重入（重复点击/按键不叠加动画）。
- MUST 在 `prefers-reduced-motion`（或无 WAAPI）下降级为直接开合 + 淡入淡出，无变形无旋转。

## 场景修改

### 场景 MS.12（替换）：点击工牌容器变形为详情弹窗
Given 底部工牌架
When 用户点击（或键盘激活）某张侧立工牌
Then 该工牌以容器变形转场慢慢变成详情弹窗（卡面转正放大淡出、面板从工牌足迹放大淡入）
And 架上原卡在转场期间隐位（占位保留）
And 落定后弹窗展示正面放大工牌、角色名、charter 与 `@<role>` 在 issue 中的用法
When 用户通过 Esc / 遮罩 / 关闭按钮关闭
Then 弹窗逆向缩回该工牌在架上的当前位置，架上原卡恢复，焦点还给触发工牌

## 新增场景

### 场景 MS.13：容器变形可降级且防重入
Given 用户开启了 prefers-reduced-motion（或浏览器无 WAAPI）
When 点击工牌打开或关闭详情弹窗
Then 弹窗直接开合（仅淡入淡出），无变形无旋转
Given 容器变形转场进行中
When 用户重复点击工牌或连按 Esc
Then 不叠加新动画、状态不错乱
