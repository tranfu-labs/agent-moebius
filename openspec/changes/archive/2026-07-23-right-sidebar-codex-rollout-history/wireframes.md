# Wireframes：right-sidebar-codex-rollout-history

基线：`docs/wireframes/pages/console.md`。页面产品事实由 `docs/product/pages/main-right-sidebar.md` 接管；本文件只记录本 change 的版式变化，归档时回流该页面 PRD。

## pages/main-right-sidebar.md

### 首次打开 · 位于最新

```text
│ 改动 × │ 开发 × │ ＋                                │
│ ─────────────────────────────────────────────────── │
│ 开发 · 这一步的完整输出                  跟随最新   │
│ ─────────────────────────────────────────────────── │
│             ↑ 向上滚动加载更早过程                  │
│ ── 第 1 次执行 · 本轮输入 ──────────────────────── │
│ 你                                                  │
│ 请检查落地页空状态。                                │
│                                                    │
│ 开发经理                                            │
│ 先核对现有实现和验收口径。@dev                      │
│ ── 本轮执行过程 ────────────────────────────────── │
│ 开发                                                │
│ 我先检查相关实现和测试。                            │
│                                                    │
│ ⌘ 读取文件                                          │
│ sites/marketeam/index.html                          │
│                                                    │
│ ⌘ 运行命令                                          │
│ pnpm test -- landing                                │
│ PASS tests/landing.test.ts                          │
└────────────────────────────────────────────────────┘
```

初次 ready 后视口锚在最后一条，不从第一条开始。顶部 sentinel 只在用户向上时加载更早页。

### 离底阅读 · 有新内容

```text
│ 开发 · 这一步的完整输出                 已暂停跟随  │
│ ─────────────────────────────────────────────────── │
│                                                    │
│ ⌘ 工具调用                                          │
│ 检查测试配置                                       │
│                                                    │
│ 开发                                                │
│ 这里还有一处边界需要核对。                          │
│                                                    │
│                              [↓ 3 条新内容 / 到最新] │
└────────────────────────────────────────────────────┘
```

新事件不改变当前阅读位置；按钮浮在内容之上但不遮住最后一个可见事件。点击后回到最新并恢复跟随。

### 多次执行

```text
│ ── 第 1 次执行 · 本轮输入 ──────────────────────── │
│ 你 / 开发经理 / 开发 的公开时间线…                  │
│ ── 本轮执行过程 ────────────────────────────────── │
│ ⌘ pnpm test                                         │
│ FAIL                                                │
│                                                    │
│ ── 第 2 次执行 · 本轮输入 ──────────────────────── │
│ …包含第一次失败后新增的公开消息                     │
│ ── 本轮执行过程 ────────────────────────────────── │
│ ⌘ pnpm test                                         │
│ PASS                                                │
```

每次执行保留自己的输入快照；不能只在标签顶部放一份共享输入。

### Codex 记录不可用

```text
│ 开发 · 这一步的完整输出                             │
│ ─────────────────────────────────────────────────── │
│                                                    │
│            Codex 过程记录文件已不可用               │
│      这一步的最终回复仍保留在主对话区。              │
│                                                    │
└────────────────────────────────────────────────────┘
```

不显示 stderr/stdout tail、最终 Agent 回复或猜测匹配的其他 rollout。
