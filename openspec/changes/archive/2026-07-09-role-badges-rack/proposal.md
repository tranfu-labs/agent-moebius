# 提案：role-badges-rack

## 背景
落地页现在的角色表示是「圆角矩形/圆形色卡 + 缩写」。用户要往**员工工牌（ID 工牌）**方向升级，并把首屏底部的技术栈条改成一排**工牌斜插在架子里**、可 hover 抬起、点击弹详情的效果——动效借自黑胶唱片架，但**主体是工牌，不是黑胶碟**。

## 提案
两块改动，全部纯 CSS/JS 自包含、不引第三方、不动正文文案事实：

1. **角色卡 → 员工工牌**：把角色的视觉从色卡升级为「公司员工证」观感（打孔/卡槽、头像区、姓名=角色名、职位=charter、公司标 moebius、条形码 + 角色 ID）。应用到 ③「Your AI team」网格与首屏圆环节点（圆环里做**简化版**工牌，避免旋转中过重）。**采用「加满/全息」档**：hover 时轻 3D 倾斜 + 光泽扫过 + 随鼠标流动的全息 foil（`mix-blend-mode:color-dodge`），倾斜角压小（±6°）不破坏气质。

2. **底部技术栈条 → 工牌架（借黑胶唱片架动效）**：把上一轮的技术栈滚动条替换为一排**侧立斜插的工牌**（perspective + rotateY 侧立、负 margin 叠放）。交互严格按用户口径：
   - **hover：只做垂直位移（抬起），不转正、不改角度**（工牌仍侧立，仅上移 + 轻微提亮）。
   - **点击：展开详情弹窗，此时才显示工牌正面**（放大正面工牌 + 角色名/charter/@mention 用法），带全息。
   - 尊重 `prefers-reduced-motion` 降级。

## 影响
- 只改 `sites/marketeam/index.html`（角色卡样式 + 首屏圆环节点 + 底部区 + 新增工牌架脚本与详情弹窗）。正文①②④文案不动。
- 修改 `marketing-site` 域 spec：**退役上一轮「底部技术栈滚动条」规则**，代之以「底部角色工牌架 + 侧立/hover 抬起/点击弹正面」交互契约；角色视觉规格从「色卡」改为「员工工牌（全息档）」。
- 回流 `docs/wireframes/pages/marketeam-landing.md` 与 `flow.md`。
- 复用暗紫基调、hairline、hover tooltip/弹窗组件；不引第三方图片/库。
