# marketing-site spec delta：rack-badge-spring-motion

转场运动引擎从贝塞尔改为弹簧曲线，补时长与落定手感契约。

## 修改（归档时生效）

### 「员工工牌与工牌架」转场规则第 1 条中的速度契约句修改为
- **转场 MUST 由弹簧曲线驱动**（欠阻尼弹簧采样烘焙进 CSS `linear()`，仍零第三方依赖）：位移 MUST 快攻到位并以微过冲（~2%）一次收束咬合，MUST NOT 出现贝塞尔式渐近死尾（落定前长段低速爬行）；全程水平速度 MUST NOT 中途归零（微过冲处的自然反向除外）；X/Y/变形面 MUST 由同一条弹簧统一驱动（旋转可用更快弹簧前置完成）。

### 「员工工牌与工牌架」新增时长规则
- 打开转场 MUST ≤ 500ms、关闭 MUST ≤ 300ms 且短于打开（业界 UI 转场时长准则）。

## 场景修改

### 场景 MS.12 的打开 Then 行修改为
Then 该工牌由弹簧驱动、沿带微过冲的连续轨迹快速飞抵弹窗内工牌位并咬合落定，转正在前半程完成，途中由简化卡面渐变为完整工牌
