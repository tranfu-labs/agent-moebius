# 提案：rack-badge-spring-motion

## 背景
弧线化修掉了顿挫（速度剖面已是单峰连续），但用户仍不满意。停手做了业界调研（英文检索），结论：

- 「卡片展开成详情」场景的公认标杆是 Framer Motion `layoutId` 共享布局（iOS App Store 的 web 复刻），其手感核心是**弹簧物理 + 一条弹簧统一驱动整场变形**，不是更讲究的贝塞尔。
- 贝塞尔的结构性缺陷是**渐近死尾**：上一版落定前 ~200ms 都在以 <5% 峰值的速度爬行，读作拖泥带水；弹簧带能量到达、以微过冲一次咬合收束。
- 业界时长准则（Emil Kowalski / Linear）：UI 转场 ≤300ms，大型 morph 配弹簧 400–500ms；上一版 600ms 属于偏沉重。
- 我给 X/Y/旋转配了三种缓动人格，物体不像被一只手移动的刚体；标杆做法是一条物理进度驱动全部分量。
- 零依赖落地路径已成熟：CSS `linear()` 缓动可把真实弹簧采样点烘焙成曲线字符串，直接喂 WAAPI（Josh Comeau / Jake Archibald 的实践）。

## 提案
只改 `sites/marketeam/index.html`，三层架构与像素级换装保留，**运动引擎整体换弹簧**：

1. **自算弹簧、烘焙 `linear()` 常量**（欠阻尼弹簧解析解采样 48 点，零依赖）：
   - 打开主弹簧（k=380, c=30, ζ≈0.77）：460ms 收束，~195ms 即到达目标、微过冲 2.3% 后咬合——快攻无死尾；
   - 旋转快弹簧（k=807, c=51, ζ≈0.9）：同窗口内 ~50% 处收平，转正前置完成（拿起读感）；
   - 关闭弹簧：280ms 收束、无过冲。
2. **一条弹簧统一驱动** X / Y / 旋转缩放 / 变形面（旋转用快弹簧前置属于唯一例外）：删除手拼抛物线 apex 关键帧——弹簧对 Y/X 的天然微过冲（越过卡位再收回）自带弧感与落定咬合。
3. **时长砍半**：打开 460ms、关闭 280ms；淡化窗口相应压前压短。
4. 关闭时旋转给 ~30% 延迟再用短时长转回（离窗时保持正面、临近插槽才侧立；起步自静止，无中途刹停）。

## 影响
- 修改 `marketing-site` 域 spec：运动契约补「弹簧驱动、落定微过冲收束、MUST NOT 渐近死尾」与时长上限。
- wireframes 注記微调（弧线 → 弹簧）。
- 零第三方依赖不变（曲线是烘焙常量）。

## 已知取舍
- `linear()` 是弹簧的定格快照，不能像 Framer 那样中断续速（重定向动画时从零速起步）——本场景开/关有防重入闸门，中断续速不在需求内。
- 微过冲会让 X/Y 越过目标 ~2%（几 px）再回——这是刻意的「咬合」手感，非误差；落定帧仍与真工牌像素对齐（曲线终值=1）。
