基线：`docs/product/pages/main-conversation.md#页面结构`（团队推进中、一步没跑起来）与 `#页面状态`。该页面 PRD 已接管主内容区版式，本文件只记录本次 change 相对基线**新增或改动**的部分；归档时回流到该页面 PRD，不写入 `docs/wireframes/`。

## pages/main-conversation.md

### 团队推进中（沿用基线，确认没有过程标记与计时）

```text
│ 官网落地页验收                                              │
│ ─────────────────────────────────────────────────────────── │
│ 你                                                          │
│ 帮我把落地页的空状态补上验收语句                            │
│                                                             │
│ (经) 开发经理                                               │
│ 这个改动只碰落地页，@开发 来做。                            │
│                                                             │
│ (开) 开发                                                   │
│ 正在补空状态验收语句…                                       │
│ │ 读取 sites/marketeam/index.html                           │
│ │ 运行测试                                                  │
│ │                                                [停下]     │
│ ─────────────────────────────────────────────────────────── │
```

每条步骤行只有它在做什么，右侧**没有**「已完成 / 进行中 / 未开始」标签，**没有**耗时。操作条只挂在最后一条正在工作的记录末尾，这一步结束后整条消失，历史里不留痕迹。

「开发经理说完 @开发」本身就是交棒，不再额外渲染「已交棒」。

### 四种事实

```text
一步没跑起来
│ (开) 开发                                                   │
│ 正在补空状态验收语句…                                       │
│                                                             │
│ ⚠ 这一步没跑起来（退出码 1）                     [重试]     │
│ ─────────────────────────────────────────────────────────── │

一步卡住了
│ ⚠ 这一步卡住了                                   [重试]     │
│ ─────────────────────────────────────────────────────────── │

你按了停（中性，不触发红点）
│ ⏹ 你让这一步停下了                                          │
│ ─────────────────────────────────────────────────────────── │

反复重试仍未成功（触发红点，不给重试）
│ ⚠ 这一步反复没跑起来，已经不再重试                          │
│   你可以说点什么，或换一个成员接手                          │
│ ─────────────────────────────────────────────────────────── │
```

四条彼此可区分，且都由落库的事件类型驱动，因此刷新或重启后仍在。

最后一条不给「重试」按钮是刻意的：产品已经替用户试过多次，再给一个按钮等于把「你自己再点点看」的责任推回去。给出路比给重复动作诚实。

文案说「这一步没跑起来」，不说「开发失败了」——这四种全部来自机器故障或用户动作，不是团队协作的失败。

### 三种不可继续状态的共同表现

```text
项目文件夹不可用
│ 官网落地页验收                                              │
│ ─────────────────────────────────────────────────────────── │
│ （历史只读）                                                │
│ ─────────────────────────────────────────────────────────── │
│   ┌──────────────────────────────────────────────────────┐  │
│   │ [▱ 项目 A ⚠]  默认工作空间  main  [◇ 开发团队]       │  │
│   │ 项目文件夹找不到了，修复后才能继续      [修复…]      │  │
│   └──────────────────────────────────────────────────────┘  │

团队已删除
│   │ [▱ 项目 A]  默认工作空间  main  [◇ 开发团队 ⚠ ▾]     │  │
│   │ 这支团队已经被删除，改选一支团队后可以继续           │  │

团队需要修复
│   │ [▱ 项目 A]  默认工作空间  main  [◇ 开发团队 ⚠ ▾]     │  │
│   │ 这支团队需要修复            [去修复]                 │  │
```

三种共用同一套骨架：历史只读、输入框被说明文案替换、对应的上下文按钮标红。区别只在说明文案和恢复动作。

「团队已删除」的恢复动作是团队按钮本身的下拉（改选一支），不额外给按钮——恢复动作就在标红的那个控件上，是最短路径。

「团队需要修复」修完之后自动恢复，用户不需要回到这里再操作一次。

### 侧边栏状态点的来源

```text
│ ▾ 项目 A       │
│    落地页验收 ● │  ← 红：这段对话里有没跑起来 / 卡住 / 反复重试仍未成功
│    构建脚本   ● │  ← 蓝：没人在工作，且最后一条没提及任何人
│    文案打磨   ◌ │  ← 闪：有成员正在工作
│    需求梳理     │  ← 无点：最后一条提及了某个成员，团队内部仍在传递
```

用户按下「停下」之后是无点，不是红点。正常完成也是无点。
