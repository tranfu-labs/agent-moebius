# 设计：rack-badge-morph-continuity

## 三层结构与层序（关键修正）
上一版把所有转场元素都放在 z-index 240 的 fly-layer（弹窗之上），导致「容器视觉」只能盖住真面板。本版层序自下而上：

```
遮罩 overlay（modal 内，最底）
  → 变形面 morph-surface（modal 内、插在 overlay 与 sheet 之间的 DOM 位置）
    → 真面板 sheet（内容淡入层）
      → 幽灵卡 fly-layer > morph-ghost（fixed，z 240，最上）
```

- 变形面在真面板**下方**：面板内容淡入时，底下始终有连续变化的容器矩形托着；两者底色相同（rgba(14,8,32,.97)），叠放期视觉无缝。
- 幽灵卡在最上：它是眼睛追踪的主体，任何阶段不被遮挡。

## 幽灵卡：双内容共容 + 像素级换装
- 容器 `morph-ghost`：尺寸 = 弹窗真工牌（`#rm-badge`）的实测矩形；内叠两个子层：
  - **子层 A**：`.rack-face` 克隆（自然 150×206），按宽度对齐缩放（×196/150）后垂直居中——数学上保证起飞帧与原卡**像素一致**（渲染宽高 = 原卡投影前自然尺寸 × rack 累积缩放）。
  - **子层 B**：`#rm-badge` 克隆（自然尺寸即容器尺寸），剥除子孙 id、带上 `--glow`。
- 容器动画为**两拍编排**（用户点名要保住黑胶唱片架「拿起转正」的优雅，一拍直线飞行会把它冲掉）：
  - **第一拍「拿起」（0–38%）**：原地上抬（-28px）、前移（translateZ 110px，透视下产生"凑近"感）、rotateY 44°→0° 转正、微放大（×1.05）、亮度微升——即原 demo 的 hover 抬起动作；此拍变形面按兵不动。
  - **第二拍「入窗」（38–100%）**：从举起姿态飞抵真工牌位落定；变形面同拍从（举起位的）足迹长成面板。
  - 关键帧姿态统一为 `translate/translateZ/rotateX/rotateY/scale` 同构列表（避免结构不一致触发矩阵分解插值）；分段缓动写在关键帧上（拿起=demo 的 `cubic-bezier(.22,1,.36,1)`，入窗=emphasized `cubic-bezier(.2,0,0,1)`）。
- 内容交叉淡化：A 1→0、B 0→1，压在运动的 20%–55% 段内完成（linear）。
- **头像收敛**：A/B 两份内容的头像块纵向差 ~40px，最亮元素交叉时双重曝光明显；A 在自己的淡出窗口内同步向 B 的头像位滑动（实测差值动画），两个头像在透明度交叉点恰好重合。关闭反向。
- **容器全程 opacity 1**：落定帧子层 B 与真工牌逐像素相同，`done()` 里同帧「真工牌解除隐藏 + 拆幽灵层」，换装不可见。
- 真工牌隐藏用状态类 `.rack-modal.is-morphing .badge { visibility: hidden }`（打开全程 + 关闭全程）。

## 变形面
- 无内容 div，面板同款圆角/底色/描边/投影；inline 定位到面板最终矩形；transform 从
  `translate(足迹中心差) scale(投影宽/面板宽, 投影高/面板高)` → `none`。
- 空面无内容可拉伸；描边与圆角的早期形变发生在低透明度 + 幽灵覆盖阶段，不可见。
- 淡入 0→30%（打开）；关闭镜像为 65%→100% 淡出。落定后 `done()` 同帧移除（真面板已 opacity 1 盖在其上）。

## 真面板
- 不再做任何 transform（终点测量因此也稳定）；WAAPI 只动 opacity：打开 45%→85% 淡入、关闭 0→25% 淡出。
- CSS `rmIn` 纯淡入保留，仅服务降级路径；morph 路径下被脚本动画压过。

## 节奏
| 项 | 打开（760ms：拿起 0–38% / 入窗 38–100%） | 关闭（480ms：离窗缩回 0–55% / 放回 55–100%） |
|---|---|---|
| 幽灵位移/旋转 | 拿起（settle 缓动）→ 入窗（emphasized 缓动） | 离窗（accelerate）→ 放回（settle 缓动） |
| 幽灵内容 A↔B | A→B，42–70%（入窗拍内） | B→A，30–58%（放回前完成） |
| 头像收敛滑动 | 同 A↔B 窗口 | 同 B→A 窗口 |
| 幽灵亮度 | 起（hover 则 1.08）→38% 处 1.08 →75% 处回 1 | 1 →55% 处 1.08 →100% 回 1 |
| 变形面 transform | 0–38% 停在举起位足迹，38–100% 长成面板 | 0–55% 缩回举起位足迹，之后保持 |
| 变形面 opacity | 0→1，38–55% | 1→0，35–55% |
| 真面板内容 opacity | 0→1，60–92% | 1→0，0–20% |
| 遮罩 | rmFade 0.25s（CSS） | is-closing → rmOut（CSS） |

> 缓动教训：初版用 Material 的 emphasized-decelerate `cubic-bezier(0.05,0.7,0.1,1)`，实测 15% 时间已走完约 70% 路程，长距离飞行读作「瞬移到位再收尾」。改用完整 emphasized `cubic-bezier(0.2,0,0,1)`（有加速起步），交叉淡化窗口右移进中速段。

原则：一切换装都发生在运动进行中；关闭比打开快。开/关关键帧不共用 flip() 反转（节奏不对称），各写各的。

## 起飞/降落帧对齐（消微跳）
- **透视**：fly-layer 的 `perspective` 与 `perspective-origin` 按 `.rack` 实测（1500px、原点=rack 矩形中心相对视口坐标）inline 设置，幽灵 44° 投影与原卡渲染一致。
- **mask 透明度**：`.rack` 两端 7% 渐隐；按卡中心在 rack 内的横向位置算出近似透明度 α，打开起帧从 α 渐至 1（0–15%），关闭末帧渐回 α（85–100%）。
- **hover 亮度**：打开时若 `item.matches(':hover')`，幽灵起帧带 `brightness(1.08)` 渐回 1（0–20%）。

## 复用与不变项
open/close 编排骨架、`animating` 防重入、`is-away` 占位隐藏、Esc/遮罩/按钮三路关闭、焦点管理、`prefers-reduced-motion`/无 WAAPI 降级、动画完成后 `cancel()` 释放填充——全部沿用上一版结构，仅 morph() 内部与状态类不同。

## 测试口径
纯展示动效，豁免单测（无可剥离纯逻辑），沿用 AI 验证流程并加码：
- 原有状态断言（开/关/防重入/焦点/清理）。
- **无双卡断言**：变形全程 `#rm-badge` computed visibility=hidden。
- **落点对齐断言**：把动画 seek 到 ~99%，幽灵容器 rect 与真工牌 rect 逐边差 ≤ 1px。
- **连续性目检**：playbackRate 调慢后按时间点连拍截图，逐帧确认单卡连续、无拉伸、无静止期换装。
