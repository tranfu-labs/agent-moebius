# 提案：rack-badge-morph-continuity

## 背景
上一轮（2026-07-09-rack-badge-fly-transition）实现了「工牌 → 弹窗」容器变形，但用户实测**中段有明显跳变、不丝滑**。诊断出四个结构性根因（按感知强度排序）：

1. **双卡同屏 + 落点错位**：幽灵卡飞向面板几何中心，而面板里的真工牌在左侧卡位随面板同步淡入——中段同时可见两张同样的卡，被追踪的那张"死在中途"、真身在别处"瞬移出生"。
2. **容器形状从不重合**：卡片（竖长）与面板（横长）轮廓在任何一帧都不重合，中段是两个不同形状物体的交叉淡化，而非"一个矩形连续变成另一个矩形"。
3. **面板内容被非等比拉伸**：面板从 `scale(0.23, 0.6)` 起步，文字/按钮压扁再回弹，读作"果冻"。
4. **节奏错位**：强 ease-out 让位移前 30% 时间走完约 85%，其后大半时间画面静止、只剩原地换装——静止期的内容切换最像"跳"。
   另有次要项：fly-layer 透视原点（视口中心）与 .rack 透视原点不一致导致起飞帧投影微跳；架子两端 mask 渐隐使边缘卡起飞瞬间变实；hover 亮度未带上；开关同速使关闭显拖。

用户已拍板**方向 B：标准容器变形**（Material container transform 的 web 实现），三层各司其职。

## 提案
只改 `sites/marketeam/index.html`，重构转场为三层：

1. **幽灵卡（共享元素）**：容器内叠两份内容——架上简化卡面与弹窗完整工牌（两者宽高比 0.728 vs 0.70 近乎一致，可无损共容）；动作为**两拍编排**（二轮用户反馈：一拍直线飞行冲掉了黑胶唱片架「拿起转正」的优雅）——第一拍原地**拿起**（上抬、前移、44°→0° 转正、微放大），第二拍从举起姿态**入窗**飞抵**弹窗内真工牌的位置与尺寸**，途中双内容交叉淡化（简化卡面 → 完整工牌）；**全程不整体淡出**，落定瞬间与真工牌像素对齐换装。眼睛追踪的物体从头到尾只有一份。
2. **变形面（容器连续）**：一块无内容的圆角面（面板同款底/描边/投影），插在遮罩与真面板之间的层序上，从卡片投影足迹连续插值到面板矩形——承担「一个矩形变成另一个矩形」的容器叙事；因无内容，缩放不产生可见拉伸。
3. **真面板（内容层）**：不再做 transform，全程停在最终位置；内容（文字、关闭按钮）在容器基本就位后（45%–85%）淡入；真工牌以状态类隐藏至落定换装。

节奏与手感：打开 620ms 用强减速缓动、关闭 420ms 用加速缓动（关闭快于打开）；所有交叉淡化压进运动进行段完成，不在静止期换装。起飞帧对齐：fly-layer 透视参数按 .rack 实际位置设置；边缘卡按 mask 透明度、hover 卡按亮度对齐起始帧。

## 影响
- 只改 `sites/marketeam/index.html`（工牌架弹窗转场一节的 CSS 与 JS）；工牌架布局、hover 行为、弹窗内容结构、降级与防重入契约均不变。
- 修改 `marketing-site` 域 spec：把上一轮的转场规则替换为带质量契约的版本（共享元素唯一且落点为真工牌位、容器连续、内容不拉伸、换装须在运动中完成、关闭快于打开）。
- wireframes 动效注記微调。
- 零第三方依赖不变。

## 已知取舍
- **变形面用 transform 非等比缩放**：面是空的，缩放只影响 1px 描边与圆角的早期形变，且该阶段面透明度低、幽灵卡盖在上方，不可见；不为此上 clip-path 方案（需要 translate+clip 联动的复杂坐标数学）。
- **幽灵双内容的高度差 ~4%**：简化卡面按宽度对齐居中放置，上下各留 ~5px；交叉淡化发生在运动中段，不可辨。
- **打开期间页面可滚动**（沿用）：关闭时重新测量架上原位。
