# 线框：agent-teams-runtime-binding

基线：`docs/product/pages/agent-teams.md`。该页面已建页面 PRD 并承载页面结构，因此基线取 PRD 而非 `docs/wireframes/`；归档时字符图回流至该 PRD 对应小节。

## pages/agent-teams.md

### 需要修复的团队横行

成员区改为占位说明。原线框在此位置显示 `[开发经理] [开发] [测试]`，而该状态下 `AGENT.md` 读不到，成员名单只能来自缓存——正是 PRD「团队横行」小节拒绝维护的那份摘要。

```text
┌────────────────────────────────┬───────────────────────────────┐
│ 我的开发团队          需要修复 │ 成员信息暂时无法读取          │
│ 团队文件夹已移动或不可访问     │ 暂时无法用于新对话            │
└────────────────────────────────┴───────────────────────────────┘
```

团队名称与一句话描述来自应用记录，用户据此识别这是哪一支团队；成员构成在修复后随实时读盘自然恢复。

### 外部修改冲突未解决时的离开阻止

当前实现里，存在未解决的外部冲突时点击返回不会有任何反应——按钮外观正常、可点击、点了没动静。改为明确说明拦截原因并列出待处理成员。

```text
┌────────────────────────────────────────────┐
│ 无法返回团队列表                           │
│                                            │
│ 以下 Agent 的文件在应用外被修改，          │
│ 需要先选择保留哪一份：                     │
│                                            │
│   · 开发经理 AGENT.md                      │
│   · 测试 AGENT.md                          │
│                                            │
│                              [ 知道了 ]    │
└────────────────────────────────────────────┘
```

关闭后停留在团队详情，冲突成员保持待处理标记。这与已有的「未保存草稿离开」三选项是两个不同弹层：草稿可以放弃，外部冲突必须先选版本，因此这里只有一个确认按钮，不提供「直接离开」。
