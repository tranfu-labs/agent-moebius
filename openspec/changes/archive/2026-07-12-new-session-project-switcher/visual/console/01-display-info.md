# 一、显示信息 — 桌面本地对话操作台

## 1. 用户画像
- **本地项目操作者**：同时打开多个代码目录，需要在创建会话时明确 Codex 将在哪个项目工作。
- **会话维护者**：会回看已有消息和运行证据，需要历史会话的项目归属稳定可信。

## 2. 用户场景

### 主场景
- **从项目新建**：操作者先识别目标项目，再从该项目行创建新会话。
- **发送前修正项目**：操作者在空白会话输入内容时发现项目不对，通过输入器文件夹菜单改到目标项目而不丢草稿。

### 次场景
- **确认历史归属**：操作者在已有消息的会话里查看当前项目、工作区和分支上下文。

### 反场景
- 不在已有消息或编排关系的会话中迁移项目。
- 不把项目管理、删除或重命名塞进本次项目菜单。
- 不把内部 project id、绝对路径或数据库信息常驻在默认表面。

## 3. 浏览路线
- **从项目新建路线**：进入操作台 → 扫描项目列表 → 点击目标项目行右侧新会话 → 进入该项目空白会话 → 输入并发送。
- **发送前修正项目路线**：进入空白会话 → 输入草稿 → 点击输入器文件夹 → 选择目标项目 → 确认项目名变化 → 发送。
- **确认历史归属路线**：打开已有会话 → 查看输入器上方固定项目名 → 继续输入或退出。

## 4. 字段清单 + 优先级

| 字段 | 数据源 | 量级 | 优先级 | 服务于哪些场景 |
|---|---|---:|---|---|
| 项目名称 | 本地项目摘要 | 每项目一项 | 必看 | 从项目新建、发送前修正、确认历史归属 |
| 项目专属新会话动作 | 受控回调 | 每项目一个 | 必看 | 从项目新建 |
| 当前项目标记 | selected project | 一个 | 必看 | 发送前修正 |
| 已打开项目选项 | 本地项目列表 | 通常一到十项 | 重要 | 发送前修正 |
| 会话是否可重绑 | 消息、运行与会话关系事实 | 一个布尔结果 | 重要 | 发送前修正、确认历史归属 |
| 工作区模式 | 当前项目摘要 | 一个 | 重要 | 确认历史归属 |
| 分支语义 | 当前项目摘要 | 一个 | 辅助 | 确认历史归属 |
| 项目内部 id 与绝对路径 | 内部状态 | 每项目一项 | 留档 | 不在默认表面展示 |

## 5. 对比/聚合单元
一条记录的粒度：
- 是：一个已打开项目及其项目名称、新会话动作和会话列表。
- 不是：一个会话或一个工作区状态。
