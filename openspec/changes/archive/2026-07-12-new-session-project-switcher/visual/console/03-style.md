# 三、风格 + 实践 — 桌面本地对话操作台

## 1. 页面性质判定
- **位置**：实用。
- **理由**：主场景是快速确认项目归属并开始工作，信息密度、可扫描性和低干扰高于装饰表达。

## 2. 组件清单

| 组件类型 | 出现于哪个区域 | 主要用途 | 变体 |
|---|---|---|---|
| 项目行 | 项目导航区 | 显示项目与承载新会话动作 | 默认、悬停 |
| 图标按钮 | 项目导航区 | 为该项目创建新会话 | 默认、悬停、焦点、禁用 |
| 项目触发器 | 输入器上下文区 | 展开项目菜单 | 默认、悬停、展开、锁定文本 |
| 项目菜单项 | 项目菜单区 | 选择目标项目 | 默认、悬停、当前选中、禁用 |
| 输入器 | 输入器区 | 保持并发送草稿 | 空白、已有草稿、禁用 |
| 错误事实文本 | 会话时间线区 | 告知重绑失败 | 默认 |

自查：显示框架全部区域和状态中的控件均已覆盖。

## 4. 选定 + 原因
- **选定**：候选「近单色 · 上下文菜单」。
- **理由**：项目切换是空白会话中的短暂动作，不应永久挤占输入器或侧栏空间；菜单也能沿用现有可访问原语的键盘与焦点语义。内联列表在多项目时会抬高输入器，削弱时间线空间。
- **选定文件**：`03-selected.html`。
