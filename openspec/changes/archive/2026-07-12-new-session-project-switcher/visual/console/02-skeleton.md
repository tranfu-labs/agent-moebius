# 二、显示框架 — 桌面本地对话操作台

## 1. 区域划分

| 区域名 | 容纳字段 | 主要服务场景 |
|---|---|---|
| 项目导航区 | 项目名称、项目专属新会话动作 | 从项目新建 |
| 会话时间线区 | 当前会话内容或空白邀请 | 三个场景 |
| 输入器上下文区 | 当前项目标记、已打开项目选项、会话是否可重绑、工作区模式、分支语义 | 发送前修正、确认历史归属 |
| 项目菜单区 | 已打开项目选项、当前项目标记 | 发送前修正 |

对应关系自查：全部“必看 / 重要”字段都已进入区域。

## 2. 状态机
- **空白新会话态**：项目导航区显示每项目新会话动作；时间线为空白邀请；输入器文件夹为可展开项目触发器。
- **项目菜单展开态**：项目菜单列出所有项目，当前项目带选中标记；草稿保持可见；选择目标后关闭菜单并保持同一会话。
- **历史会话锁定态**：时间线有消息或运行/编排事实；输入器显示固定项目文本，没有菜单箭头。
- **重绑失败态**：菜单关闭或保持可操作，项目与会话仍是原值；默认错误区域给出可读失败提示。

## 3. 区域间关系
- 点击项目导航区某项目的 `＋` → 在该项目创建并选中空白会话 → 输入器显示该项目。
- 空白会话点击输入器文件夹 → 项目菜单展开。
- 选择另一项目 → 持久化层原子校验 → 成功时项目导航分组和输入器项目名同步刷新。
- 任一消息或父子关系事实出现 → 输入器项目入口转为锁定态。

## 4. 浏览路线落到区域
- **从项目新建路线** = 项目导航区点击目标项目 `＋` → 空白新会话态 → 输入器输入 → 发送。
- **发送前修正项目路线** = 空白新会话态 → 输入器项目触发器 → 项目菜单展开态 → 选择目标 → 返回空白新会话态。
- **确认历史归属路线** = 项目导航区选择历史会话 → 历史会话锁定态 → 查看固定项目上下文。
