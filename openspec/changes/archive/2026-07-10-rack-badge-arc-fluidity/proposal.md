# 提案：rack-badge-arc-fluidity

## 背景
三轮迭代后用户仍反馈「一点也不丝滑」。反思结论（用户已确认）：

1. **方法论错误**：此前全部用定格帧验证运动，帧帧正确 ≠ 播放丝滑——丝滑住在帧与帧之间（速度连续性、帧率），定格验证在结构上抓不到顿挫。
2. **主犯：时间两拍 = 速度归零再重启**。上一版把「拿起→入窗」做成时间轴上的两段缓动拼接，38% 拼接点所有运动分量同时刹停再重新加速，播放读作「卡抬起来、停一下、再出发」。iOS 丝滑的本质是一条连续速度曲线，从不「到位一个姿态再出发」。
3. **「拿起再飞」的正确表达是空间弧线，不是时间分拍**：水平方向一条曲线全程不停（人眼判定丝滑的底线）；垂直方向先升后落的抛物弧（顶点垂直速度为零是物理自然的，不产生顿挫感）；转正在弧线前段平滑完成。三个分量各自平滑、并行插值，合成即「拿起划弧转正飞入」。
4. 次因：opacity 全用 linear+硬拐点（变化率突变）；760ms 偏长放大瑕疵；遮罩 `backdrop-filter: blur(6px)` 在动画期间每帧重合成背景，可能真掉帧。

## 提案
只改 `sites/marketeam/index.html`，三层容器变形结构（幽灵/变形面/真面板、像素级换装）保留，重写时间编排：

1. **单段时长**：打开 600ms、关闭 380ms，无任何时间分拍。
2. **幽灵卡拆三条并行平滑曲线**（嵌套三层 wrapper，各动一轴）：
   - 外层 X 平移：一条连续缓动，从架上到弹窗**全程不停**；
   - 中层 Y 平移：抛物弧——升至高于两端的弧顶（顶点零速自然）再落入卡位；
   - 内层 rotateY+scale：前重单曲线，转正在弧线前段完成（关闭反之，临近插槽才转回侧立）。
3. **淡化全部换平滑缓动**（ease-in-out），消灭 linear 拐点的变化率突变；交叉淡化/头像收敛窗口随弧线节奏微调。
4. **遮罩 blur 挪到落定后**：变形期间遮罩纯色压暗，落定加 `is-settled` 类过渡启用 backdrop blur——把最贵的每帧背景重合成从动画期挪走，保帧率。
5. **验证方法升级**：真实播放中逐帧采样幽灵位置，断言水平位移增量序列**不得中途降到零再回升**；再辅以关键帧截图。

## 影响
- 修改 `marketing-site` 域 spec：「拿起」从时间动作改为弧线契约 + 新增速度连续性质量契约（水平速度不得中途归零）。
- wireframes 动效注記同步（两拍 → 弧线）。
- 零第三方依赖、降级与防重入契约不变。

## 已知取舍
- 变形面起步给 ~12% 延迟后单曲线长成——元素**从静止开始**动一次是自然的（万物起步皆从零速），要消灭的是**中途**刹停重启；
- 抛物弧顶点垂直速度为零被保留——这是物理正确的「抛起悬停瞬间」，与水平持续前进叠加后无顿挫感。
