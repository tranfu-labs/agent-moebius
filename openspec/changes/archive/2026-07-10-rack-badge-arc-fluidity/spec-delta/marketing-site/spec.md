# marketing-site spec delta：rack-badge-arc-fluidity

「拿起」从时间分拍改为空间弧线契约，新增速度连续性质量契约（修复顿挫）。

## 修改（归档时生效）

### 「员工工牌与工牌架」转场规则第 1 条修改为
- 工牌架点击/激活 MUST 打开详情并**此时才显示工牌正面**：转场为标准容器变形，且 MUST 保留黑胶唱片架「拿起」的观感——但拿起 MUST 表达为**空间弧线**而非时间分拍：被点击的工牌作为**共享元素**沿先上升、后滑入的连续弧线飞抵**弹窗内工牌位**（非面板中心），由侧立 ≈44° 的转正 MUST 在弧线前段平滑完成；**转场全程水平位移速度 MUST NOT 中途归零**（所有运动分量 MUST NOT 同时刹停后重启）；途中由简化卡面交叉淡化为完整工牌，落定后与真工牌像素对齐换装；转场全程同一时刻 MUST 只存在一份该工牌的可见实例，MUST NOT 双卡同屏或让被追踪的卡中途消失；弹窗呈现正面放大工牌 + 角色名、charter、`@<role>` 用法；MUST 键盘可达（focus + Enter）。

## 场景修改

### 场景 MS.12 的打开 Then 行修改为
Then 该工牌沿先上升（拿起）后滑入的连续弧线飞抵弹窗内工牌位，转正在弧线前段完成，全程水平速度不断流，途中由简化卡面渐变为完整工牌

### 场景 MS.12 的关闭 Then 行修改为
Then 以短于打开的时长逆向弧线：工牌升离弹窗、临近插槽才转回侧立并落回架上当前位置，架上原卡恢复，焦点还给触发工牌
