# 提案：restore-last-console-selection

## 需求基线

| 文件 | 小节 | 变更 | 状态 |
| --- | --- | --- | --- |
| `docs/product/pages/main-left-sidebar.md` | 入口与去向 | 明确桌面启动恢复上次成功打开的项目与对话，并定义历史选择不可恢复时进入未选项目的新建对话 | 已写入 |
| `docs/product/pages/main-left-sidebar.md` | 指标与验收 | 增加跨重启恢复与安全回退的可核查验收 | 已写入 |

## 背景

桌面 renderer 当前用固定的 `local/default` 作为每次挂载的初始选择。后端会优先接受这组仍然存在的记录，因此应用启动后总是打开默认项目的历史默认会话，即使用户上次工作在其他项目与对话。侧边栏 PRD 已要求用户从最近使用的位置继续，但历史选择不可恢复时的产品回退此前没有定义。

## 提案

- 记住最后一次成功展示的用户根对话及其项目，并在下一次桌面启动时验证后恢复。
- 没有记录、记录损坏、项目已移除、对话已归档或返回对象不是用户根对话时，进入未选择项目的新建对话，拒绝把后端兼容回退误认成用户选择。
- 只有成功提交的状态刷新可以推进记忆；失败、过期刷新、轮询和子会话右栏展开不得改写选择。
- 保留 `local/default` 作为本地 API 兼容入口，但不再把它当成桌面产品的固定启动目的地。

## 影响

- `desktop/src/console-page/`：增加选择偏好纯逻辑并接入 renderer 启动与成功选择路径。
- `desktop/tests/`：覆盖偏好容错、有效恢复、安全回退与确认后持久化。
- `openspec/specs/local-console/spec.md`：归档时合并桌面启动选择 Requirement。
- 页面布局与视觉结构不变，不新增 `wireframes.md`。
