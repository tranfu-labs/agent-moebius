# 线框：main-conversation-subsession-cards

基线：`docs/product/pages/main-conversation.md#页面结构`（子会话卡片）与 `#响应式与窗口行为`。该页面 PRD 已接管主内容区版式，本文件只记录本次 change 相对基线**新增或改动**的部分；归档时回流到该页面 PRD，不写入 `docs/wireframes/`。

## pages/main-conversation.md

### 卡片在时间线中的位置（沿用基线）

```text
│ (经) 开发经理                                               │
│ 这个目标要动三个模块，我拆成三段并行推进。                  │
│ ┌─────────────────────────────────────────┐                 │
│ │ ▸ 落地页文案          开发    进行中    │                 │
│ │ ▸ 空状态验收          测试    没跑起来  │                 │
│ │ ▸ 构建脚本            开发    已结束    │                 │
│ └─────────────────────────────────────────┘                 │
```

卡片紧跟触发拆分的那句话，是时间线上的一条记录，不是浮在时间线之外的面板。三列固定：子任务、负责成员、当前状态。状态列不可省——子会话的内容不在主时间线上，不给状态用户就只能挨个点开才知道跑完几个、有没有一个挂了。

### 右侧展开（沿用基线）

```text
┌ 侧边栏 ──┬ 主会话（右半被覆盖）───────┬ 子会话 ─────────────┐
│          │ (经) 开发经理              │ 空状态验收      [×] │
│          │ ┌──────────────────┐       │ ─────────────────── │
│          │ │ ▸ 落地页文案     │       │  内容、输入与操作   │
│          │ │ » 空状态验收     │       │  由右侧展开区需求   │
│          │ │ ▸ 构建脚本       │       │  另行定义           │
│          │ └──────────────────┘       │                     │
└──────────┴────────────────────────────┴─────────────────────┘
```

被展开的那一行在卡片上标出（`»`），父会话左半仍在视野中。关闭后回到父会话原来的滚动位置。

展开区内部本片只交付外壳，内容先接现有会话视图——PRD 已把它的内部结构、输入方式和操作划给该区域的独立需求，越界实现会让后续需求讨论被现状反向限制。

### 侧边栏不再有子会话

```text
改前                          改后
│ ▾ 项目 A       │            │ ▾ 项目 A       │
│    官网落地页  │            │    官网落地页  │
│    落地页文案  │            │                │
│    来自：官网… │            │                │
│    空状态验收  │            │                │
│    来自：官网… │            │                │
│    构建脚本    │            │                │
│    来自：官网… │            │                │
```

三个子任务从侧栏消失，改由卡片承载。侧栏长度不再随团队的拆分行为膨胀。

### 窄窗收敛顺序

```text
宽    [▱ 项目 A] [默认工作空间 ⇄] [main] [◇ 开发团队 ▾]
↓     [▱ 项目 A] [默认工作空间 ⇄]        [◇ 开发团队 ▾]     ← 先收分支
↓     [▱ 项目 A]                         [◇ 开发团队 ▾]     ← 再收工作空间
窄    [▱ 项目 A] [◇ 开发团队 ▾]
```

分支最先收，因为它是四项里唯一不可点击的纯信息。项目和团队最后收——PRD 点名它们最后收敛，而且它们是仅有的两项还能改的上下文。
