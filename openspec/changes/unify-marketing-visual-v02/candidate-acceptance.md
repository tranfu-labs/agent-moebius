# 三路线候选验收尺

## 结论规则

- `PASS`：无 P0/P1/P2；允许记录不阻断选择的 P3。
- `REWORK`：存在任一 P0/P1/P2，必须列出文件、位置、可观察证据和具体修复要求，发回原作者。
- 复验必须由原审查者执行，并明确逐条关闭上一轮问题；不得仅凭作者自述通过。

## 共同底线

1. `styleN.html` 是可操作样张，不是静态效果图；中心作品必须能逐步推进/回退或播放，并可冻结状态验收。
2. `indexN.html` 使用正式 Moebius 叙事，不出现“视觉令牌、样张、设计方向”等设计说明文案。
3. 必须清楚表现：角色接棒、一次带原因退回、复核通过、真人决策边界、交付证据。
4. 三条路线不得共享同一首屏骨架换皮；中心作品、信息层级和主要交互模型必须可辨识。
5. 不得复刻 Kimi 的蓝色双行标题、研究报告封面、芯片爆炸图、胶囊章节导航或点阵画布。
6. 不修改正式 `style.html` / `index.html`；不依赖 Kimi、D3、外部 SDK 或远程业务数据。

## 浏览器验收

- desktop：1440×1000。
- tablet：1024×768。
- mobile：390×844。
- 三档均不得有页面级横向滚动、遮挡主要控件、不可读文字或顺序错乱。
- 核心控件必须可用键盘操作，有可见焦点；展开控件同步 `aria-expanded` / `aria-controls`。
- `prefers-reduced-motion: reduce` 下信息完整、无强制循环动画。
- no-JS 下仍能看到完整叙事、角色、证据与至少一个稳定流程终态。
- 控制台无页面自身的 error；资源无 404。

## 路线特有判据

### 路线 1 · 运行中底单

- 六行记录是第一视觉主角；时间、负责人、动作、状态、证据可扫读。
- 退回行与原执行行有明确关系，真人决策行使用非颜色的独特形状。
- 时间轴控制会同步更新记录可见性、路径或当前行，不能只是按钮文字变化。

### 路线 2 · 接棒场

- 五个角色站位在任务到达前已存在，所有权沿连续主线移动。
- 红色回线明确从评审返回执行，真人决策缺口打断自动路径，交付绿线只在通过后出现。
- 站位、任务片段和证据之间至少有一组 hover/focus 关系联动。

### 路线 3 · 证据织机

- 角色经线、证据纬线和主任务线在视觉上可区分，证据随交接累积而非一次出现。
- 返工回线重新进入执行段，通过段与真人决策缺口均可定位。
- 交付收据可展开，并能从收据条目定位对应角色或流程段。

## 严重级别

- P0：页面不可用、核心流程无法操作、严重可访问性或脚本崩溃。
- P1：中心作品不成立、路线与目标图/路线定义严重偏离、正式叙事缺失、明显照抄 Kimi。
- P2：响应式溢出、核心状态联动不完整、键盘/ARIA/reduced-motion/no-JS 缺口、重要层级或可读性问题。
- P3：不影响理解与操作的细节润色。
