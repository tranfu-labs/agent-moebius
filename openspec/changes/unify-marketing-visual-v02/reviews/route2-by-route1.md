## 阻断发现

### [P2] no-JS 终态自相矛盾，两页都不能形成稳定流程终态

- 文件：`sites/marketeam/style2.html`、`sites/marketeam/index2.html`
- 位置：源 HTML 的接棒场初始值，以及 `.no-js` 终态覆盖规则。
- 复现：用 CSP `script-src 'none'` 实际禁止页面脚本后打开任一页面。
- 可观察证据：
  - `<html>` 正确保留 `class="no-js"`，控制器隐藏；黑色进度走到末端，绿色交付线为 `matrix(1, 0, 0, 1, 0, 0)`。
  - `<noscript>` 同时声明“静态终态……最终方案与全程证据已归档”。
  - 但页头“当前持有者”仍为“定义者”，步数仍为 `00 / 07`。
  - 五个站位仍分别显示“等待任务 / 尚未接手 / 尚未接手 / 等待决策 / 等待交付”。
- 影响：同一画面同时声称任务已完成和任务尚未开始，读者无法判断稳定终态；不满足共同底线中的 no-JS 完整叙事与稳定流程终态。
- 具体修复要求：
  1. 把源 HTML 默认值改成真实的第 7 步终态：持有者为交付者、步数 `07 / 07`、前四站已完成、交付站已通过，交付事件与证据归档一致。
  2. JS 启动后再调用 `render(0)` 恢复交互初始态；无 JS 时则自然停在一致的最终态。
  3. `style2.html` 与 `index2.html` 必须同时修复，并重新在禁止脚本的真实页面中验证，不得只改 `<noscript>` 文案。

### [P2] hover/focus 只联动站位与证据卡，没有联动任务片段

- 文件：`sites/marketeam/style2.html`、`sites/marketeam/index2.html`
- 位置：站位 `mouseenter` / `focus` 的 `show()` 处理，以及 `.task-token` / `.task` 的视觉状态。
- 复现：桌面端聚焦第 3 站“审阅者”，再按 Tab 移到第 4 站“决策点”。
- 可观察证据：
  - 第 3 站获得 `is-related` / `rel`，关系卡透明度变为 `1`，内容更新为 `03 审阅者 → review_1.md`。
  - Tab 到第 4 站后，关系卡更新为 `04 决策点 → human_decision_01`。
  - 任务片段始终只有原类名 `task-token` / `task`，边框与位置没有联动状态，相关主线段也没有变化。
  - 卡片文字虽然写着“站位、任务片段与证据……保持关联”，画面上实际只证明了“站位 ↔ 证据卡”。
- 影响：没有满足路线 2 特有判据“站位、任务片段和证据之间至少有一组 hover/focus 关系联动”。
- 具体修复要求：
  1. 在站位 hover/focus 时，为任务片段及对应的所有权线段增加同一可见关联状态；不能只改关系卡文案。
  2. 建议让任务片段移动或套准到对应站位，或让对应主线段与任务片段同时强调，并保留证据卡更新。
  3. mouseleave/blur 必须完整清理联动状态；键盘 Tab 切换站位时，三者应同步切换。
  4. 修复后复验 390px 第 5 站 focus，确保关系卡和任务片段联动不会重新制造横向溢出。
