# 设计：unify-marketing-visual-v02

## 方案

### 1. 单向事实链
视觉规则按以下方向流动，后项不得反向发明新规则：

`视觉宪法 v0.2 → style.html 样张 → 人工验收闸门 → index.html → spec / wireframe 归档事实`

视觉宪法定义“为什么”和封闭规则；`style.html` 定义可见令牌、组件、版式和交互原语；`index.html` 只能组合已在样张验收通过的原语。

### 1.1 三路线候选实验（2026-07-18 用户指令）
在正式事实链之前并行制作三组隔离候选，不把 Kimi 的表层构图当作母版：

1. `style1.html` + `index1.html`：运行中底单（Live Ledger），以逐行累积的可审计记录为中心作品。
2. `style2.html` + `index2.html`：接棒场（Handoff Field），以固定角色站位、接力棒、退回弧线和真人决策缺口为中心作品。
3. `style3.html` + `index3.html`：证据织机（Proof Loom），以角色经线、证据纬线、返工回线和最终收据为中心作品。

六个编号文件仅是候选实验，不替代 `style.html` / `index.html`，也不解除正式 `index.html` 的人工闸门。每组由独立作者实现；首轮完成后采用循环交叉验收（1 审 2、2 审 3、3 审 1）。审查发现 P0/P1/P2 时 MUST 打回原作者修复，并由原审查者复验；只有三组都通过候选验收后，才向用户并排展示并选择后续收敛方向。

三条路线可以共享视觉宪法 v0.2 的语义纪律、三声部、状态色和无障碍底线，但 MUST 在中心作品、信息结构与交互模型上保持可辨识差异；不得把 Kimi 的蓝色双行标题、研究报告封面、芯片爆炸图、胶囊章节导航或点阵画布作为共同模板。

### 2. 阶段一：style.html 验收实验室
样张按六个层次组织：

1. 唯一潜台词：明确“你可以走开，因为每一步都有人管、有痕迹”。
2. 基础令牌：纸、墨、灰阶、对齐蓝、通过绿、退回橙，以及判断/解释/记录三声部。
3. 版式语法：692px 阅读栏、924px 证据作品和全宽中心作品三种节奏；留白必须有明确指向。
4. 中心作品：参数化点阵、套准标、任务轨道、底单记录和可冻结的 `render(t)` 状态。
5. 交互与动效：底单行与站点关系型 hover/focus、可下钻底单、播放/暂停/拖动、四种受控动效原语。
6. 降级矩阵：desktop、tablet、mobile、`prefers-reduced-motion`、no-JS。

宽屏可展示叙事栏与证据栏的并置关系；窄屏 MUST 将所有证据收回正文流，不保留常驻侧栏，不产生页面级横向滚动。

692px / 924px 是 v0.2 对“阅读与证据形成尺度反差”的样张实现值，不先写死为不可调整的品牌常量。检查点 A 通过后，它们才成为本轮迁移到正式页的已验收版式原语。

阶段一分为三个检查点：

- 检查点 A：潜台词、令牌、三声部与三种版式宽度；只判断层级、节奏和色彩纪律。
- 检查点 B：中心作品、固定帧、底单行 ↔ 站点关系交互；只判断核心作品和流程语义。
- 检查点 C：desktop/tablet/mobile、reduced-motion、no-JS、键盘与 ARIA；形成样张最终验收包。

每个检查点先展示证据并消化反馈，再进入下一检查点；只有检查点 C 通过后才可请求最终“style.html 验收通过”。

### 3. 动效模型的可测性
`style.html` 的 HTML/CSS/文案属于纯展示，单元测试豁免，使用截图与 DOM 验收。`render(t)` 中的位置、进度、状态和可见事件属于可测逻辑，MUST 收敛为纯 `frameAt(t)` 模型，再由 DOM adapter 应用到 SVG 与底单。

为保持样张单文件、自包含且不引入新的运行时文件，纯模型继续内联在 `style.html`，并只在样张页暴露冻结的 `window.__MOEBIUS_STYLE_CONTRACT__` 测试接口。Vitest 通过 jsdom 加载真实样张脚本、注入最小浏览器 API stub，再直接调用该接口；测试不得复制一份平行的运动算法。正式 `index.html` 在阶段二迁移时以内联实现和固定帧契约测试核对一致性，不依赖样张页的测试全局。

单元测试覆盖：
- `t` 边界钳制；
- `t=0.47` 退回状态；
- 任务位置允许回退，但最远进度痕迹不得回退；
- `t=0.84` 拍板状态；
- `t=0.92` 交付/签收状态；
- 底单事件到轨道站点的一一映射；
- reduced-motion 的稳定完整静态帧。

DOM 展开、hover/focus 联动、键盘操作和响应式不重复伪造单元模型，统一由浏览器验收脚本覆盖。

### 4. 阶段一验收证据
验收脚本通过本地 HTTP 打开 `style.html`，至少生成：
- 1440×1000、1024×768、375×812 三档页面截图；
- `?t=0`、`?t=0.47`、`?t=0.84`、`?t=0.92` 中心作品固定帧；
- reduced-motion 静态终态；
- 交互、控制台、横向滚动和可访问状态的结构化证据。

用户看到证据后，只有明确回复“style.html 验收通过”，阶段二任务才可开始。

### 5. 阶段二：迁移到 index.html
正式页保留现有六板块业务叙事和单文件部署契约，只迁移已通过的视觉系统：
- 衬线负责判断，Sans 负责解释，Mono 负责记录；
- 首屏跟踪板成为唯一中心作品；
- 正文在阅读栏、证据作品和全宽场景之间形成节奏；
- 每屏至少一个尺度、明度或密度反差锚点；
- hover/focus 只揭示业务关系；
- 移动端与 reduced-motion 继续保持完整可读。

正式页不得引入 D3、Kimi SDK、外部研究数据或与 Moebius 叙事无关的 Canvas 系统。

## 权衡
- 选择先验收样张再迁移正式页，牺牲一次性交付速度，换取明确的视觉基准和更低的正式页返工成本。
- 选择迁移编辑部式层级与节奏，而非像素复刻 Kimi 页面，保留 Moebius 的六段产品叙事、诚实证据纪律和轻量单文件部署。
- `style.html` 可承担更完整的解释和控件；`index.html` 只消费通过的结果，不把设计系统说明文字带到生产页面。
- 不在本变更处理发布目录暴露 `style.html` 等部署债，避免视觉验收与运维治理耦合。

## 风险
- 三声部与更强反差可能破坏原有“安静”感；以色彩语义、动效预算和每屏单一主角约束音量。
- 点阵、套准标和中心作品若遍地使用会退化为装饰；样张和正式页都限制为每页唯一中心作品。
- 单文件正式页继续存在样张与实现漂移风险；阶段二通过视觉契约测试、固定帧和归档事实源降低风险。
- Google Fonts 不可用时字面宽度会变化；必须验证系统衬线/Sans/Mono 回退仍保持层级与可读性。
- 回滚按阶段进行：阶段一可恢复样张与宪法旧版本；阶段二可恢复上一稳定 `index.html`，不涉及数据迁移。
