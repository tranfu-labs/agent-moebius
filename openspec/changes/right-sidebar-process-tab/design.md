## 本 change 负责的验收落点

| 验收 | 落点 | 现状 |
| --- | --- | --- |
| #12 过程标签完整输出含原始错误 | `process-tab.tsx` 消费 `runOutput` 的 `stdout`/`stderr`（完整文件） | 入口 / 数据源已有，呈现新建 |
| #13 标题 = 成员名、同成员追加序号、不自编描述 | `process-tab.tsx` 标题取意图 `role` → 既有 role→名映射；同成员序号计数 | 映射族已有，标题生成器新建 |

## 本 change 承接的规则句

- **R10「过程标签标题就是成员名；不得凭猜测生成描述性标题」**：标题必须取自意图 `role` 的 role→名映射（`resolveChildSessionMember` 那套「取不到则成员未知，从不编描述」的范式），**不得**从 `safeRunSummary` / liveMarkdown 派生——summary 本就不可信（含机器词时会被换成「正在推进这一步…」）。
- **R24「完整输出含原始错误」**、**R25「追加、不摘要、不折叠、不美化改写」**：过程标签**不走** `run-block.tsx` 的 markdown 美化通道，纯文本追加。
- **R26「同步骤重试每次执行同标签依次追加、标第几次、不覆盖」**：`runOutput` 是单 runId，本片按 (成员, 步骤) 聚合多个 run。
- **R27「只读但内容可复制、文件路径同样可复制」**：**规则冲突落点** —— `sanitizeMachineText` 会把路径抹成「已隐藏」。过程标签要求路径可复制，**必须绕开** sanitize 走原始通道。这是失语项（PRD 一面要过程标签露原始路径、一面在会话区结论抹路径以保护普通用户，总边界未画清）——本片默认：**过程标签露原始路径，其余区域仍抹**。
- **R51「输出超上限允许截断，但必须让用户看出被截断」**：现状是 64KB 尾部截断，信号是机器串 `tail-truncated`，无 UI 呈现。本片把它转成用户可见标记。这是失语项（截断方向 PRD 未表态，现状是「截头留尾」，按时序读时开头被截反直觉）——本片默认：**沿用现状截尾留头，但显式标出被截断**；不承诺无论多大都逐字保留。
- **R45/R1（只读白名单）**：过程标签只读，只允许选中 / 复制 / 关标签，不得出现任何写操作。

## PRD 失语项与本片默认取值（供 loop 审计与用户裁决对照）

1. **过程标签路径可复制 vs 全局 machine-text 抹路径**（R27）：本片默认过程标签绕开 sanitize 露原始路径，其余区域不变。
2. **截断方向**（R51）：现状 64KB 尾部（截头留尾）；本片默认沿用，但把截断变成可见标记。PRD「已知隐患」已明确「截断尺度不在本文规定 + 必须让用户看出」，故方向按现状、可见性必须做。

## 方案

关键决策：

1. **过程标签的数据 = evidence-outlets 的 `runOutput`，本片加聚合层**。`runOutput(sessionId, runId)` 给单 run 的完整 `stdout/stderr/fallback`。本片按「同一成员同一步骤的多次 run」聚合成有序分段（第 1 次执行 / 第 2 次执行 …），过程标签按段渲染。
2. **runDir 在 `/tmp`，读不到就明说，不假装完整**。runDir 被清后 `stdout/stderr` 为 null——本片降级到 `fallback`（会话事实里的错误 / 正文），并显式说明「原始输出已不可用」，与「这一步没有产生输出」是不同措辞。是否把 runDir 迁出 `/tmp` 求跨重启保真属已知隐患，本片不强求迁移，但保证读不到时不静默留白。
3. **标题生成器独立于内容**：标题只吃意图 `role`，走 role→名映射；同成员序号在标签条层面计数（这段对话内第几个该成员的过程标签）。内容再脏也不回灌标题。
4. **截断可见**：把留存侧的 `tail-truncated` 诊断转成过程标签顶部 / 底部的可见「此处已截断」标记，注意别被 machine-text 抹掉。

## 权衡

- **绕开 machine-text 的边界风险**：过程标签露原始路径，若将来复制粘贴场景被质疑泄露，属失语项、可由用户裁决收回。本版按 PRD「过程标签给原始输出 + 路径可复制」的明文取原始通道。
- **不迁 runDir 出 /tmp**：迁移是更大的持久化改造，本片不背；代价是跨重启后老步骤的原始输出可能读不到（降级到 fallback + 明说）。这与 PRD「已知隐患」接受的后果一致。

## 风险

- **`runOutput` 是 evidence-outlets 既有入口**：加聚合 / 改语义须回扫调用点（evidence-outlets 单视图降级也用它），callsite_sweep 覆盖。
- **`server.ts` 与 ③ 的共享写点**：两片都加只读路由，追加式 + rebase-before-archive 处理。
- **实时追加与定格的切换**：正在工作时实时追加、结束后定格，不能因组件重挂丢历史段。缓解：分段数据以 (成员, 步骤, 第几次) 为键累积。
