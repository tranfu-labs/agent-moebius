# 线框：main-conversation-new-page

基线：`docs/product/pages/main-conversation.md#页面结构`。该页面 PRD 已接管主内容区版式，本文件只记录本次 change 相对基线**新增或改动**的部分；归档时按 `openspec/changes/AGENTS.md` 第 3 步回流到该页面 PRD，不写入 `docs/wireframes/`。

## pages/main-conversation.md

### 新对话 · 项目未选定（沿用基线，补禁用原因）

基线已定义顶部进入时的整体版式。本次新增的是**发送禁用原因的位置**——PRD 只写了「发送禁用」，没写用户如何知道为什么。

```text
┌ 侧边栏 ────────┬──────────────────────────────────────────────────────┐
│ ＋ 新建对话  » │  新对话                                              │
│ ⌕ 搜索         │                                                      │
│ ◇ Agent 团队   │                                                      │
├────────────────┤            描述你的目标，团队会开始推进              │
│ 项目           │                                                      │
│ ▾ 项目 A    ＋ │                                                      │
│    对话 1      │   ┌──────────────────────────────────────────────┐   │
│ ▸ 项目 B    ＋ │   │ [▱ 选择项目 ▾] [◇ 开发团队 ▾]                │   │
│                │   │ 描述你的目标…                             ⊘ │   │
├────────────────┤   └──────────────────────────────────────────────┘   │
│ ⚙ 设置         │     选择一个项目后才能发送                           │
└────────────────┴──────────────────────────────────────────────────────┘
```

禁用原因是一行常驻文本，不是禁用按钮的悬停提示。这是必须先解决才能往下走的阻塞，藏在悬停后面等于让用户对着灰按钮猜。

### 项目按钮展开（沿用基线，补「添加项目…」的分隔）

```text
   ┌──────────────────────┐
   │ ✓ 项目 A             │
   │   项目 B             │
   │ ──────────────────── │
   │ ＋ 添加项目…         │
   └──────────────────────┘
   ┌──────────────────────────────────────────────┐
   │ [▱ 项目 A ▴] [默认工作空间 ⇄] [main]         │
   │ 未发送的输入草稿                          ↑ │
   └──────────────────────────────────────────────┘
```

分隔线之上是选择，之下是新增。添加项目是本页面唯一的项目创建入口，侧边栏不再承担。

### 一个项目都没有时

```text
│   ┌──────────────────────────────────────────────┐   │
│   │ [▱ 选择项目 ▾] [◇ 开发团队 ▾]                │   │
│   │ 描述你的目标…                             ⊘ │   │
│   └──────────────────────────────────────────────┘   │
│     还没有项目，从上面的项目按钮添加一个              │
```

与「项目未选定」同一种版式，只有禁用原因的措辞不同。不为空项目列表另做一套空态插画——PRD 的立场是新对话页只有「有没有选项目」这一个区别，再分叉会让同一个页面出现三种长相。

### 发出第一条消息后

```text
│ 帮我把落地页的空状态补上验收…                              │
│ ─────────────────────────────────────────────────────────── │
│ 你                                                          │
│ 帮我把落地页的空状态补上验收语句                            │
│ ─────────────────────────────────────────────────────────── │
│   ┌──────────────────────────────────────────────────────┐  │
│   │ ▱ 项目 A  [默认工作空间 ⇄]  main  [◇ 开发团队 ▾]     │  │
│   │ 说点什么，或 @ 一个成员…                          ↑ │  │
│   └──────────────────────────────────────────────────────┘  │
```

页面标题由「新对话」变为首句开头文字。项目从可点击的下拉变成不可点击的文本 `▱ 项目 A`（无 `▾`），工作空间与团队保持可点。这是同一个页面的状态变化，不是页面跳转。
