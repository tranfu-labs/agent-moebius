# UI 设计风格：conversation-console

线框与信息架构见 [wireframes.md](wireframes.md)，交互推导见 [design.md](design.md)；本文件只承载视觉风格与文案语气的决策及推导，供实现阶段作为样式层的事实源。

## 一、定位输入与已定决策

目标用户是**普通用户，不是程序员**。产品底层跑 codex / gh 是实现细节，不是产品气质；界面上不保留任何终端质感（等宽字体铺面、ANSI 式高饱和状态色、命令原文直接示人）。

与用户逐轮确认的决策：

| 决策点 | 结论 |
|---|---|
| 风格基调 | 安静工作台风，参照 Notion / Linear 一脉：低饱和中性底、细边框、靠间距与字重分层 |
| 明暗模式 | 双模式跟随系统，用同一套语义变量翻转 |
| 强调色 | 靛紫，全局唯一强调色 |
| 角色识别 | 彩色首字母头像（低饱和淡色）；头像是**可替换插槽**，后续另有替换方案 |
| 信息密度 | 舒适档（Notion 感），行高宽松、卡片有呼吸感 |
| 机器术语 | 全面转人话，原始术语与输出原文藏进展开层 |

## 二、风格总纲：安静是功能，不是审美偏好

design.md 已经写死：界面越安静系统越健康，不得存在任何催促阅读的元素。视觉层把这条哲学翻译成三条铁律：

1. **颜色出现即信号**。健康状态（没有事等人）下，整个页面接近单色：中性灰阶 + 少量靛紫交互控件。琥珀色出现 = 有事等你；红色出现 = 有东西失败或被中断。颜色永远不做装饰。
2. **一屏至多一个实心强调按钮**。其余按钮一律描边或幽灵样式。多个实心按钮抢注意力，与「注意力义务单调递减」的排序哲学相拧。
3. **动效只服务状态感知**。全页面唯一的持续动效是运行块的呼吸指示；列表条目只在状态迁移时以短动画移位（列表动 = 有事发生）；禁止装饰性动画、禁止任何形式的角标脉冲。

## 三、色彩系统

### 3.1 层次结构

色彩分四层，权责互不越界：

| 层 | 用途 | 色相 | 约束 |
|---|---|---|---|
| 中性层 | 背景、卡片、边框、正文 | 暖灰阶 | 页面主体，占比 ≥ 90% |
| 强调层 | 主按钮、选中态、链接、焦点环 | 靛紫 | 交互专用，不表达状态 |
| 注意力层 | 「等你」的一切呈现 | 琥珀 | **琥珀只属于「等你」**，任何其他元素不得使用 |
| 事实层 | 验收通过 / 失败、运行错误、中断 | 绿 / 红 | 只标注已发生的事实，不用于预警渲染气氛 |

角色身份色（头像专用）游离在这四层之外：低饱和淡底 + 同族深字，只出现在头像圆圈内，不得渗透到按钮、文本、边框——保证「颜色出现即信号」不被身份色污染。

### 3.2 指示值（实施可微调，语义不可变）

浅色态：

- 页面底 `#F7F6F3`，卡片底 `#FDFCFA`，输入控件底 `#FFFFFF`
- 正文 `#1F1E1B`，次要文字 `#6F6E69`,提示文字 `#A3A29C`
- 边框细线 `rgba(0,0,0,0.08)`，强边框 `rgba(0,0,0,0.14)`
- 靛紫强调 `#534AB7`，淡靛紫底（选中态）`#EEEDFE`
- 琥珀文字 `#9A6700`，淡琥珀底 `#FAEEDA`
- 事实绿 `#3B6D11`（淡底 `#EAF3DE`），事实红 `#A32D2D`（淡底 `#FCEBEB`）

深色态按同一语义翻转：底色落在近黑暖灰（`#17161A` 一带），文字翻为浅灰阶，靛紫与琥珀提亮一档保证对比度。所有颜色一律通过语义变量引用（如 `--surface-*`、`--text-*`、`--accent`、`--attention`），组件代码不得出现裸色值。

### 3.3 角色身份色盘（首版）

| 角色 | 界面名 | 头像底 / 字 |
|---|---|---|
| ceo | CEO | 石墨灰（权威中性，避免与强调靛紫相撞） |
| dev | 开发 | 蓝 |
| qa | 测试 | 绿 |
| dev-manager | 技术负责人 | 青 |
| product-manager | 产品 | 粉 |
| hermes-user | 用户代表 | 珊瑚 |
| 真人用户 | 你 | 中性灰 |

头像是插槽：首版渲染「身份色圆 + 首字」，后续替换方案（形象、图片等）只换插槽内容，身份色映射表保留作降级。

## 四、字体与排版

- **全界面使用系统中文字体栈**（PingFang SC / Microsoft YaHei / 系统默认），不引入外部品牌字体，不使用等宽字体做界面元素。
- 唯一例外：展开层里的**命令输出原文、日志原文**按等宽排版——那是内容的本来面目，不是界面风格；折叠态的概括行（「正在运行测试」）仍是普通字体人话。
- 字号层级（舒适档）：页面标题 18、卡片标题 / 会话名 15、正文 14、辅助信息 12，最小不低于 12。字重只用两档：常规与中黑（500），不用粗黑。
- 行高 1.6 上下；消息卡内边距宽松（约 12×16），侧栏条目单行但行高 36 左右，保证普通用户的点击容错。

## 五、形状、层级与图标

- 圆角：控件 8，卡片 12。不做直角，也不做胶囊化滥用（胶囊只给计数徽记与状态签）。
- 边框：0.5–1px 细线为主，靠底色差与间距分区，不靠粗框线。
- 阴影：常态卡片零阴影或极轻;只有浮层（菜单、@ 补全面板、抽屉）用一层柔和投影，同屏浮层至多一层。
- 图标：线性描边风格单一图标库，粗细统一;不混用实心图标，不用表情符号做功能图标（线框里的 ✋ / ⣾ 在实现层替换为同语义的描边图标与呼吸动效点）。

## 六、关键组件的风格落点

对照 wireframes.md 逐个锚定：

- **侧栏会话行**：单行、无卡片感，悬停淡灰底，选中淡靛紫底。「等你」条目：琥珀图标 + 常规文字（不整行染色——行级染色会让多条「等你」把侧栏变成警报墙）。运行中条目：中性色呼吸点，不占色相。
- **顶栏计数**：「N 等你」是琥珀胶囊签，「N 运行中」是纯文字。永远不用红点。
- **用户消息**：白底卡片、全文显示，视觉分量略高于 agent 消息（人是闸口，人说的话是锚点）。
- **agent 折叠消息**：无卡片、缩进文本行——「角色头像 + 中文角色名 + 人话 stage + 时间 + 结论一行 + 交给谁」。展开后进卡片，原文与原始术语在卡片内。折叠态的低视觉分量本身就在说「这些你可以不读」。
- **运行块**：白底卡片 + 中性呼吸指示 + 人话概括行（「正在运行测试 · 已进行 3 分 12 秒」）。中断按钮是红色描边幽灵按钮——事实层颜色只在按钮上，不染卡片。点开可看原始输出（等宽排版的内容层）。
- **交棒节点**：消息间的细分隔线 + 居中小字「开发 → 测试」，是标点不是事件卡。
- **「轮到你了」验收卡**：整卡淡琥珀底 + 琥珀左饰条，是页面上唯一允许「响」的组件；逐条通过 / 不通过是成对描边按钮，选中后分别染事实绿 / 红。「提交验收结果」是琥珀实心按钮（此屏唯一实心，强调层让位注意力层）。
- **CEO 提案卡**：白卡 + 靛紫描边确认按钮；「确认提案」可用靛紫实心（它是推进型操作，不是阻塞型义务）。
- **@ 补全面板**：浮层列出「头像 + 中文角色名 + 一句职责」，选中生成协议合法 mention；用户界面上看到的是中文名，协议层句柄由控件代管。
- **账本树**：缩进树 + 状态签（✓ 绿签 / 呼吸点 / 琥珀签），节点行悬停出现「打开会话」链接按钮。
- **空态**：一句邀请式文案 + 输入框，不放插画堆砌;语气按 8.2 节。

## 七、术语人话化对照表

界面层展示人话，协议层保持原文；对照表是实现时的映射事实源：

| 协议原文 | 界面展示 |
|---|---|
| dev / qa / dev-manager / product-manager / hermes-user / ceo / secretary | 开发 / 测试 / 技术负责人 / 产品 / 用户代表 / CEO / 秘书 |
| plan-written | 方案已写好 |
| code-verified | 代码已验证 |
| in-progress | 进行中 |
| 运行输出尾行（如 `running pnpm test`） | 概括为「正在运行测试」等人话短语，原文进展开层 |
| 交棒 → @qa | 交给「测试」审查 |
| 等待真人：… | 等你：…（配琥珀呈现） |
| 验收走查 `N. 通过 — 依据` | 逐条勾选控件，提交时生成协议文本 |

概括短语生成不到位时降级显示原文，宁可露术语也不得空白或误译。

## 八、语气与文案

1. 句子说人话、动词开头、不用感叹号：「已交给测试审查」而不是「交棒成功！」。
2. 错误先说发生了什么、再说能做什么：「运行被中断。可以重新发消息让开发继续。」
3. 永远不催促：文案里不得出现「快去看看」「你有 N 条未读」这类召唤;「等你」的措辞是陈述义务（「等你验收」），不是营销推送。
4. 空态是邀请不是道歉：「描述你的目标，@ 一个角色开始」，不写「这里空空如也」。

## 九、设计走查清单（哲学的可验证断言）

实现或改版后逐条走查，全部应当成立：

1. 打开一个没有任何「等你」事项的项目 → 整页除头像圆与靛紫交互控件外无彩色，无任何角标、红点、脉冲动画。
2. 任意一屏截图 → 实心强调按钮数量 ≤ 1。
3. 全局搜索界面文案 → 无「未读」字样，无催促式措辞。
4. 出现一条「等你」 → 琥珀同时出现在顶栏计数、侧栏条目图标、会话内验收卡三处，且页面其他任何组件无琥珀。
5. agent 消息折叠态 → 不出现 stage 英文原文与命令原文;展开后可以找到原文。
6. 运行中的会话在侧栏 → 输出流动十分钟，条目位置不变;运行结束或交棒到人 → 条目以一次短动画移位。
7. 切换系统深浅模式 → 页面即时翻转，无裸色值残留（对比度抽查正文与琥珀签）。
8. 用鼠标走完「验收三条 + 提交」 → 全程零手写格式，提交产物为协议合规文本。
