# local-console spec delta：conversation-console

新增业务域「本地对话操作台」。范围声明：本 delta 只规定页面呈现与交互行为；数据来源、写入通道、触发与中断的实现通道、进程边界由后续对接设计 delta 补充，本 delta MUST NOT 被解读为已放行任何具体的读写实现方式。

## 新增行为规则

### 信息架构
- MUST 以「项目 → 会话」两层组织侧栏；会话在项目内平铺，MUST NOT 出现会话层级缩进或会话树。
- MUST 支持三种会话：编排会话（每个目标一个，人与 CEO 对话及查收处）、任务会话（对应账本任务，含修复子会话）、自由会话（不进账本）。
- MUST 在任务会话头部显示指回父编排会话的面包屑；父子与派生关系只通过面包屑、编排会话内事件条目、账本视图表达。

### 侧栏
- MUST 让会话行保持一行：状态图标 + 会话名称。
- 状态 MUST 只有三态：✋ 等你（人不行动流程即阻塞）、⣾ 运行中、无（静止）。
- MUST NOT 提供未读徽标、未读计数或任何催促人阅读 agent 消息的界面元素。
- MUST 按「✋ 等你 > 编排会话 > ⣾ 运行中 > 静止」排序，已完成任务会话自动收入折叠分组；状态先于类型（编排会话处于 ✋ 时升入 ✋ 段）。
- 排序 MUST 只随状态迁移（开始运行、运行结束、交棒到人、验收通过）变化；MUST NOT 因运行输出流动、时间流逝重排。
- 状态 MUST 只从被折叠不可见处向上冒泡：项目折叠时行尾聚合计数；项目展开时 MUST NOT 在项目行或编排会话行重复聚合子会话状态；已完成分组永不冒泡。
- 顶栏 MUST 显示跨项目全局计数（⣾ N 运行中 · ✋ M 等你）。

### 对话时间线
- MUST 单时间线多角色混排，每条消息带角色与 stage 标识。
- agent 消息 MUST 默认折叠为「角色 + stage + 结论 + 交棒行」，可展开全文；用户消息 MUST 始终全文。
- 交棒 MUST 渲染为显式的消息间节点或折叠态内的显式交棒行。
- 运行中 MUST 渲染为时间线内的活动块：运行角色、耗时、最近输出、中断入口。
- 「等待真人」MUST 渲染为显式行动卡片；验收类卡片 MUST 按验收语句逐条提供 通过 / 不通过 控件与依据输入，提交时 MUST 生成协议合规走查文本（每条独立一行 `N. 通过 — 依据` 或 `N. 不通过 — 依据`，外加独立一行 `验收结论：通过/不通过`）。
- CEO 阶段提案 MUST 渲染为确认卡片，确认动作 MUST 生成含提案 key 的合规确认回复。
- 输入框 MUST 提供合法角色的 @ 补全，且 MUST 约束一条消息至多一个合法 mention。

### 编排会话（查收点）
- 编排会话时间线 MUST 包含子会话里程碑事件的自动回流条目（任务会话创建、验收通过 / 失败、修复会话派生、运行开始 / 结束），每条 MUST 可跳转到对应子会话。
- 编排会话头部 MUST 常驻目标进展摘要（已验收 / 运行中 / 等你 计数）。

### 账本视图
- MUST 提供项目维度的账本视图：goal → milestone → task 树、阶段与闸口、派生关系；任务节点 MUST 提供直达对应会话的入口。
- 账本视图是唯一以树形展示结构全貌的地方。

## 新增场景

### 场景 C1：运行输出不引起侧栏重排
Given 侧栏有两个 ⣾ 运行中的任务会话 A 与 B
When A 的 codex 持续产生新输出而两者状态均未迁移
Then 侧栏顺序保持不变
And 只有当 A 运行结束或交棒到人时 A 才移位

### 场景 C2：状态先于类型
Given 编排会话「目标：导出体验」的 CEO 提案等待用户确认
When 侧栏渲染
Then 该编排会话显示 ✋ 并位于 ✋ 段
And 用户确认提案后它回落到编排会话段

### 场景 C3：冒泡只发生在折叠处
Given 项目 tranfu-site 折叠且其下有一个 ✋ 会话
And 项目 tranfu-agents-app 展开且其下有一个 ✋ 会话
When 侧栏渲染
Then tranfu-site 行尾显示 ✋1
And tranfu-agents-app 的项目行与编排会话行都不显示聚合计数
And 顶栏全局计数为 ✋2

### 场景 C4：验收卡片生成合规文本
Given 一个任务会话交棒等待真人验收且有 3 条验收语句
When 用户在验收卡片逐条选择 通过/不通过 并提交
Then 生成的消息包含每条独立一行的 `N. 通过 — 依据` 或 `N. 不通过 — 依据`
And 包含独立一行 `验收结论：通过/不通过`
And 用户没有手写任何格式

### 场景 C5：查收不读原文
Given 目标下任务 T1 验收通过、T2 运行中、T3 等人验收
When 用户打开该编排会话
Then 会话头显示 `1✓ · 1⣾ · 1✋`
And 事件流按时间列出上述里程碑事件
And 点击 T3 事件可直达 T3 会话

### 场景 C6：渐进披露
Given 任务会话里有一条 dev 的 plan-written 长响应
When 时间线渲染
Then 默认只显示角色、stage、结论与交棒行
And 展开后可见完整正文
And 用户自己的消息始终全文显示

### 场景 C7：验收通过自动沉底
Given 任务会话 T1 的验收 fact 为 passed
When 侧栏渲染
Then T1 进入「已完成」折叠分组
And 不在主列表占位
And 可从账本视图重新打开
