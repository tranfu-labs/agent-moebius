# 线框：conversation-console

基线：[docs/wireframes/pages/observer.md](../../../docs/wireframes/pages/observer.md)（现状本地只读观察页）。本 change 以全新「对话操作台」整页取代观察页：归档时新增 `docs/wireframes/pages/console.md`、移除 `pages/observer.md`，并更新 `flow.md` 的 Observer 节。

视觉风格与术语人话化规范见 [ui-design.md](ui-design.md)。下文字符图中的角色名、阶段、运行提示均按人话化对照表展示（如 `开发`、`方案已写好`、`正在运行测试…`）；协议原文（`dev`、`plan-written`、命令输出）只存在于展开层与协议层，界面控件负责双向转换。

## pages/console.md

### 主视图（桌面，打开一个运行中的任务会话）

```text
┌───────────────────────────────────────────────────────────────────────────────┐
│ Moebius            ⣾ 2 运行中 · ✋ 1 等你                            [＋ 新会话] │
├──────────────────┬──────────────────────────────────────────────┬─────────────┤
│ ▾ tranfu-agents  │ ↰ 属于:目标·导出体验          任务 T2 · 执行中│ 上下文       │
│   ✋ 失败汇总      │                                              │ 任务 T2      │
│   目标:导出体验  │ ┌ 你 · 14:02 ────────────────────────────────┐│  里程碑 M1   │
│   目标:观察页改版│ │ @开发 给卡片导出加实时进度提示               ││  阶段 执行中 │
│  »⣾ 进度提示      │ └─────────────────────────────────────────────┘│             │
│   ⣾ 失败汇总修复  │                                              │ 验收语句     │
│     导出功能重构  │ ▸ 开发 · 方案已写好 · 14:05         (点开全文)│  1✓ 2✓ 3○   │
│   ▸ 已完成 (6)    │ │ 结论:进度事件复用导出回调,新增进度条…      │             │
│                  │ │ 交给「测试」审查                             │ 运行记录     │
│ ▸ tranfu-site ✋1 │                                              │  14:05 开发 ✓│
│ ▸ agent-moebius  │ ▸ 测试 · 进行中 · 14:11                       │  14:12 测试 ✓│
│                  │ │ 结论:六项评审通过,建议补零卡片系列用例      │  14:31 开发 ⣾│
│ ──────────────── │ │ 交回「开发」实现                             │             │
│ ⌂ 账本            │                                              │ 产物         │
│                  │ ┌ 开发 · 运行中 3分12秒 ───────────── [中断] ┐│  (暂无)      │
│                  │ │ ⣾ 正在运行测试…                            ││             │
│                  │ └─────────────────────────────────────────────┘│             │
│                  ├──────────────────────────────────────────────┤             │
│                  │ ┌──────────────────────────────────────────┐ │             │
│                  │ │ @ 输入消息,@角色 交棒…            [发送] │ │             │
│                  │ └──────────────────────────────────────────┘ │             │
└──────────────────┴──────────────────────────────────────────────┴─────────────┘
```

要点：
- 单时间线多角色混排；agent 消息默认折叠为「角色 + 阶段 + 结论 + 交棒行」（`▸` 可展开全文），用户消息永远全文；角色名与阶段展示人话，stage 原文进展开层。
- 交棒渲染为消息间显式节点（折叠态并入「交棒 →」行）。
- 运行中是时间线内的活动块：角色、耗时、最近输出的人话概括（原始输出进展开层）、中断按钮。
- 任务会话头部有面包屑「↰ 属于：目标 · X」，点击回父编排会话。
- 右侧上下文面板：关联任务、验收语句逐条状态、运行记录、产物，可折叠。

### 侧栏特写（状态与排序）

```text
┌────────────────────┐
│ Moebius   ⣾2 · ✋1  │   ← 顶栏全局计数,跨项目
├────────────────────┤
│ ▾ tranfu-agents-app│   ← 第一层:项目
│   ✋ 失败汇总        │   ← ① 等你:义务,永远最上
│   目标:导出体验    │   ← ② 编排会话:查收锚点,按创建序
│   目标:观察页改版  │
│   ⣾ 进度提示        │   ← ③ 运行中:车间在动,可不看
│   ⣾ 失败汇总修复    │   ←   修复子会话同样平铺,无缩进
│     导出功能重构    │   ← ④ 静止:裸名,无任何徽标
│   ▸ 已完成 (6)      │   ← ⑤ 验收通过自动沉入折叠组
│                    │
│ ▸ tranfu-site   ✋1 │   ← 折叠项目行尾冒泡计数
│ ▸ agent-moebius    │   ← 无事的折叠项目无计数
├────────────────────┤
│ ⌂ 账本              │   ← 结构视图入口
└────────────────────┘
```

要点：
- 会话行一行制：状态图标 + 名称；状态三态 ✋ / ⣾ / 无，**没有未读**。
- 排序 ✋ > 目标 > ⣾ > 静止 > 已完成；状态先于类型（编排会话 ✋ 时升 ✋ 段）。
- 只在状态迁移时移位，运行输出流动不重排；状态只从被折叠不可见处向上冒泡。

### 编排会话（查收点）

```text
┌ 目标:导出体验 ──────────────────── 任务 1✓ · 1⣾ · 1✋ ┐
│                                                        │
│ ┌ 你 ──────────────────────────────────────────┐      │
│ │ @CEO 我想优化导出体验                          │      │
│ └───────────────────────────────────────────────┘      │
│ ▸ CEO · 采访(4 问) / 你的回答 / ◎ 提案已确认           │
│                                                        │
│ ── 进展 ────────────────────────────────────────       │
│ ◇ 已创建 3 个任务会话:T1 · T2 · T3          昨天       │
│ ◇ T1 一键导出:验收通过(测试 走查 3/3)     2 小时前   │
│ ◇ T3 失败汇总:验收未过 → 已派生修复会话     1 小时前   │
│ ◇ T2 进度提示:开发 运行中                   12 分钟前  │
│ ◇ ✋ T3 修复完成,等你验收                    刚刚 → 去验收│
│                                                        │
│ ┌ @ 对 CEO 说点什么…                          [发送] ┐ │
└────────────────────────────────────────────────────────┘
```

要点：
- 上半段是人与 CEO 的真实对话（采访、提案、确认），历史部分默认折叠。
- 「进展」是子会话里程碑事件自动回流的紧凑事件流，每条一行、可点击跳转子会话。
- 会话头常驻进展摘要。查收 = 扫会话头 + 事件流，不读子会话原文。

### 「轮到你了」验收卡片（✋ 会话时间线底部）

```text
┌ ✋ 轮到你了 · 「用户代表」请你验收 ──────────────────────┐
│ 按验收语句逐条走查:                                     │
│  1. 跑 export-all → 产出 12 个文件       [通过] [不通过] │
│  2. 空系列 → 友好提示且退出码 0          [通过] [不通过] │
│  3. 部分失败 → 跳过并汇总                [通过] [不通过] │
│ 依据(选填):[______________________________]           │
│        [提交验收结果]      [先不验,回复别的]            │
└──────────────────────────────────────────────────────────┘
```

要点：提交时由界面生成协议合规走查文本（`N. 通过 — 依据` 每条一行 + 独立一行 `验收结论：通过/不通过`），人不手写格式。

### CEO 提案确认卡片（编排会话内）

```text
┌ ◎ CEO · 阶段提案 ────────────────────────────────────────┐
│ 里程碑:M1 导出体验 → M2 分享链路                         │
│ 阶段一任务:                                              │
│  T1 一键导出全系列(验收 3 条)   T2 进度提示(验收 2 条) │
│        [确认提案,建任务会话]      [有意见,继续聊]       │
└───────────────────────────────────────────────────────────┘
```

要点：确认按钮生成合规确认回复（含提案 key），取代手写。

### 输入框 @ 补全

```text
┌──────────────────────────────────────────┐
│ @▌                                       │
│ ┌────────────────────────┐               │
│ │ @开发        写代码     │               │
│ │ @测试        审查方案   │               │
│ │ @技术负责人  技术决策   │               │
│ └────────────────────────┘               │
└──────────────────────────────────────────┘
```

要点：@ 补全只列合法可触发角色；一条消息至多一个合法 mention，由输入框约束。面板显示中文角色名与一句职责，选中后由控件在协议层生成合法英文 mention，人不接触英文句柄。

### 账本视图（项目维度）

```text
┌ tranfu-agents-app · 账本 ───────────────────────────────────┐
│ 目标:导出体验                  阶段一:执行中(1/3 已验收) │
│ ├ 里程碑 M1 导出体验                                        │
│ │  ├ T1 一键导出   ✓ 验收通过          [打开会话]           │
│ │  ├ T2 进度提示   ⣾ 开发 运行中        [打开会话]           │
│ │  └ T3 失败汇总   ✋ 等你验收           [打开会话]           │
│ │     └ 修复会话   ⣾ 运行中             [打开会话]           │
│ ├ 里程碑 M2 分享链路(未开始)                               │
│ └ 闸口:阶段一 → 阶段二,需你做集成验收                     │
└─────────────────────────────────────────────────────────────┘
```

要点：唯一以树形展示全结构的地方；任务节点直通会话；闸口与派生关系在此可见。

## flow.md 流转说明（归档时回流）

```text
console 页面导航
  侧栏会话行 ──────────→ 中栏对话时间线
  任务会话头面包屑 ────→ 父编排会话
  编排会话事件流条目 ──→ 对应子会话
  ⌂ 账本 → 项目账本树 → [打开会话] → 对应会话
  顶栏计数只显示,不是独立视图(第一版无收件箱页)

呈现层数据方向(实现通道由对接设计裁决)
  读:项目清单、会话时间线、运行状态与输出尾行、账本投影、验收进度、运行记录、产物
  写:用户消息(@ 交棒)、验收走查提交、提案确认、中断请求
```
