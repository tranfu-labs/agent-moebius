# 线框：conversation-console

基线：[docs/wireframes/pages/observer.md](../../../docs/wireframes/pages/observer.md)（现状本地只读观察页）。本 change 以全新「对话操作台」整页取代观察页：归档时新增 `docs/wireframes/pages/console.md`、移除 `pages/observer.md`，并更新 `flow.md` 的 Observer 节。

视觉风格与术语人话化规范见 [ui-design.md](ui-design.md)。下文字符图中的角色名、阶段、运行提示均按人话化对照表展示（如 `开发`、`方案已写好`、`正在运行测试…`）；协议原文（`dev`、`plan-written`、命令输出）只存在于展开层与协议层，界面控件负责双向转换。

## pages/console.md

### 主视图（桌面，打开一个运行中的任务会话）

```text
┌───────────────────────────────────────────────────────────────────────────────┐
│ Moebius            ⣾ 2 运行中 · ✋ 1 等你 ▾                          [＋ 新会话] │
├──────────────────┬──────────────────────────────────────────────┬─────────────┤
│ ▾ tranfu-agents  │ ↰ 属于:目标·导出体验          任务 T2 · 执行中│ 产物         │
│   ✋ 失败汇总      │                                              │ ┌─────────┐ │
│   目标:导出体验  │ ┌ 你 · 14:02 ────────────────────────────────┐│ │进度条截图│ │
│   目标:观察页改版│ │ @开发 给卡片导出加实时进度提示               ││ └─────────┘ │
│  »⣾ 进度提示      │ └─────────────────────────────────────────────┘│  (点开预览) │
│   ⣾ 失败汇总修复  │                                              │             │
│     导出功能重构  │ ▸ 开发 · 方案已写好 · 14:05         (点开全文)│ 验收语句     │
│   ▸ 已完成 (6)    │ │ 结论:进度事件复用导出回调,新增进度条…      │  1✓ 2✓ 3○   │
│                  │ │ 交给「测试」审查                             │             │
│ ▸ tranfu-site ✋1 │                                              │ 运行记录     │
│ ▸ agent-moebius  │ ▸ 测试 · 进行中 · 14:11                       │  14:05 开发 ✓│
│                  │ │ 结论:六项评审通过,建议补零卡片系列用例      │  14:12 测试 ✓│
│ ──────────────── │ │ 交回「开发」实现                             │  14:31 开发 ⣾│
│ ⌂ 账本            │                                              │             │
│                  │ ┌ 开发 · 运行中 3分12秒 ───────────── [中断] ┐│ 任务信息     │
│                  │ │ ✓ 1. 写进度事件桥      ✓ 2. 接进度条组件   ││  任务 T2     │
│                  │ │ ⣾ 3. 正在运行测试…     ○ 4. 自测走查       ││  里程碑 M1   │
│                  │ │ (点开任一步看原始输出)                      ││  阶段 执行中 │
│                  │ └─────────────────────────────────────────────┘│             │
│                  ├──────────────────────────────────────────────┤             │
│                  │ ┌──────────────────────────────────────────┐ │             │
│                  │ │ @ 输入消息,@角色 交棒…            [发送] │ │             │
│                  │ └──────────────────────────────────────────┘ │             │
│                  │  发消息不会打断运行 · 开发在第 3 步            │             │
└──────────────────┴──────────────────────────────────────────────┴─────────────┘
```

要点：
- 单时间线多角色混排；agent 消息默认折叠为「角色 + 阶段 + 结论 + 交棒行」（`▸` 可展开全文），用户消息永远全文；角色名与阶段展示人话，stage 原文进展开层。
- 交棒渲染为消息间显式节点（折叠态并入「交棒 →」行）。
- 运行中是时间线内的活动块：角色、耗时、**计划步骤逐条推进**（✓ 已完成 / ⣾ 进行中 / ○ 未开始，进行中步骤配人话概括）、中断按钮；点开任一步看该步原始输出。拿不到步骤数据时降级为单行人话概括（「正在运行测试 · 已进行 3 分 12 秒」），宁可单行也不得空白。
- 任务会话头部有面包屑「↰ 属于：目标 · X」，点击回父编排会话。
- 右侧上下文面板以**产物为主体**：产物预览置顶（缩略图 / 文件卡，点开大预览——普通用户关心成果，不关心管理字段），之下验收语句逐条状态、运行记录，任务号 / 里程碑 / 阶段沉底；可折叠。产物为空时整区收起，不放「（暂无）」占位。
- 输入框下方常驻一行等待状态文案，陈述「现在发消息会怎样」（「发消息不会打断运行 · 开发在第 3 步」「等你验收，也可以先回复别的」），消除普通用户「说话会不会打断它」的顾虑。

### 侧栏特写（状态与排序）

```text
┌────────────────────┐
│ Moebius   ⣾2 · ✋1  │   ← 顶栏全局计数,跨项目
├────────────────────┤
│ ▾ tranfu-agents-app│   ← 第一层:项目
│   ✋ 失败汇总        │   ← ① 等你:义务,永远最上
│   目标:导出体验    │   ← ② 编排会话:查收锚点,按创建序
│   目标:观察页改版  │
│   ⣾ 进度提示        │   ← ③ 运行中:车间在动,可不看
│   ⣾ 失败汇总修复    │   ←   修复子会话同样平铺,无缩进
│     导出功能重构    │   ← ④ 静止:裸名,无任何徽标
│   ▸ 已完成 (6)      │   ← ⑤ 验收通过自动沉入折叠组
│                    │
│ ▸ tranfu-site   ✋1 │   ← 折叠项目行尾冒泡计数
│ ▸ agent-moebius    │   ← 无事的折叠项目无计数
├────────────────────┤
│ ⌂ 账本              │   ← 结构视图入口
└────────────────────┘
```

要点：
- 会话行一行制：状态图标 + 名称；状态三态 ✋ / ⣾ / 无，**没有未读**。
- 排序 ✋ > 目标 > ⣾ > 静止 > 已完成；状态先于类型（编排会话 ✋ 时升 ✋ 段）。
- 只在状态迁移时移位，运行输出流动不重排；状态只从被折叠不可见处向上冒泡。

### 顶栏「等你」清单浮层

```text
│ Moebius            ⣾ 2 运行中 · ✋ 2 等你 ▾ │   ← 点「等你」计数展开
│              ┌ 等你 (2) ────────────────────────────┐
│              │ ✋ 失败汇总 · 等你验收    tranfu-agents │
│              │ ✋ 首页文案 · 提案等确认  tranfu-site   │
│              └─────────────────────────────────────┘
```

要点：
- 跨项目汇聚所有「等你」事项，一行 = 图标 + 会话名 + 一句义务描述 + 项目名，点行直达会话。
- 它是跳转菜单不是收件箱页——不承载已读状态、不分页、无操作按钮，兑现 design.md「顶栏计数 + 侧栏排序两个入口」的表述。
- 「N 运行中」计数纯文字不可点：义务给出口，车间不给召唤。

### 编排会话（查收点）

```text
┌ 目标:导出体验 ──────────────────── 任务 1✓ · 1⣾ · 1✋ ┐
│                                                        │
│ ┌ 你 ──────────────────────────────────────────┐      │
│ │ @CEO 我想优化导出体验                          │      │
│ └───────────────────────────────────────────────┘      │
│ ▸ CEO · 采访(4 问) / 你的回答 / ◎ 提案已确认           │
│                                                        │
│ ── 进展 ────────────────────────────────────────       │
│ ◇ 已创建 3 个任务会话:T1 · T2 · T3          昨天       │
│ ◇ T1 一键导出:验收通过(测试 走查 3/3)     2 小时前   │
│ ◇ T3 失败汇总:验收未过 → 已派生修复会话     1 小时前   │
│ ◇ T2 进度提示:开发 运行中                   12 分钟前  │
│ ◇ ✋ T3 修复完成,等你验收                    刚刚 → 去验收│
│                                                        │
│ ┌ @ 对 CEO 说点什么…                          [发送] ┐ │
└────────────────────────────────────────────────────────┘
```

要点：
- 上半段是人与 CEO 的真实对话（采访、提案、确认），历史部分默认折叠。
- 「进展」是子会话里程碑事件自动回流的紧凑事件流，每条一行、可点击跳转子会话。
- 会话头常驻进展摘要。查收 = 扫会话头 + 事件流，不读子会话原文。

### 「轮到你了」验收卡片（✋ 会话时间线底部）

```text
┌ ✋ 轮到你了 · 「用户代表」请你验收 ──────────────────────────┐
│ 改了什么: 导出时新增实时进度提示;失败文件跳过并计入汇总     │
│ 已自测:   测试走查 12 项用例全部通过         (点开看记录)    │
│                                                              │
│ 按验收语句逐条走查:                                         │
│  1. 跑 export-all → 产出 12 个文件           [通过] [不通过] │
│     └ 证据: 产物「导出结果(12 文件)」[打开] · 自测 ✓        │
│  2. 空系列 → 友好提示且退出码 0              [通过] [不通过] │
│     └ 证据: 自测记录第 8 项 ✓                (点开看输出)    │
│  3. 部分失败 → 跳过并汇总                    [通过] [不通过] │
│     └ 证据: 产物「失败汇总示例」[打开] · 自测 ✓             │
│ 依据(选填):[______________________________]                │
│        [提交验收结果]      [先不验,回复别的]                │
└──────────────────────────────────────────────────────────────┘
```

要点：
- **证据先于裁决**：卡头先给「改了什么」（人话摘要）与「已自测」（agent 走查记录入口），每条验收语句下挂证据行（关联产物直达 + agent 自测结果），人在卡内即可完成判断，不需要回读时间线取证。证据行的产物与右侧面板指向同一对象。
- 某条语句没有可挂证据时，该行降级为裸验收语句——宁可裸也不得编造证据。
- 「在我电脑上重跑验收命令」的入口依赖与 runner 的对接设计（见 design.md 风险节），本版只预留证据行位置，不承诺重跑能力。
- 提交时由界面生成协议合规走查文本（`N. 通过 — 依据` 每条一行 + 独立一行 `验收结论：通过/不通过`），人不手写格式。

### CEO 提案确认卡片（编排会话内）

```text
┌ ◎ CEO · 阶段提案 ────────────────────────────────────────┐
│ 里程碑:M1 导出体验 → M2 分享链路                         │
│ 阶段一任务:                                              │
│  T1 一键导出全系列(验收 3 条)   T2 进度提示(验收 2 条) │
│        [确认提案,建任务会话]      [有意见,继续聊]       │
└───────────────────────────────────────────────────────────┘
```

要点：确认按钮生成合规确认回复（含提案 key），取代手写。

### 输入框 @ 补全

```text
┌──────────────────────────────────────────┐
│ @▌                                       │
│ ┌────────────────────────┐               │
│ │ @开发        写代码     │               │
│ │ @测试        审查方案   │               │
│ │ @技术负责人  技术决策   │               │
│ └────────────────────────┘               │
└──────────────────────────────────────────┘
```

要点：@ 补全只列合法可触发角色；一条消息至多一个合法 mention，由输入框约束。面板显示中文角色名与一句职责，选中后由控件在协议层生成合法英文 mention，人不接触英文句柄。

### 账本视图（项目维度）

```text
┌ tranfu-agents-app · 账本 ───────────────────────────────────┐
│ 目标:导出体验                  阶段一:执行中(1/3 已验收) │
│ ├ 里程碑 M1 导出体验                                        │
│ │  ├ T1 一键导出   ✓ 验收通过          [打开会话]           │
│ │  ├ T2 进度提示   ⣾ 开发 运行中        [打开会话]           │
│ │  └ T3 失败汇总   ✋ 等你验收           [打开会话]           │
│ │     └ 修复会话   ⣾ 运行中             [打开会话]           │

│ └ 闸口:阶段一 → 阶段二,需你做集成验收                     │

```

要点：唯一以树形展示全结构的地方；任务节点直通会话；闸口与派生关系在此可见。

## flow.md 流转说明（归档时回流）

```text
console 页面导航
  侧栏会话行 ──────────→ 中栏对话时间线
  任务会话头面包屑 ────→ 父编排会话
  编排会话事件流条目 ──→ 对应子会话
  ⌂ 账本 → 项目账本树 → [打开会话] → 对应会话
  顶栏「等你」计数 → 跨项目清单浮层 → 点行直达会话(跳转菜单,仍无独立收件箱页)
  顶栏「运行中」计数只显示,不可点

呈现层数据方向(实现通道由对接设计裁决)
  读:项目清单、会话时间线、运行状态与输出尾行、账本投影、验收进度、运行记录、产物
  写:用户消息(@ 交棒)、验收走查提交、提案确认、中断请求
```
