# UI 生成指令：conversation-console demo

给 HTML 生成器的提示词。本文件不复述信息架构与视觉规范，只做四件那两份做不到的事：
拆掉线框的假精度、钉死全局骨架、排序视觉分量、给出自检闭环。

## 0. 任务与输入

- 输入三份，优先级从高到低：**本文件（构图/权重/禁令/自检）> [ui-design.md](ui-design.md)（视觉语义变量、逐组件风格）> [wireframes.md](wireframes.md)（信息架构、文案、状态）**。
- 产出：单个自包含 HTML，样式内联，系统深浅双模同一套语义变量翻转。
- 主帧 = 「打开一个运行中的任务会话」这一屏（wireframes.md 主视图）。
- 冲突时以本文件为准；尤其当线框的盒子形状与第 2 节骨架矛盾时，听第 2 节。

## 1. 怎么读线框（第一条，最重要：拆假精度）

- 线框里的盒子宽高、边框字符、对齐，**只表达拓扑**（谁挨着谁、谁包含谁），不表达比例、间距、字号。
- **不许把字符数当真值**。侧栏 18 字符、右栏 13 字符是等宽画图的副作用，不是设计意图。列宽/行高一律听第 2 节与 ui-design 字号表。
- 线框里的 ✋ ⣾ ○ 是语义占位，渲染为 ui-design §5 的描边图标 + 呼吸点，不许把符号本身搬上界面。

## 2. 钉死的全局骨架（线框不负责、三版 demo 每次都跑偏的地方）

以下数值是**钉死的起点**：可整体等比缩放，但比例关系与顺序不许改。

- 三栏：左侧栏固定 **260px**；中栏自适应、内容最大宽 **720px 居中**；右侧上下文栏固定 **300px**、可折叠。
- 中栏纵向结构，固定顺序，只有这三段：**面包屑行 → 单列时间线（消息/交棒/运行块混排）→ 底部常驻输入区（输入框 + 下方状态行）**。
- **中栏顶部禁止自造「当前任务英雄卡 / 步骤概览卡 / 欢迎横幅」**。任务信息只活在右栏。时间线从第一条消息直接开始。
- 垂直节奏：消息块间距 24px；卡片内边距 12×16；时间线左侧头像栏宽 32px。

## 3. 视觉分量阶梯（线框拉平了、必须显式排序的地方）

从强到弱。强 = 更大字号 / 更重字重 / 更多留白 /（仅验收卡）允许用色：

1. **「轮到你了」验收卡** —— 全页最明确的结构义务出口：普通中性白卡、位于时间线底部，卡头给证据，提交按钮用 indigo。
2. **用户消息** —— 白底卡片，分量高于 agent 消息（人是闸口）。
3. **运行块** —— 白卡 + 步骤清单，有存在感但不压过验收卡。
4. **右栏「产物预览」** —— 右栏的主体/英雄：缩略图给足尺寸（至少占满右栏宽、有真实预览感），不是一行小卡。
5. **agent 折叠消息** —— 无卡片、缩进文本行；低分量本身在说「可以不读」。
6. **右栏「验收语句 / 运行记录」** —— 次级列表。
7. **右栏「任务信息」（任务号/里程碑/阶段）** —— 全页最弱：小字、灰、沉底。

**反例（必须避免）**：把右栏四段做成等权重堆叠列表。这是三版 demo 都犯的错——
执行了「产物放最上面」的顺序，却没执行「产物是主体」的权重。产物必须明显大于其它三段。

## 4. 视觉锚（用真实竞品定基线，别用形容词）

- 生成前请附上 **Linear 的 issue 列表** 与 **Notion 的页面** 两张真实截图作为参考图（本 md 无法内嵌，由发起人附上）。「安静工作台风」写成字就是中位数，喂真实截图才是锚。
- 无参考图时用可测代理：侧栏行高 36；正文/次要/提示三级灰阶拉开到肉眼可辨；边框 ≤1px、靠底色差分区不靠粗线；常态零阴影，只有浮层一层柔和投影。
- 「安静」的判定：安静态截图里，除头像圆与靛紫控件外不应有任何彩色。

## 5. 生成哪些态（别把状态机压成一张海报）

- 主帧：运行中的任务会话（含验收卡）。
- 同一 HTML 内置一个角落切换器，至少可切：**安静态（无任何等你）、编排会话（查收点）、空态**，以及 **浅 / 深** 主题。
- 切换器是 demo 自检工具，不是产品功能，样式上要弱、可无视。

## 6. 禁令（直接掐掉中位数生成的坏习惯）

- 一屏实心强调按钮 ≤ 1，其余描边/幽灵。
- 无红点、无角标脉冲、无「未读」；「等你」不用专属色相，靠置顶排序、顶栏计数、清单浮层与验收卡位置承载注意力。
- 无渐变、无装饰性阴影、无等宽字体铺面（仅展开层的命令/日志原文用等宽）。
- 不用表情符号做功能图标；状态一律描边图标 + 呼吸点。
- 不自造规范里没有的模块（英雄卡、进度大数字、欢迎语）。

## 7. 生成后自检（逐条过，不过就重画，别直接交付）

- 逐条走 ui-design.md §9 的 13 条断言，全部应成立。
- 额外三条硬检查：
  1. 右栏产物预览的视觉尺寸明显大于验收语句/运行记录/任务信息三段。
  2. 中栏顶部没有自造英雄卡，时间线从第一条消息开始。
  3. 三栏列宽 = 260 / 自适应(≤720) / 300。
