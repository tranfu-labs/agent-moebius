# 设计：main-conversation-evidence-outlets

## 基线是「对话开始时的提交」，必须落库而不是现算

结果卡片回答的是「这段对话期间改了几个文件」，右侧栏之后要用同一个基线回答「改了什么」。基线必须在**对话开始那一刻**就记下来，理由是它事后无法重建：项目文件夹的 HEAD 会随用户自己的提交移动，等到一轮结束再去问「对话开始时是哪个提交」已经没有答案。

因此在会话创建（首条消息落库）时持久化基线提交，与工作空间模式同一时机。这也解释了为什么 `main-conversation-workspace-lock` 必须成立：基线只有在一段对话全程待在同一个地方时才有意义，两片是同一条产品逻辑的两端。

## 计数覆盖两种工作空间，不能只做 worktree

现有 `affectedFiles` 只在 worktree 模式且带 `code-verified` 标记时生成，覆盖不到默认工作空间——而默认工作空间恰恰是首次使用时的默认值，也是最需要给用户证据的那种。计数按模式分流：

- 默认工作空间：统计项目文件夹相对基线提交的改动，含已提交与未提交两部分（用户要知道的是「这段对话期间这个文件夹变了几个文件」，不是「团队提交了几个文件」）。
- 独立工作空间：同样的口径，统计对象换成隔离副本。
- 非 Git 项目：返回不可用，卡片不出现。PRD 已明确这类项目没有改动视图。

**不做改动归因。** 产品包的是第三方 CLI，拿不到工具级事件，无法区分改动来自成员还是用户自己动手。措辞如实说「这段对话期间有几个文件发生改动」。这条与右侧栏 PRD 同源，猜错一次归因就毁掉全部信任。

## 「一轮结束」的判据取「无成员接力」，不取「某一步跑完」

PRD 说的是「一轮工作结束、没有成员在继续接力时」。判据不能是单个步骤结束——团队内部交棒时每一步都会结束，按步骤出卡片会让时间线被卡片铺满，正是 PRD 要避免的噪音。取「没有任何成员在工作且没有待处理的交棒」，与状态点「蓝点」的判据同源（`status-dot.ts` 已有 `isRunning` 与 `lastMessageMentionsAgent` 两个事实），复用同一组事实，不另起一套。

## 完整输出入口与开发者诊断是两件事

当前 `operator-console.tsx` 只有一个 `onOpenDiagnostics`，服务的是「操作台遇到问题，请打开开发者诊断查看日志」。它和 PRD 的「完整输出」在语义、触发条件和目标区域上都不同：前者是产品自身故障时的兜底，后者是用户随时可以调阅的某一步的过程记录。

**不复用同一个回调**。复用会让两者在右侧栏就绪后纠缠不清，也会让「查看日志」这句带机器味的文案泄漏到正常路径上。新增独立的打开意图，携带足以定位到那一步的信息（会话、运行、成员），由桌面层决定落到哪。

## 历史记录上的入口靠数据而不是靠组件状态

「这一步结束后『完整输出』保留在历史记录上」意味着入口不能挂在实时运行块上——运行块结束就消失了。入口要挂在时间线的历史记录上，其可用性取决于该步骤的输出是否仍可取回。因此：

- 运行中：操作条同时给「停下」和「完整输出」。
- 结束后：`RunBlock` 整体消失（C 片已交付的 #11 行为不动），历史记录条目上保留「完整输出」。
- 四种事实的记录同样带该入口——用户最需要看原始输出的时刻正是「这一步没跑起来」。

注意 PRD 待讨论里「出问题时按需展开的原始错误输出」指的是**展开形态与截断策略**，不是入口本身是否存在。本片只交付入口与打开意图，不定义右侧栏里怎么呈现、超长如何截断——那两件事分别属右侧栏 PRD 与待讨论。

## 降级显示的边界

右侧栏多标签能力就绪前，打开意图接现有 `sub-session-panel.tsx` 显示单一内容。**MUST NOT** 在本片里长出标签条、类型选择、文件树或加号新建——那是右侧栏 PRD 的定义范围，抢先实现会制造第二个事实源，也会让右侧栏里程碑开工时面对一份需要先拆除的实现。这正是 `main-conversation-workspace-lock` 正在付的代价，不重犯。

降级期间的取舍：一次只能看一个，打开改动会替换掉正在看的过程内容。接受——这是临时形态，且比一个抢跑的半成品标签条便宜。

## 验收方式

- 纯逻辑走 Vitest：基线提交落库与幂等、两种工作空间的计数口径、非 Git 返回不可用、零改动与有改动的分支、「一轮结束」判据与蓝点事实的一致性。
- 页面表现走真实桌面窗口（CDP `9222`）：跑一轮 → 卡片出现且数字与实际改动一致；跑一轮不改任何文件 → 卡片如实说明；非 Git 项目 → 无卡片；运行中看到「完整输出」，步骤结束后它仍在历史记录上并可再次打开。
